🆘 高频痛点急救包

需求变更、资源冲突、进度延期 — 游戏项目管理最常遇到的3大问题，一键直达解决方案

🔄
需求频繁变更
🔴 TOP 1 痛点 · 出现率 87%

策划临时加需求、改需求，美术已完成的工作被推翻，团队情绪崩溃

💊 急救方案：
① 建立变更评审机制（CR流程）
② 制作变更成本速查表给策划
③ 设定风格冻结节点写入项目章程
📝 需求对接规范 → 💥 跨部门冲突案例 → 🔄 需求变更案例 →
⚔️
资源冲突与抢人
🟡 TOP 2 痛点 · 出现率 72%

多项目并行抢美术资源，产能不足时外包与自研如何平衡分配

💊 急救方案：
① 建立产能负载可视化看板
② 使用外包 vs 自研决策矩阵辅助判断
③ 提前做资源预分配（Sprint 规划时锁定）
⚖️ 外包vs自研决策 → 📅 排期管理 → 📊 外包工作量模型 →
⏰
进度延期与排期爆炸
🟠 TOP 3 痛点 · 出现率 68%

排期过于乐观、依赖链断裂、加班恶性循环，项目进度失控

💊 急救方案：
① 排期必须加20% 缓冲（Murphy定律）
② 建立延期应急决策树快速响应
③ 使用燃尽图每日监控进度偏差
🚨 延期应急方案 → ⏰ 延期案例复盘 → 📈 进度可视化 →
