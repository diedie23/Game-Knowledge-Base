自动 Mask 通道生成器 v2.0 🟢 在线

LAB 色彩空间 K-Means++ 聚类 · 一键生成换色遮罩图
🔥
解决的实际问题：角色换色 Mask 遮罩图过去全靠美术手工在 PS 中逐通道绘制，一张图平均耗时 30~60 分钟。外包 CP 经常交付通道互斥出错、边缘硬切的 Mask，返工率高达 40% 以上。
✅
现在：上传原图后 3 秒内自动完成颜色区域识别与通道分配，内置自动修复（纯黑/纯白/通道冲突检测+修正），零 PS 基础的外包也能产出合格 Mask。
❌ BEFORE
  • 手工 PS 绘制每张 30~60 min
  • 外包 CP 不理解通道互斥，返工频繁
  • 验收全靠肉眼，无法量化质量
  • 纯黑/纯白像素导致 HSV 染色失效
✅ AFTER
  • 上传原图 → 3 秒自动生成
  • 算法保证通道互斥，输出即合规
  • 内置质量评分 + 自动修复
  • 支持 RGBA / RGB 双模式，适配不同引擎
📤 外包 CP 资源交付 👤 角色换色系统 🔄 批量资源生产 ✅ 美术验收

  • 零学习成本，打开即用

矩形 / 套索 / 魔棒选区 · 拓扑延展 · Atlas 导出
🔥

✅
现在：上传原画后用选区工具（矩形/套索/魔棒）圈选部件，工具自动裁切并执行拓扑延展（内容感知/边缘扩展/镜像/纯色 4 种模式），最终导出含 Atlas 图集 + Spine JSON 配置的 ZIP 包。
❌ BEFORE
  • PS 手动裁切+补边，一张角色 2~4 小时
  • 边缘延展全靠经验，不同人结果差异大
  • 手动拼 Atlas 图集容易坐标出错
  • 外包对 Spine 格式理解偏差，返工率高
✅ AFTER
  • 可视化选区圈选，15~30 分钟完成拆分
  • 4 种拓扑延展模式，效果标准化
  • 自动生成 Atlas 图集 + POT 尺寸
  • 直出 Spine 兼容 JSON，零配置导入

贴图规范检测器 🟡 规划中

批量检测 PNG/TGA 尺寸、Alpha、色彩规范等
🔥
解决的实际问题：外包交付的贴图资源经常出现尺寸非 2^N、缺少 Alpha 通道、JPEG 伪装成 PNG、颜色空间错误等问题。人工逐张检查效率极低，漏检造成后期集成阶段才暴露问题。
💡
规划方案：批量拖入贴图文件，自动检测尺寸/格式/Alpha/色彩空间等规范项，生成检测报告，支持导出 CSV 作为验收记录。
❌ BEFORE
  • 手动打开每张图在 PS 中查看属性
  • 100 张图验收需要 1~2 小时
  • 漏检在引擎集成阶段才暴露
  • 验收标准靠口头约定，无客观记录
✅ AFTER（规划）
  • 批量拖入 → 秒级全量扫描
  • 红/黄/绿三级问题标记
  • 导出 CSV 检测报告，作为验收凭证
  • 自定义检测规则，适配不同项目
📋
APM 应用场景：外包交付验收节点，将检测报告附在验收单中，作为资源质量的客观依据。可大幅缩短验收周期，减少主观误判。

美术资源批量重命名 🟡 规划中

按命名规则批量重命名 · 预览 · 冲突检测
🔥
解决的实际问题：多 CP 并行时命名规范难统一，常见问题：中文文件名/空格/特殊字符/大小写混乱/编号不连续。引擎导入后因命名冲突导致资源丢失，排查代价极高。
💡
规划方案：拖入文件列表后，按自定义模板（类型_性别_部件_编号_变体）批量重命名，预览更改，检测名称冲突，导出命名对照表。
❌ BEFORE
  • 手动逐个改名，100 个文件耗时 30min+
  • 不同 CP 命名风格各异
  • 中文/空格导致引擎报错
  • 编号跳跃或重复，缺乏全局校验
✅ AFTER（规划）
  • 拖入 → 选模板 → 一键重命名
  • 强制统一命名规范
  • 自动过滤非法字符
  • 导出新旧命名对照表
📋
APM 应用场景：在资源入库环节前置使用，确保所有资源命名符合项目规范。配合出图规范文档中的命名章节使用。

🔄

资源版本对比器 🟡 规划中

图像 Diff · 像素级差异高亮 · 变更记录
🔥
解决的实际问题：美术资源多版本迭代中，肉眼难以确认"这次修改了哪里"。CP 说已修改但其实没改、或改了不该改的地方，缺乏快速可视化对比手段。
💡
规划方案：上传旧版/新版两张图，自动生成差异热力图 + 滑动对比 + 逐像素 Diff，支持导出对比报告。
❌ BEFORE
  • PS 叠图肉眼对比，微调难以察觉
  • CP 版本声称已改，无法快速验证
  • 无变更记录，责任追溯困难
✅ AFTER（规划）
  • 上传两张 → 秒级像素 Diff
  • 差异区域热力图高亮
  • 滑块拖动对比 + 闪烁切换模式
  • 导出变更报告，留档可追溯
📋
APM 应用场景：每次外包修改交付时使用，客观记录每个版本的差异。可作为验收依据，也可用于复盘"修改了几轮才达标"的效率分析。
