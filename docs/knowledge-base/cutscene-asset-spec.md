## 1. 概述与适用范围

本规范定义了游戏项目中所有**过场演出（Cutscene / Cinematic）**资产的制作标准、交接流程、版本管理和质量门禁，覆盖实时演出（引擎内 Sequencer）与离线预渲染（CG/PV）两大类型。

### 适用场景

| 场景类型 | 引擎 | 典型案例 |
|---------|------|---------|
| 主线剧情过场 | UE5 Sequencer / Unity Timeline | 章节转场、BOSS 登场、结局动画 |
| 角色必杀技演出 | 引擎实时 | 大招释放 → 特写镜头 → 慢动作回放 |
| 抽卡/获得演出 | 引擎实时 + 后期合成 | SSR 角色抽取 → 全屏演出 → 立绘展示 |
| PV / CG | 离线渲染（Maya/Blender → AE/Nuke） | 宣传 PV、版本更新 CG |
| 剧情对话演出 | 引擎内轻量 Sequence | NPC 对话 → 表情/动作切换 → 镜头微运动 |
| 过场转场（Loading） | 引擎实时 / 预渲视频 | 地图加载过渡动画 |

**核心原则：** 演出是玩家情感体验的"放大镜"——所有规范的终极目标是保障**叙事节奏感**和**技术稳定性**的统一。

## 2. 过场演出类型分类体系

### 2.1 按制作方式分类

| 类型 | 代号 | 技术方案 | 典型时长 | 参考工期 |
|------|------|---------|---------|---------|
| 实时引擎演出（A 类） | RT | UE5 Level Sequence / Unity Playable | 15s ~ 3min | 3~10 人天/分钟 |
| 预渲染 CG（B 类） | CG | Maya/Blender → After Effects/Nuke | 30s ~ 5min | 15~40 人天/分钟 |
| 2D+3D 混合演出（C 类） | HY | Spine/Live2D + 引擎粒子 + 后期合成 | 10s ~ 1min | 5~15 人天/分钟 |
| 轻量对话演出（D 类） | DLG | 引擎 Timeline + 表情/动作数据驱动 | 5s ~ 30s | 0.5~2 人天/段 |

### 2.2 按叙事功能分类

| 功能标签 | 说明 | 优先级 | 可跳过 |
|---------|------|--------|--------|
| 🎭 主线剧情 | 推动主线叙事、关键剧情节点 | P0 | 首次不可跳过 |
| ⚔️ 战斗演出 | 必杀技、QTE、处决动画 | P0 | 可设置关闭 |
| 🎁 获得演出 | 抽卡、装备获得、成就解锁 | P1 | 可加速/跳过 |
| 💬 对话演出 | NPC 对话、支线任务过场 | P2 | 可跳过 |
| 🌄 环境演出 | 场景转场、天气变化、时间流逝 | P2 | 可跳过 |
| 📺 宣传 PV/CG | 版本宣传、活动预告片 | P1 | 可跳过 |

## 3. 制作全流程与状态机

### 3.1 制作流程总览

```
分镜策划 → 分镜绘制 → [分镜评审] → Layout → 动画制作 → 特效/光照 → 音频集成 → [技术审查] → 打包出包 → [终验收]
```

### 3.2 演出资产状态机

- **Draft** — 分镜/脚本编写中，不可引用
- **Storyboard_Review** — 分镜评审中（策划 + 导演确认）
- **Layout_WIP** — 引擎内 Layout 粗动/镜头摆放
- **Animation_WIP** — 角色/相机动画精调中
- **Polish** — 特效/光照/音频集成精修
- **Tech_Review** — 性能/兼容性技术审查
- **Approved** — 通过验收，可出包
- **Rejected** — 被驳回，标注原因后回退至对应阶段


### 4.2 命名规范

| 资产类型 | 命名规则 | 示例 |
|---------|---------|------|
| Master Sequence | `CUT_{类型}_{章节}_{序号}_Master` | `CUT_MS_C01_001_Master` |
| 相机 Sequence | `CUT_{类型}_{章节}_{序号}_Cam` | `CUT_MS_C01_001_Cam` |

**类型代号速查：** MS=主线, BT=战斗, GA=抽卡, DLG=对话, ENV=环境, PV=宣传片

## 5. 分镜文件交接 SOP

### 分镜文件必填要素

- 镜头编号（Shot_001 ~ Shot_N）
- 景别标注（ECU/CU/MS/FS/LS/ELS）
- 运镜方式（固定/推/拉/摇/移/跟/升降/环绕）
- 时长标注（每镜秒数 + 总时长）
- 台词/旁白（含语种标注）
- 表情/动作描述
- 转场方式（硬切/溶解/淡入淡出/划/自定义）

### 核心原则

分镜板是全组"合约"——未通过评审的分镜不得进入引擎制作阶段。任何分镜修改必须走变更流程。

## 6. Level Sequence 组织架构

### Master / Sub-Sequence 分层设计

```
CUT_MS_C01_001_Master (Master Sequence)
├── CUT_MS_C01_001_Cam      # 相机轨道
├── CUT_MS_C01_001_Anim     # 角色动画轨道
├── CUT_MS_C01_001_FX       # 特效轨道
├── CUT_MS_C01_001_Audio    # 音频轨道
└── CUT_MS_C01_001_Light    # 灯光轨道
```

### 锁定规则

| 子 Sequence | 负责人 | 锁定时机 |
|-------------|--------|---------|
| _Cam | 演出导演 | Layout 完成后锁定 |
| _Anim | 动画师 | 精调完成后锁定 |
| _FX | 特效师 | Polish 完成后锁定 |
| _Audio | 音频设计师 | 音频集成完成后锁定 |
| _Light | 灯光师 | 与 FX 同步锁定 |
| _Master | 演出 Lead | 终验收后锁定 |

| 景别 | 代号 | 取景范围 | FOV 参考 |

## 8. 性能预算与红线（移动端）

| 指标 | 红线值 | 目标值 |
|------|--------|--------|
| 帧率 | ≥ 30 fps | ≥ 45 fps |
| Draw Call | ≤ 600 | ≤ 400 |
| 三角面 | ≤ 200 万 | ≤ 120 万 |
| 粒子数量 | ≤ 5000 同屏 | ≤ 3000 |
| 动态灯光 | ≤ 4 | ≤ 2 |
| 内存增量 | ≤ 80 MB | ≤ 50 MB |
| 加载时间 | ≤ 3 秒 | ≤ 1.5 秒 |

## 9. 多人协作与锁规范

- 所有 `.uasset` 默认 Exclusive Checkout
- 演出资产统一在 `cinematic/dev` Stream 操作
- 单次锁定不超过 4 小时（超时自动通知）
- 提交备注格式：`[CUT] {演出ID} - {阶段} - {变更摘要}`

## 12. 常见踩坑记录

1. **Master Sequence 丢失引用** — 移动子 Sequence 文件前需先 Remove Track，移动后重新 Add
2. **低端机演出崩溃** — 粒子材质采样层数超限，需建立低配材质变体
3. **音画不同步** — Audio Event 触发点为嘴型动画起始帧 -2 帧
4. **字幕安全区** — Widget 内嵌 SafeZone 节点，底部预留 ≥ 5%
5. **Perforce 锁死** — 设置 4 小时超时自动通知机制
