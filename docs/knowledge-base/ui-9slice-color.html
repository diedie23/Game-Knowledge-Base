🧩 9宫格与颜色空间规范 UI 规范

9宫格（Nine-Slice / Nine-Patch）是 UI 中最重要的优化技术之一，合理使用可以极大减少内存占用。同时，颜色空间的正确设置直接决定了美术资源在引擎中是否"所见即所得"。

📂 UI · 界面 🎯 适用：所有引擎的 UI 开发 👥 受众：UI美术 / 技术美术 / 外包CP 📅 更新：2026-04

🧩 一、9宫格基础概念

1.1 什么是9宫格？

9宫格（Nine-Slice Scaling）是一种将图像分为 9 个区域的拉伸策略：4个角落保持原样，4条边只在一个方向拉伸，中心区域双向拉伸。这使得一张小尺寸的按钮底图可以适配任意大小，且边角不变形。

左上角
🔒 不变
上边缘
↔ 水平拉伸
右上角
🔒 不变
左边缘
↕ 垂直拉伸
中心
↔↕ 双向拉伸
右边缘
↕ 垂直拉伸
左下角
🔒 不变
下边缘
↔ 水平拉伸
右下角
🔒 不变

1.2 为什么9宫格如此重要

场景无9宫格有9宫格节省
10种尺寸的按钮 10张切图 ≈ 200KB 1张 20×20 切图 ≈ 2KB ~99%
各种大小的面板 每种单独切图 ≈ 500KB 1张小面板底图 ≈ 5KB ~99%
对话气泡 多种宽度逐个切 ≈ 100KB 1张气泡底图 ≈ 3KB ~97%
💼
APM 视角：9宫格使用率直接影响包体大小。一个中型小游戏如果不用9宫格，UI 资源可能占 5~10MB；合理使用后可压到 1~2MB。外包交付时务必要求标注哪些资源为9宫格资源，并验证拉伸后的效果。

📐 二、9宫格制作规范

2.1 切图要求

要素规范原因
尺寸 尽量小！只需包含完整的边角装饰 中心和边缘会被拉伸，大尺寸无意义
角落区域 包含完整的圆角/装饰，不可截断 角落不会拉伸，截断会导致缺失
边缘区域 必须是可重复/可拉伸的纯色或纹理 拉伸后需要看起来自然
中心区域 纯色或均匀渐变（避免复杂花纹） 双向拉伸后花纹会变形
对称性 建议上下/左右对称设计 不对称时需要更精确的边距设置

2.2 边距标注

9宫格的核心参数是四个边距值（Border / Padding）：左（L）、右（R）、上（T）、下（B），单位为像素。

💡
标注方法：美术交付时需要在资源清单中标注每个9宫格资源的边距值。推荐格式：
NS_Btn_Blue.png → L:12 R:12 T:12 B:12
NS_Panel_Dark.png → L:20 R:20 T:24 B:16
⚠️ 外包常见错误
  • 不标注边距：只给切图不给数值，程序需反复试错
  • 边距包含了渐变区域：导致角落和边缘不干净
  • 图片太大：给了 512×128 的按钮底图做9宫格，浪费大量内存
  • 中心有花纹：复杂纹理在中心区域，拉伸后严重变形
  • 不对称但不标注：上边有装饰下边没有，但 T/B 给了相同值

3.1 UE 的 Margin 换算公式

📐
像素 → 归一化：
Left Margin = Left_Pixels / Image_Width
Right Margin = Right_Pixels / Image_Width
Top Margin = Top_Pixels / Image_Height
Bottom Margin = Bottom_Pixels / Image_Height

举例：图片 64×32，边距 12px → Left Margin = 12/64 = 0.1875，Top Margin = 12/32 = 0.375

🎨 四、颜色空间：sRGB vs Linear

4.1 为什么要关注颜色空间？

颜色空间决定了像素值如何映射到实际亮度。如果美术在 sRGB 空间下制作，但引擎按 Linear 空间读取，画面会变亮/变灰；反之会变暗/过饱和。

颜色空间特点适用贴图
sRGB 非线性 Gamma 编码（≈ γ=2.2），人眼感知均匀
Photoshop 默认工作空间		 所有颜色贴图：Base Color / Albedo / Diffuse / UI 切图 / 图标
Linear 线性空间，物理准确
数值与亮度成正比		 数据贴图：法线图 / 金属度 / 粗糙度 / Mask / 高度图
✅
核心原则：
• 包含颜色信息的贴图 → 标记为 sRGB
• 包含数据/数值的贴图 → 标记为 Linear
• UI 切图 99% 情况应标记为 sRGB

🔲 五、Alpha 通道注意事项

5.1 Premultiplied Alpha vs Straight Alpha

模式原理优势使用场景
Straight Alpha
（标准 Alpha）		 RGB 和 A 独立存储
混合时：final = src × α + dst × (1-α)		 PS 中查看 RGB 通道是原色 Photoshop 默认输出
大部分 UI 切图
Premultiplied Alpha
（预乘 Alpha）		 RGB 已经乘以 A
混合时：final = src + dst × (1-α)		 避免半透明边缘黑边
混合效率更高		 Unity Sprite 默认
UE Texture 可选

5.2 透明边缘黑边问题

⚠️
最常见的 Alpha 问题：图标/按钮边缘出现一圈黑边或暗色边缘。
原因：Photoshop 导出的 PNG 是 Straight Alpha，但引擎混合时假设 Premultiplied Alpha。
解决方案：
① 导出时在 PS 中选择 Premultiplied（需插件/脚本）
② 在引擎中设置 Alpha 类型为 Straight（Unity: Alpha Is Transparency ☑️）
③ 导出前对透明像素的 RGB 做出血填充（将边缘颜色扩展到透明区域）

5.3 各引擎 Alpha 处理

设置项 🔷 Unity 🟣 UE 🟢 Cocos
Alpha 模式 Texture → Alpha Is Transparency ☑️
自动处理边缘膨胀		 Texture → Compression Settings
选择 UserInterface2D (RGBA)		 默认 Straight Alpha
Premultiply Alpha 可在材质选项中开启
防黑边方案 勾选 Alpha Is Transparency
→ 自动膨胀 RGB 到透明区域		 Texture Editor → Padding: Dilate
→ 膨胀填充		 美术导出时手动出血
或用 TexturePacker 的 Premultiply 选项
⚠️ 外包常见 Alpha 错误
  • 白底导出：把 PSD 背景层也导出了，Alpha 全是 255
  • 透明像素 RGB 为黑色：导致边缘采样时出现黑边
  • Alpha 不干净：边缘有 0~10 的微弱 Alpha 值，肉眼不可见但引擎会渲染为半透明噪点
  • JPG 交付：JPG 不支持 Alpha 通道，透明信息全部丢失
  • 导出 PNG-8 而非 PNG-24/32：PNG-8 的 Alpha 只有开/关两种，无法半透明

🔄 六、Before / After 对比

❌ BEFORE — 无规范时
  • 每种按钮尺寸单独切一张大图，UI 包体 8MB+
  • 9宫格不标注边距，程序反复试错花费半天
  • Unity 项目用 UE 的归一化边距值，显示全错
  • 切图颜色在引擎中偏亮/偏暗，反复调色
  • 所有贴图都设 sRGB，Mask 通道插值失准
  • 图标边缘一圈黑边，不知道怎么解决
✅ AFTER — 执行规范后
  • 复用9宫格，同类按钮只需1张小图，包体降 80%
  • 交付清单含像素边距+UE归一化值，一次导入成功
  • 多引擎项目有各自的参数表，无需换算
  • 颜色贴图 sRGB、数据贴图 Linear，颜色精准
  • Mask 正确设为 Linear，换色效果准确
  • Alpha 出血处理，零黑边问题
