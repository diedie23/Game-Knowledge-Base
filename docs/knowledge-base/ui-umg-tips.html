🧠 一、Prompt 技巧总结 —— 5 大原则

使用 AI 工具生成 UMG Widget 时，Prompt 的质量直接决定输出质量。以下 5 条原则从实际项目经验中提炼，每条配有正反对比示例。

1

明确指定容器类型

▼

AI 默认倾向使用 Canvas Panel + 绝对坐标定位，这会导致适配灾难。务必在 Prompt 中明确要求使用哪种容器布局。

❌ 不推荐写法
请帮我做一个商城界面，顶部有标题栏，下面是商品列表。
问题：AI 会用 Canvas Panel 绝对定位所有元素，不同分辨率下布局崩溃。
✅ 推荐写法
用 Vertical Box 作为根容器：
- 第一个子元素：Horizontal Box 作为标题栏，高度固定 64px，子元素间距 16px，垂直居中对齐
- 第二个子元素：Scroll Box 包裹一个 Uniform Grid Panel，列数 3，作为商品网格列表，Fill 占满剩余空间
明确了容器嵌套关系、尺寸约束和对齐方式，AI 输出的蓝图可直接使用。
2

描述响应式行为而不是固定数值

▼

写死像素值是最常见的适配问题根源。用"填满""自适应""比例"等相对描述替代绝对数值。

❌ 不推荐写法
按钮宽度 200px，高度 50px，距离左边 100px，距离顶部 300px。
绝对坐标 + 固定尺寸 = 只能在一种分辨率下正确显示。
✅ 推荐写法
按钮宽度填满父容器（Fill），高度自适应内容（Auto），
内边距：上下 12px、左右 24px。
如果需要固定宽度，使用 Size Box 包裹并设置 Width Override = 200。
使用 Fill/Auto 等 UMG 原生布局概念，配合 Size Box 做必要约束。
3

用锚点描述位置关系

▼

对于需要绝对定位的元素（如 HUD 元素），用锚点 + 偏移量描述比用坐标更可靠。

❌ 不推荐写法
在屏幕右下角放一个小地图，位置 X=1720, Y=880。
1920x1080 以外的分辨率下小地图位置完全错误。
✅ 推荐写法
在 Canvas Panel 中放置小地图组件：
- 锚点（Anchor）设置为右下角（1.0, 1.0）
- 对齐（Alignment）设置为 (1.0, 1.0)
- 偏移量（Position）：X = -20, Y = -20
- 使用 Size Box 固定尺寸 240×240
锚点定位在任何分辨率下都保持正确的相对位置。
4

指定适配目标分辨率

▼

不同项目的目标分辨率差异很大。明确告知 AI 设计基准和需要适配的范围。

❌ 不推荐写法
做一个手机游戏的主界面。
AI 不知道你的基准分辨率，可能按 PC 1920x1080 设计。
✅ 推荐写法
设计基准分辨率为 1920×1080（DPI Scale = 1.0）。
需要适配的目标设备：
- PC：1920×1080, 2560×1440
- 手机竖屏：750×1334（iPhone 8）, 1170×2532（iPhone 14）
- 平板：2048×1536（iPad）

Canvas Scaler 策略使用 DPI Scale Rule = Shortest Side。
UI 元素最小可触区域 ≥ 44×44 逻辑像素。
给出完整的适配上下文，AI 能输出考虑多分辨率的布局方案。
5

说明交互状态

▼

按钮等交互元素如果不指明状态，AI 只会生成默认外观。明确列出所有需要的交互状态和视觉反馈。

❌ 不推荐写法
在底部放一个"开始游戏"按钮。
只会得到一个静态按钮，无交互反馈。
✅ 推荐写法
"开始游戏"按钮需要 4 个状态：
- Normal：背景色 #4A90D9，文字白色 16px Bold
- Hovered：背景色变亮 #5AA0E9，增加 2px 外发光
- Pressed：背景色变暗 #3A80C9，按钮整体缩放 0.95
- Disabled：背景色 #555555，文字色 #888888，不响应点击

状态切换使用 UButton 的 Style 属性配置。
需要在 Hovered 时播放一个音效提示（仅预留接口）。
完整的状态描述让 AI 输出可直接用于生产的交互配置。

📦 二、常见 UMG 组件使用场景速查表

在写 Prompt 时，准确使用 UMG 组件名称能大幅提升 AI 输出质量。以下是常用组件的适用场景速查。

🔲 布局容器类

组件适用场景Prompt 中的描述方式⚠ 注意事项
Canvas Panel HUD 覆盖层、自由定位元素 "在 Canvas Panel 中放置，锚点设置为…" 尽量只用于最外层或 HUD，内部布局避免使用
Vertical Box 纵向排列（列表、表单、侧栏） "用 Vertical Box 纵向排列，间距 8px" 子元素用 Slot Padding 控制间距
Horizontal Box 横向排列（工具栏、Tab 栏、按钮组） "用 Horizontal Box 横向排列，子元素垂直居中" 注意子元素的 Fill / Auto 设置
Overlay 层叠（背景图 + 前景内容） "用 Overlay 层叠：底层背景图 + 上层内容" 子元素按添加顺序层叠，后加的在上方
Size Box 强制固定/限制子元素尺寸 "用 Size Box 包裹，Width Override = 300" 常用于限制头像、图标等的尺寸
Scale Box 整体等比缩放 "用 Scale Box 包裹，Stretch = Scale To Fit" 适合需要等比缩放的独立 UI 组件
Scroll Box 可滚动列表、长页面 "用 Scroll Box 包裹列表，纵向滚动" 子元素高度必须大于 Scroll Box 才会出现滚动
Uniform Grid Panel 等宽等高网格（背包、商城） "用 Uniform Grid Panel，每行 4 列" 子元素大小由 Cell 统一控制
Wrap Box 自动换行（标签云、技能列表） "用 Wrap Box 自动换行排列标签" 子元素宽度不一致时自动排列
Widget Switcher Tab 页切换、多状态面板 "用 Widget Switcher 切换页面内容" 一次只显示一个子 Widget

🔍 三、生成后验收要点

AI 生成的 UMG 代码/蓝图常见以下问题，验收时需重点检查：

⚠️ 适配问题（最高频）

  • 是否使用了绝对坐标定位（Canvas Panel 内部嵌套布局）
  • 是否有硬编码像素值（Width/Height 写死而非 Fill/Auto）
  • 锚点是否设置正确（不应全部在左上角 0,0）
  • 在非基准分辨率下是否布局正常（切换 16:9 / 18:9 / 4:3 测试）
  • DPI Scale Rule 是否配置（特别是移动端项目）

⚠️ 层级与嵌套问题

  • 容器嵌套是否过深（超过 5 层需优化）
  • 是否存在不必要的 Canvas Panel 嵌套
  • Overlay 内的子元素 Z-Order 是否正确
  • Scroll Box 内是否嵌套了另一个 Scroll Box（冲突）
  • Widget Switcher 的子页面是否正确索引

⚠️ 性能问题

  • 是否有大量 Tick 更新（应使用事件驱动替代）
  • 列表是否使用虚拟化（超过 20 项应用 List View）
  • 材质/特效是否过重（半透明叠加层数）
  • 是否有未释放的 Widget 引用（内存泄漏）
  • Invalidation Box 是否用于静态区域

✅ 合格标准

  • 在 3 种以上分辨率下布局无错位
  • 所有交互元素有 4 种状态（Normal/Hover/Press/Disable）
  • 文本支持本地化（无硬编码中文，使用 Text Table）
  • 容器嵌套层级 ≤ 5 层
  • 列表 > 20 项使用 List View 虚拟化
⚠️ 外包常见返工原因 TOP 3：① 只在 1920×1080 下正确，切换分辨率全乱 ② 按钮没有 Hover/Pressed 状态，交互体验缺失 ③ Canvas Panel 滥用导致无法自适应

📝 四、外包沟通 UMG 需求模板

向外包 CP 描述 UMG 需求时，使用标准化模板可以大幅减少沟通歧义和返工。以下是实战验证过的模板格式。

📋 模板 A：通用 UI 界面需求 日常需求

【需求名称】商城主界面
【目标引擎】UE 5.3
【基准分辨率】1920×1080
【适配要求】16:9 和 18:9 竖屏（手机），最小安全区 1080×1920

【界面结构描述】
根容器：Vertical Box
├─ 顶部栏（Horizontal Box，高度 64px Fixed）
│  ├─ 返回按钮（左侧，Size Box 48×48）
│  ├─ 标题文本（Fill，居中）
│  └─ 货币显示（右侧，Auto）
├─ Tab 栏（Horizontal Box，高度 48px Fixed）
│  └─ 4 个 Tab 按钮（均分 Fill）
└─ 内容区（Scroll Box，Fill）
   └─ Uniform Grid Panel（3 列，间距 12px）
      └─ 商品卡片 × N

【交互要求】
- Tab 按钮：选中态底部 2px 高亮线 + 文字变色
- 商品卡片：Hover 放大 1.02 + 边框高亮
- 购买按钮：4 态（Normal/Hover/Press/已购买灰色）

【交付物】

□ 分辨率适配截图（1080×1920 / 1170×2532 / 2048×1536）

📋 模板 B：HUD 元素需求 实时显示

【需求名称】战斗 HUD
【目标引擎】UE 5.3
【Canvas Scaler】DPI Scale Rule = Shortest Side

【元素列表】
1. 血条（左上角）
   - 锚点：(0, 0)，偏移 (20, 20)
   - Progress Bar + 数值文本叠加
   - 低血量（<30%）时红色闪烁动画

2. 小地图（右上角）
   - 锚点：(1, 0)，对齐 (1, 0)，偏移 (-20, 20)
   - Size Box 200×200，圆形遮罩

   - 锚点：(0.5, 1)，对齐 (0.5, 1)，偏移 (0, -20)

【性能要求】
- 血条数值变化用事件驱动，禁止 Tick
- 小地图刷新率 ≤ 10 FPS
- 使用 Invalidation Box 包裹静态装饰

🎯 完整示例：背包界面

请用 UE5 蓝图为我创建一个背包界面的 UMG Widget Blueprint：

## 基础信息
- 引擎：UE 5.3
- 基准分辨率：1920×1080
- 需适配：2560×1440 / 1080×1920（手机竖屏）
- DPI Scale Rule：Shortest Side

## 界面结构
根容器：Overlay
├─ 层 0：全屏半透明黑色背景（Image，Tint = 0,0,0,0.6）
└─ 层 1：居中面板（Size Box 包裹，Width Override = 800, Height Override = 600）
   └─ Border（背景色 #1a1d2e，圆角 12px，内边距 24px）
      └─ Vertical Box
         ├─ 标题栏（Horizontal Box，高度 Auto）
         │  ├─ "背包" Text Block（字号 22，Bold，白色，Fill）
         │  └─ 关闭按钮 ×（Size Box 36×36，右对齐）
         │     - 4 态：Normal 灰色 / Hover 白色 / Press 红色 / Disable 隐藏
         ├─ 分隔线（Border，高度 1px，背景色 #333）
         ├─ 筛选栏（Horizontal Box，高度 Auto，间距 8px，Padding Top 12）
         │  └─ 5 个筛选按钮："全部" "武器" "防具" "消耗品" "材料"
         │     - 选中态：底色 #6c8cff，文字白色
         │     - 未选态：底色透明，文字 #888
         └─ 物品网格（Scroll Box，Fill，Padding Top 12）
            └─ Uniform Grid Panel（每行 5 列，Slot Padding 6px）
               └─ 物品格子 Widget（Overlay）
                  ├─ 底图 Image（64×64 边框图）
                  ├─ 物品图标 Image（Fill）
                  ├─ 数量 Text Block（右下角，字号 11，描边）
                  └─ 品质边框 Image（颜色按品质：白/绿/蓝/紫/橙）

## 交互
- 物品格子 Hover：放大 1.05 + 边框高亮
- 物品格子 Click：弹出详情 Tooltip
- 关闭按钮 Click：播放淡出动画后 Remove From Parent

## 约束
- 禁止内部使用 Canvas Panel
- 嵌套 ≤ 5 层
- 所有 Text Block 使用 FText，预留本地化
- 关键节点添加中文注释
💡 提示：使用此模板时，可以根据项目实际情况调整分辨率、容器类型和组件配置。关键是保持结构化描述（从外到内）+明确的约束条件（禁止/必须），让 AI 减少猜测。

📋 六、APM 管理建议

🔑 核心原则：AI 是加速器，不是替代者。APM 需要建立"AI 生成 → 人工审核 → 规范沉淀"的闭环流程，把 AI 输出转化为团队可复用的资产。

6.1 AI 辅助 UMG 开发的管理流程

阶段负责人动作产出
需求描述APM / 策划使用模板 A/B 撰写结构化需求需求文档
AI 生成TA / 美术使用模板 C 输入 AI 工具初版 Widget
自审TA / 美术按验收要点 Checklist 自查自查报告
评审APM + 主美多分辨率测试 + 交互走查修改意见
沉淀APM合格 Widget 归档 + Prompt 模板更新组件库/模板库

6.2 外包场景注意事项

⚠️ 外包常见风险

  • 外包可能直接用 AI 生成不做任何适配测试就交付
  • 不同 CP 使用不同 AI 工具，输出风格和质量差异大
  • Prompt 模板不统一导致每次需求沟通成本高
  • AI 生成的代码缺少注释和文档，后续维护困难

✅ APM 对策

  • 在合同/需求文档中附加本页的验收 Checklist
  • 统一发放 Prompt 模板，要求 CP 按模板描述需求
  • 要求 CP 提交时附带 3 种分辨率截图
  • 建立内部 UMG 组件库，优先复用而非每次重新生成
💡 进阶建议：将高频使用的 UI 组件（按钮、弹窗、列表）封装为标准 Widget 模板，让 AI 在此基础上组合拼装，而非每次从零生成。这样既提升效率，又保证视觉一致性。
