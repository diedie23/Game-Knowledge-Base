📋 1. 概述与适用范围

本规范定义了游戏项目中所有 音频资产（Audio Assets） 的分类标准、制作规格、集成流程、版本管理和质量门禁。覆盖 BGM、SFX、环境音、语音（VO）、UI 音效五大类别。

适用项目类型

项目类型音频中间件引擎典型规模
手游（二次元 / MMO）Wwise / CRI ADX2Unity / UE55000~20000 音频事件
端游 / 主机Wwise / FMOD StudioUE5 / 自研引擎20000~80000 音频事件
小型/独立游戏FMOD / 引擎原生Unity / Godot500~3000 音频事件
💡 核心原则：声音是游戏世界的"第二画面"——优秀的音频设计让玩家"闭眼也能感知游戏状态"。所有音频规范围绕沉浸感、信息传达和性能可控三大目标展开。

5.3 关键技术指标

指标标准说明
底噪≤ -60 dBFS静默段测量，VO 要求更严格 ≤ -65 dBFS
Peak（峰值）≤ -1 dBTPTrue Peak 不可超过 -1 dB
DC Offset≤ 0.01交付前必须消除 DC 偏移
循环点零交叉点 ± 1 sampleLoop 素材必须在零交叉点剪辑
头尾静默头: 0~5ms / 尾: ≤ 50msSFX 不含多余静默（影响触发响应）
采样率转换禁止上采样44.1k → 48k 必须使用 SRC 高质量插值
🚫 禁止行为：① 提交 MP3 作为源文件（有损不可逆）；② 源文件采样率低于 44.1kHz；③ VO 文件包含环境噪声/呼吸声未处理；④ 循环素材首尾有爆音 Click。

8.4 VO QA 检查清单

  • 文件名与台本 VO_ID 完全一致
  • 采样率 48kHz / 24bit / Mono
  • 底噪 ≤ -65 dBFS（VO 专用标准）
  • 无爆音 / 齿音过重 / 口水音
  • 情绪与场景描述匹配
  • 时长在参考时长 ±30% 以内
  • 头部无多余静默（≤ 5ms）
  • 尾部自然衰减，无截断
  • 同角色不同台词音量一致性（±2 LUFS）
  • 变体间差异明显（避免重复感）

💣 13. 常见问题与踩坑记录

Case 1：BGM 切换时断裂

现象：从探索场景切换到战斗时 BGM 直接截断，出现不和谐的"硬切"。
根因：Event 使用了 Stop + Play 而非 Transition Segment，未等待当前小节结束。
解法：① BGM 使用 Interactive Music 系统，配置 Transition Rules（Exit/Entry Cue）；② 设置最小 1 Bar 的 Cross-Fade；③ Stop Event 使用 Fade-Out Time ≥ 500ms。

Case 2：安卓低端机音频卡顿

现象：战斗中同时释放多个技能时音效出现延迟/丢失。
根因：同时发声数超过 32 上限 + Stream 音频 IO 争用导致缓冲区饥饿。
解法：① 降低 Max Voice 至 24（低端机配置）；② 战斗短 SFX 全部改为 In-Memory；③ 设置 Voice Priority 确保玩家操作 SFX 优先级最高。

Case 3：VO 多语种时长不一致

现象：日语配音 1.5s，英语配音 3.2s，导致对话界面等待时间差异过大。
根因：英语翻译过长，录制时未做时长控制。
解法：① 台本阶段设置各语种参考时长上限（容差 ±30%）；② 录制时给声优看倒计时；③ 引擎对话系统增加"最小等待 / 最大等待"参数。

Case 4：iOS 后台 BGM 中断

现象：游戏切后台再返回时 BGM 消失，需重启才恢复。
根因：iOS Audio Session 被系统回收后未正确重新激活。
解法：① 配置 AVAudioSession Category 为 AVAudioSessionCategoryPlayback + MixWithOthers；② 监听 AudioSessionInterruption 通知，中断结束后重新 Post BGM Event。

Case 5：Wwise Bank 版本冲突

现象：程序更新引擎版本后，音频 Bank 加载报错 "Invalid Bank Version"。
根因：Wwise SDK 版本与 Bank 生成版本不一致。
解法：① 锁定 Wwise SDK 版本号到项目全局配置；② Bank 重新 Generate 前必须检查 SDK 版本；③ CI/CD 流程增加 Wwise 版本校验步骤。
