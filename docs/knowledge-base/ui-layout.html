📱 1. 多分辨率适配原则

1.1 设计基准分辨率
项目统一基准：以 1920×1080（16:9）为设计稿基准分辨率，所有标注值基于此尺寸。
目标设备分辨率范围宽高比适配策略
iPhone SE / 小屏手机1334×75016:9等比缩放，无裁切
iPhone 15 Pro / 主流手机2556×117919.5:9两侧留黑或扩展背景
iPad / 平板2048×15364:3上下留黑或扩展背景
1080P PC1920×108016:9基准，1:1 显示
2K / 4K PC2560×1440 / 3840×216016:9DPI 缩放
1.2 缩放策略

🔴 铁律：禁止使用固定像素坐标定位 UI 元素。所有 UI 元素必须通过锚点（Anchor）+ 偏移量（Offset）定位。固定坐标在非基准分辨率下会导致 UI 偏移出屏幕。

策略适用场景引擎实现
等比缩放（Scale with Width）横屏游戏主流方案UE5: DPI Scaling Rule = Shortest Side
Unity: Canvas Scaler → Scale With Screen Size
安全区裁切（Safe Zone）异形屏（刘海/挖孔）UE5: SafeZone 组件
Unity: Screen.safeArea
固定 DPI 缩放PC 端 2K/4KUE5: DPI Scale 1.5/2.0

⚓ 2. 锚点 Anchor 打法铁律

⛔ 核心原则：UI 元素的 Anchor 决定了它相对于父容器哪个点定位。搞错 Anchor = 换分辨率就炸。
2.1 三大 Anchor 模式
模式Anchor 设置行为适用元素
全屏拉伸
(Stretch)		Min(0,0) Max(1,1)四边始终贴合父容器边缘，大小随容器变化全屏背景、遮罩层、根 Canvas
居中固定
(Center)		Min(0.5,0.5) Max(0.5,0.5)始终在父容器正中心，大小固定不变弹窗面板、Logo、主菜单
贴边吸附
(Edge Pin)		锚定到对应边与指定边缘保持固定偏移距离返回按钮（左上）、设置按钮（右上）、底部导航栏
2.2 贴边吸附详表
UI 元素吸附位置Anchor MinAnchor Max偏移方向
返回按钮左上角(0, 1)(0, 1)Right + Down
设置/菜单按钮右上角(1, 1)(1, 1)Left + Down
聊天输入框左下角(0, 0)(0, 0)Right + Up
小地图右下角(1, 0)(1, 0)Left + Up
血条/战斗 HUD顶部居中(0.5, 1)(0.5, 1)Down
底部导航栏底部拉伸(0, 0)(1, 0)Up，宽度随屏幕
侧边栏左侧拉伸(0, 0)(0, 1)Right，高度随屏幕

🔴 铁律 #1：角落元素（返回键/设置键）必须锚定到对应角，禁止锚定到中心再偏移。否则宽屏设备上按钮会"飘"到屏幕中间区域。

🔴 铁律 #2：底部导航栏必须使用水平拉伸锚点 Anchor(0,0)→(1,0)，禁止固定宽度。否则窄屏设备上导航栏不够宽，宽屏设备上两侧空白。

🔴 铁律 #3：弹窗/对话框必须居中锚定 Anchor(0.5,0.5)，禁止贴边。弹窗在不同分辨率上应始终在视觉中心。

✅❌ 3. 图文红黑榜

3.1 锚点选错 → 异形屏 UI 跑偏

❌ Don't — 返回按钮锚定到中心

[此处插入图片：16:9 下返回按钮在左上角位置正确；切到 21:9 超宽屏后，返回按钮飘到了屏幕偏中间的位置，远离左上角]

后果：Anchor 设为 Center(0.5,0.5)，偏移量 (-800,-450) 计算到左上角。在 21:9 屏幕上，中心点向右偏移，按钮也跟着偏移，不在左上角了。

✅ Do — 返回按钮锚定到左上角

[此处插入图片：16:9 和 21:9 两种分辨率，返回按钮始终固定在左上角相同距离处]

方法：Anchor = (0, 1)，Offset = (20, -20)。无论屏幕多宽多窄，按钮始终距左边 20px、距顶部 20px。

3.2 导航栏固定宽度 → 窄屏溢出

❌ Don't — 底部导航栏固定 1920px 宽度

[此处插入图片：720p 设备上导航栏超出屏幕右侧，右侧按钮不可见；4:3 iPad 上导航栏左右两侧有大块空白]

后果：固定宽度在低分辨率上超出屏幕被裁切，在宽屏上两侧大量空白。

✅ Do — 导航栏水平拉伸 + 内容等分

[此处插入图片：导航栏 Anchor(0,0)→(1,0) 水平拉伸，内部按钮均匀分布，各分辨率表现一致]

方法：Anchor Min(0,0) Max(1,0)，固定高度，宽度自适应。内部按钮使用 HorizontalBox + 等分权重。

❓ 5. 常见问题排查 FAQ

💬 Q1：iPad（4:3）上 UI 上下被裁切？

原因：缩放策略设为 "Scale with Width"，4:3 竖向比例更大，高度不够导致上下裁切。

修复：改用 Shortest Side 或 Expand 策略。关键信息元素不要放在顶部/底部 10% 区域内。

💬 Q2：刘海屏上状态栏按钮被遮挡？

原因：未考虑 Safe Zone Insets，UI 元素侵入了刘海/药丸区域。

修复：UE5 使用 SafeZone 组件包裹顶部 UI；Unity 使用 Screen.safeArea 计算 padding。所有顶部按钮偏移量加上 Safe Zone Inset。

💬 Q3：HorizontalBox 内按钮在宽屏上间距过大？

原因：按钮设为等分权重（Fill），宽屏上每个按钮被拉宽，间距视觉上太大。

修复：内容区域设最大宽度限制（Max Width = 1200px），超出部分用居中 + 两侧留白。或改用 "Auto Size" + 固定间距。
