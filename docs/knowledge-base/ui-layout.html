📐 Layout 界面拼接规范 UI 规范

涵盖锚点系统、多分辨率适配策略、三大引擎 Layout 差异对比，以及小游戏场景下的安全区处理。帮助美术和程序在界面拼接环节减少沟通成本与返工。

📌 一、基础概念

1.1 锚点（Anchor）

锚点定义了 UI 元素相对于父容器的参考位置。当屏幕尺寸变化时，元素会以锚点为基准进行位置计算。

UI 元素推荐锚点原因
顶部状态栏（血条/金币）上中 / 上方拉伸始终贴顶，横向适配
底部操作按钮下中 / 下方拉伸始终贴底，手指操作区
返回/关闭按钮右上角固定位置，不随内容变化
全屏弹窗/面板居中水平垂直居中，与分辨率无关
聊天/信息列表上下拉伸高度自适应，内容可滚动
小地图右上角固定在角落，不干扰主要操作

1.2 对齐方式

对齐方式说明典型场景
绝对定位相对锚点的固定偏移量（px）固定位置的图标、按钮
比例定位相对父容器的百分比需要随屏幕等比移动的元素
自动布局Horizontal/Vertical Layout 自动排列列表项、网格背包、底部Tab栏
拉伸填充锚点设为两端，元素随容器拉伸全宽进度条、分割线、背景

1.3 间距系统

📏
推荐使用 8px 基准间距系统：
• 基础间距：8px（元素内部间距）
• 标准间距：16px（元素之间）
• 大间距：24px / 32px（区块之间）
• 所有间距值应为 4 的倍数，避免出现 5px / 7px / 13px 等奇数值
• 统一间距系统有助于视觉节奏一致，也方便程序批量调整

📱 二、多分辨率适配策略

📌 固定尺寸

UI 元素使用固定像素值，不随屏幕缩放。适合需要像素精准的场景。
✅ 适用场景：
  • 像素风格游戏
  • 固定分辨率的街机/主机游戏
  • 需要锐利边缘的 HUD 元素
❌ 不适用：
  • 需要跨多种分辨率的手游
  • 小游戏（设备差异极大）

📐 弹性布局（推荐）

使用锚点+约束实现自适应。元素位置相对父容器确定，尺寸可拉伸或保持。
✅ 适用场景：
  • 大部分手游 UI（推荐默认方案）
  • 微信小游戏 / H5 游戏
  • 需要适配横竖屏的项目
❌ 不适用：
  • 对布局精度要求极高的场景

🔍 等比缩放

以设计稿分辨率为基准，整体等比缩放适配目标屏幕。简单但可能浪费屏幕空间。
✅ 适用场景：
  • 休闲小游戏（快速开发优先）
  • 竖屏固定比例的游戏
  • 原型/Demo 阶段
❌ 不适用：
  • 横屏全屏游戏（16:9 vs 18:9 差异大）
  • 需要充分利用屏幕空间的项目

⚠️ 三、常见拼接错误

错误类型描述后果正确做法
硬编码位置 所有元素都用绝对坐标定位 换分辨率后元素跑出屏幕或重叠 使用锚点+相对偏移
未考虑分辨率适配 只在 1920×1080 上验证 手机端元素太小看不清或溢出 至少在 3 个分辨率下测试
层级混乱 UI 元素 Z-Order 随意设定 弹窗被背景遮挡、按钮无法点击 制定层级规范（Background → HUD → Popup → Toast）
忽略安全区 刘海屏/挖孔屏下内容被遮挡 按钮/文字被手机刘海挡住 获取系统安全区 Insets，留出安全边距
文本溢出 固定宽度文本框但内容超长 多语言/长文本被截断或溢出 设置最大宽度 + 省略号 + 自适应字号
Scroll 区域设置错误 滚动列表没有 Mask/ClipRect 列表项溢出到其他区域 添加 Mask 组件或 Clip Children
⚠️ 外包交付常见 Layout 问题
  • PSD 标注不含锚点信息：只标注了像素位置，没有说明哪些元素需要居中/贴边/拉伸
  • 设计稿只有一个分辨率：没有提供竖屏适配或宽屏适配的方案变体
  • 元素间距不统一：有的间距 10px 有的 13px 有的 17px，没有节奏
  • 层级未标注：多个面板的前后关系不清楚，交付后需要反复调整
  • Safe Area 未留白：重要交互元素放在刘海区域，用户无法操作

⚠️
小游戏场景特别注意：
• 微信小游戏通过 wx.getSystemInfoSync() 获取 safeArea 字段
• 横屏小游戏时刘海在左/右侧，需要左右留白而非上下
• 底部虚拟键区域（如 iPhone 的 Home 指示条）不仅视觉遮挡，还会拦截触摸事件！
• 不同 Android 厂商的安全区数据可能不准确，建议额外留 4~8px 容错
💼
APM 外包管理要点：
• 在 UI 设计稿中必须标注安全区边界（红线/虚线框）
• 外包交付的设计稿必须附带至少 2 个分辨率的适配方案（标准 + 刘海屏）
• 验收时必须在真机上测试，模拟器的安全区数据可能不准
• Layout 相关返工是外包中最常见的返工原因之一（约占 UI 返工的 40%），应前置沟通

📑 六、UI 层级（Z-Order）规范

层级Z-Order 范围包含内容说明
背景层0 ~ 99全屏背景、场景 UI最底层
主界面层100 ~ 199主 HUD、Tab 栏、状态栏常驻显示
弹窗层200 ~ 299弹出面板、对话框需配半透明遮罩
浮动层300 ~ 399Tooltip、下拉菜单跟随鼠标/手指
Toast/提示层400 ~ 499消息提示、Loading最高优先级显示
系统遮罩层500+全屏 Loading、系统弹窗阻断所有交互

🔄 七、Before / After 对比

❌ BEFORE — 无规范时
  • 元素全部用绝对坐标，换手机就乱
  • 设计稿只有 1920×1080 一种
  • 锚点全在左上角，窄屏看不到右侧元素
  • 刘海屏按钮被遮挡，用户投诉
  • 弹窗被背景遮挡，找不到层级问题
  • 外包交付后每次适配改 2~3 天
  • 元素间距随意，界面看着"不舒服"
✅ AFTER — 执行规范后
  • 锚点+弹性布局，自动适配多种分辨率
  • 设计稿附带标准/刘海屏两套方案
  • 锚点根据功能合理分配（贴边/居中/拉伸）
  • 安全区适配，所有设备操作正常
  • 层级按规范分区，弹窗/Toast 不会被遮挡
  • 一次交付基本通过，返工减少 60%
  • 8px 间距系统，视觉节奏统一专业
