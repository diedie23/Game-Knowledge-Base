3️⃣ 风险系数加成

🚨 [P0] 风险系数可叠加，但有上限 min(连乘, 2.0)。
风险因素加成系数说明
🆕 首次合作 CP×1.20磨合成本，预期返工率高
🌏 跨时区沟通×1.10沟通延迟，反馈周期拉长
🔄 风格未锁定×1.30需求可能反复，预留探索空间
⏰ 紧急赶工×1.25加班费 + 质量风险
🆕 新品类/新技术×1.15学习曲线成本
📋 修改轮次多 (>3轮)×1.10/轮每多一轮加 10%
核心红线：风险系数可叠加，但有上限：最终风险系数 = min(各风险系数连乘, 2.0)
示范：首次合作 (1.2) × 风格未锁定 (1.3) = 1.56

🚨 典型问题

🔴 案例：评估偏差导致外包预算严重超支 高频

🎬 典型场景还原

APM："这批 5 个角色建模，我评的 B 级复杂度，预估 60 人天。"

主美（收到成品后）："这些角色都有可拆卸武器和复杂配饰，明显是 A 级啊。"

财务："实际耗时 92 天，偏差率 53%，预算超支 ¥48,000。"

APM："当时凭感觉定的级……没用打分表。"

🔍 问题根因拆解

• APM 按 B 级复杂度评估，但实际需求为 A 级
• 未使用五维度打分表量化评估，凭"感觉"定级
• 风险系数未叠加"首次合作 CP"的 ×1.2 加成

💡 APM 破局思路

1. 回溯使用五维度打分表重新评估
2. 与 CP 协商追加预算，按实际 A 级结算
3. 将偏差数据录入历史校准库
4. 所有评估必须使用五维度打分表，禁止"拍脑袋"
5. 每季度进行评估准确率复盘，偏差 > 20% 的工种重新校准
6. 首次合作 CP 强制叠加 ×1.2 风险系数

📌 Do / Don't 示例

场景说明：外包角色建模报价评估 — APM 收到策划提交的 3 个新英雄建模需求，需要评估人天并生成报价单给 CP。

✅ Do（正确示范）

• 使用五维度打分表逐个评估每个角色复杂度

• 综合打分后确定等级：英雄 A 为 S 级（3.7 分），英雄 B/C 为 A 级（3.0 分）

• 查表填入基准人天：S 级 = 17d，A 级 = 12d × 2 = 24d

• 叠加风险系数：首次合作 ×1.2，风格未锁定 ×1.3，合计 ×1.56

• 生成报价单并提交三级评审

❌ Don't（错误示范）

• "这 3 个角色差不多都是 A 级吧" → 未量化评估，一刀切定级

• 未考虑风险系数 → 首次合作 CP 的磨合成本被忽略

• 报价只跟 1 家 CP → 没有竞价基线

• 评估完直接执行，未走三级评审 → 缺乏校验环节
