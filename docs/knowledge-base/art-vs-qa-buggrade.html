🚦 1. 严重度等级定义

💡 核心原则：Bug 严重度由 视觉缺陷对游戏体验的影响程度 决定，与修复难度无关。
🏷️ 1.1 四级严重度
🏷️ 等级🔖 代号📝 定义💥 典型案例
🔴 S 级 (致命)Blocker导致游戏无法正常进行的美术问题角色全身黑/纯白；主界面无法显示；闪退
🟠 A 级 (严重)Critical明显影响游戏体验的视觉问题角色穿模；贴图错乱；动画卡死；特效遮挡操作
🟡 B 级 (一般)Major有瑕疵但不影响功能的视觉问题接缝可见；贴图精度不足；颜色偏差；动画不自然
🟢 C 级 (轻微)Minor轻微瑕疵，需仔细观察才能发现远景接缝；微小穿模；轻微锯齿；图标对齐偏差
🔍 1.2 详细判定标准

🔴 S 级 — 致命 Blocker 出现即阻断游戏正常流程，必须立即响应。

💥 表现📌 判定依据
角色/场景渲染异常全黑/全白/全透明/闪烁
必经路径无法通过碰撞体缺失导致卡死
UI 功能缺失按钮不可见/不可点击
闪退 / Crash加载特定美术资产时崩溃
严重性能问题进入特定场景帧率 < 10 FPS

🟠 A 级 — 严重 Critical 明显可感知的视觉问题，影响游戏体验。

💥 表现📌 判定依据
角色穿模身体部件相互穿插
贴图错乱UV 错位/贴图加载异常
动画卡死停留在某个动画状态不切换
特效异常特效未播放/不消失/遮挡UI
适配问题特定机型 UI 严重变形/遮挡
资产错用A 角色显示了 B 角色的模型

🟡 B 级 — 一般 Major 有瑕疵但不影响核心功能，可在当前 Sprint 内修复。

💥 表现📌 判定依据
接缝可见角色/场景模型拼接处有缝隙
贴图精度近景查看明显模糊/像素化
颜色偏差与原画/设计稿色差明显
动画不自然过渡生硬/速度异常
特效位置特效偏移/不跟随角色
阴影异常阴影形状异常/缺失

🟢 C 级 — 轻微 Minor 轻微瑕疵，需仔细观察才能发现，优先级最低。

💥 表现📌 判定依据
远景瑕疵需拉近镜头才能发现
微小穿模特定动作下极轻微穿模
对齐偏差UI 元素 1~2px 偏差
锯齿/走样轻微边缘锯齿
命名不规范不影响功能但不符合规范

📊 2. 优先级矩阵

🧮 2.1 优先级 = 严重度 × 影响范围
🌐 全局🛤️ 常见路径🎲 偶发/边缘
🔴 S 级⚡ P0 立即修⚡ P0 立即修🔴 P1 当天修
🟠 A 级🔴 P1 当天修🟡 P2 本Sprint修🟡 P2 本Sprint修
🟡 B 级🟡 P2 本Sprint修🟢 P3 下Sprint修🟢 P3 下Sprint修
🟢 C 级🟢 P3 下Sprint修⚪ P4 排期候选⚪ P4 不修/积压
⏱️ 2.2 修复时效要求
🚨 优先级⏰ 响应时间🔧 修复时限📢 升级节点
⚡ P030 分钟4 小时超时 → 主美/制作人
🔴 P12 小时当天超时 → APM
🟡 P2当天本 Sprint超时 → 组长
🟢 P33 天内下 Sprint—
⚪ P4不限有空时处理—
⚡ APM 金句："P0 不过夜，P1 不隔天——这是美术品质的底线。"

🔄 4. 美术修复工作流

📐 4.1 标准修复流程
🐛 QA 提 Bug
→
📋 APM 分配
→
🎨 美术确认
→
🔧 修复中
→
📦 提交验证
→
🔍 QA 验证
→
🏁 关闭
⚠️ P0/P1 级 Bug 修复需在修复完成后 立即通知 QA 验证，不等集中验证。验证未通过则 🔄 重开回到美术确认。
🏷️ 4.2 状态定义
🔖 状态📝 说明
NewQA 新提交
AssignedAPM 已分配给具体美术
Confirmed美术确认是 Bug
In Progress修复中
Resolved美术修复完毕，提交验证
VerifiedQA 验证通过
Closed关闭
ReopenedQA 验证未通过，重新打开
Won't Fix设计如此 / 性价比过低不修
Duplicate重复 Bug
🚫 4.3 拒绝 Bug 的合理理由
📌 理由📝 说明👤 需要谁确认
设计如此 (By Design)表现符合设计意图策划 + 主美 确认
性价比过低 (Won't Fix)修复成本远大于收益APM + 主美 确认
重复 (Duplicate)与已有 Bug 重复提供原 Bug ID
无法复现 (Cannot Reproduce)按步骤无法复现与 QA 确认环境
