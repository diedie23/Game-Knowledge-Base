🏗️ 1. 材质体系架构总则

"材质体系是美术表现力和运行性能之间的'翻译官'——好的 Master Material 架构能让美术自由表达、让 TA 灵活扩展、让程序安心优化。三方共赢的前提是'规矩在前面'。"

— 资深制作人经验

1.1 三层架构模型

层级名称职责维护者修改权限
L0Master Material定义光照模型、功能开关、参数架构TA Lead仅 TA Lead + 制作人审批
L1Material Function / Layer可复用计算模块（噪声、溶解、流光等）TATA 内部 Code Review
L2Material Instance具体资产的参数配置（颜色、贴图、数值）美术美术自由调节（参数范围内）
🚨 红线规则：美术不允许创建新的 Material（L0），只能创建 Material Instance（L2）。新增材质类型需求走 TA 需求单 → TA 评估 → 制作人审批 流程。

1.2 材质类型分类体系

⚙️ 2. Master Material 设计规范

2.1 功能开关（Static Switch）设计原则

核心思路：一个 Master Material 通过 Static Switch 控制功能模块的编译开关，而非为每个效果创建独立 Material。这样可以最大化实例化复用率，同时通过关闭未使用功能减少 Shader 指令数。
开关名称功能默认值开启代价
bUseNormalMap法线贴图ON+2 Texture Sample
bUseEmissive自发光OFF+1 Tex + 2 指令
bUseDetailNormal细节法线叠加OFF+2 Tex + 4 指令
bUseRimLight边缘光/菲涅尔OFF+8 指令
bUseDissolve溶解效果OFF+1 Tex + 12 指令
bUseVertexAnim顶点动画（植被摆动）OFF+6 指令
bUseMaskTintMask 区域换色OFF+1 Tex + 8 指令
bUseSSS次表面散射OFF光照模型切换

📌 红线 #1：单个 Master Material 的 Static Switch 数量 ≤ 12 个。超过 12 个开关意味着功能范围过大，应拆分为两个 Master Material。

📌 红线 #2：Static Switch 的排列组合不得产生 超过 64 个有效变体。需在设计时明确哪些组合是合法的（用 Variant Map 管理）。

2.2 参数分组与暴露策略

分组参数示例暴露给美术参数范围
Base ColorBaseColor Texture / Tint Color / Brightness✅Color: 0~1; Brightness: 0.5~2.0
NormalNormal Map / Normal Intensity✅Intensity: 0~2.0
PBR ControlRoughness / Metallic / Specular Override✅各 0~1
EmissiveEmissive Color / Intensity / Pulse Speed✅Intensity: 0~50; Speed: 0~5
EffectDissolve Threshold / Rim Power / Rim Color✅由 TA 定义安全范围
⚠️ InternalUV Tiling / Vertex Offset Mask / LOD Bias❌ 不暴露TA 内部调控
⚠️ 制作人提醒：参数暴露过多 = 美术调崩风险大；参数暴露过少 = 美术需求走 TA 单太慢。建议每类材质暴露 8~15 个可调参数，并配备 预设参考值（Default Preset）。

📌 红线：实例化深度 ≤ 3 层。超过 3 层的嵌套实例在引擎中查找参数来源困难，且增加编辑器卡顿风险。

3.2 参数继承与锁定规则

  • 锁定参数：一级实例中设定的光照模型开关（如 SSS ON），下级实例不可覆盖
  • 继承参数：数值型参数（Roughness、Metallic）可被下级实例覆盖
  • 范围约束：TA 可为暴露参数设置 min/max 范围，防止美术调出物理不合理的值（如 Metallic=0.5 的塑料）

⚡ 4. Shader 变体管控与性能红线

"Shader 变体膨胀是'温水煮青蛙'——单个开关看似无害，但组合爆炸后编译时间翻十倍、包体膨胀数百 MB、首帧加载卡顿——等制作人意识到已经来不及了。"

— TA Lead 血泪教训

4.1 变体数量控制策略

控制手段适用场景效果
Variant Stripping剔除平台无用变体（如 PC 剔除移动端 LDR 路径）变体数 -40~60%
Feature Level Gating按画质档位裁剪高端特性低端包体 -30%
Shader Warmup启动时预编译高频变体消除首帧卡顿

合法组合白名单明确哪些 Switch 组合允许共存防止组合爆炸

📦 5. 贴图通道打包与内存优化

5.1 通道打包标准（Channel Packing）

核心原则：通道打包可以将多张灰度图合并为一张 RGBA 贴图的不同通道，减少 Texture Sample 次数和内存占用。这是 TA 最重要的优化手段之一。
打包贴图名R 通道G 通道B 通道A 通道压缩格式
T_{Name}_MRAMetallicRoughnessAO—（RGB only）BC7 / ASTC 4×4
T_{Name}_MRAEMetallicRoughnessAOEmissive MaskBC7 / ASTC 4×4
T_{Name}_Mask换色区域 1换色区域 2换色区域 3Alpha/透贴BC7 / ASTC 4×4
T_{Name}_DetailDetail Normal XDetail Normal YDetail AOCavityBC5(RG) + BC4(BA)

📌 红线：禁止出现"一个参数用一张独立贴图"的情况。Metallic、Roughness、AO 必须打包到同一张贴图的不同通道。违反此规则每多一张贴图 = 多一次 Texture Sample = 额外 GPU 开销。

5.2 贴图内存预算（单角色）

贴图类型尺寸格式压缩后内存数量合计
BaseColor2048×2048ASTC 4×4 / BC7~2.7 MB12.7 MB
Normal2048×2048ASTC 4×4 / BC5~2.7 MB12.7 MB
MRA Pack1024×1024ASTC 4×4 / BC7~0.7 MB10.7 MB
Mask/换色512×512ASTC 4×4~0.17 MB10.17 MB
单角色贴图总预算（移动端）≤ 8 MB
单角色贴图总预算（PC/主机）≤ 16 MB

5.3 Mipmap 与 Streaming 策略

  • 所有 2D 贴图必须生成完整 Mipmap Chain，禁止关闭 Mipmap（UI 贴图除外）
  • Texture Streaming：场景贴图必须开启 Streaming，角色贴图近景保持 Full Mip、远景自动降级
  • Virtual Texture（UE5 专属）：大世界地表材质推荐使用 RVT 减少同时加载贴图数
  • LOD Bias：LOD2+ 的模型自动使用降 1~2 级的 Mip，减少无效高清贴图占用

📋 7. 材质版本管理与迭代 SOP

7.1 材质修改分级

修改等级定义影响范围审批流程
L1 微调Instance 参数调整（颜色/数值）仅影响单个资产美术自行操作，无需审批
L2 变更新增 Instance / 替换贴图影响单个角色/物件美术组长确认
L3 重大修改 Master Material / 新增开关影响所有使用该 Master 的实例TA Lead + 制作人双签

7.2 Master Material 修改 SOP

  1. 分支开发：在版本管理中创建 feature/mat_{功能名} 分支
  2. 影响评估：列出所有引用此 Master 的 Instance 列表，确认兼容性
  3. 自动化测试：运行材质编译检查脚本，确保所有 Instance 编译通过
  4. 性能测试：对比修改前后的指令数/采样数/帧时间
  5. 视觉回归：截图对比关键角色/场景的材质表现，确认无意外变化
  6. Code Review：TA 团队内部 Review
  7. 制作人终审：确认修改必要性和风险可控
  8. 合入主干：合并并通知所有美术"材质已更新，请检查自己负责的资产"
⚠️ 关键提醒：Master Material 的修改是"牵一发动全身"的操作。一次不小心的修改可能导致数百个 Material Instance 出现视觉异常。因此 L3 修改必须有完整的 Before/After 对比截图存档。

🟢🔴 9. Do's & Don'ts 红黑榜

✅ Do — 正确做法

  • 所有新材质从已有 Master Material 创建 Instance
  • 贴图 Metallic/Roughness/AO 打包为 MRA 贴图
  • Static Switch 开关使用语义化命名 bUse{Feature}
  • 每个 Master Material 配备参数文档 + 预设值表
  • 材质修改后运行全量 Instance 编译验证
  • TA 工具暴露调试模式供美术自查（BaseColor Only / Roughness Only 等）
  • 移动端材质使用 half 精度（除 WorldPosition 外）
  • 场景地表使用 Virtual Texture 减少 Draw Call

❌ Don't — 错误做法

  • 美术自行创建 Material 文件（应创建 Instance）
  • Metallic、Roughness、AO 各用一张独立灰度贴图
  • 一个 Master Material 塞 20+ 个 Static Switch（变体爆炸）
  • 修改 Master Material 不通知下游美术（导致视觉回归失败）
  • 把不需要透明的材质设为 Translucent（性能 3x 开销）
  • 全量使用 float 精度（移动端 GPU 吃不消）
  • 关闭 Texture Mipmap（远景贴图锯齿 + 显存浪费）
  • 材质中使用 sin(Time) 不加 Period 限制（长时间运行精度丢失）

✅ 10. 出包前材质审查 Checklist

📋 材质审查清单（发版前必检）

01Master Material 数量：是否在预算范围内（不超过项目规范上限）
02Shader 变体总数：移动端 ≤ 5000 / PC ≤ 20000
03超标材质清零：无任何材质超过平台指令数红线
04通道打包验证：无独立 Metallic/Roughness/AO 灰度贴图存在
05Mipmap 完整性：所有 3D 贴图已生成完整 Mip Chain
06Texture Streaming：场景贴图全部开启 Streaming
07PSO/Shader Cache：已生成并集成到安装包
08跨平台一致性：关键角色在 iOS/Android/PC 三平台截图对比通过
09材质权限审计：无美术越权创建的 Material 文件（只应有 Instance）
10废弃材质清理：无未引用的 Material / Instance 残留在包内
💡 自动化建议：以上 10 项中的 1~7 项均可通过 CI/CD 脚本自动检测。建议 TA 团队在构建流水线中集成材质健康检查 Step，每次出包自动生成报告。
