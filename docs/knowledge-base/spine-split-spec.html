1️⃣ 性能红线与规格限制

⛔ 红线原则：Spine Atlas 图集单张尺寸 不得超过 2048×2048。超过此限制将触发 QA 自动报错，需拆分为多张 Atlas 或缩减部件尺寸。目标移动端项目建议控制在 1024×1024 以内。

单部件最大尺寸512×5121024×1024单个部件不应占满整张 Atlas

3️⃣ 图文红黑榜 (Do's & Don'ts)

案例一：部件边缘黑边/白边（Fringing）

❌ Don't — 裁剪时未扩展边缘

直接按角色轮廓精确裁剪部件，expandPx = 0。部件 PNG 的边缘像素就是角色的抗锯齿半透明像素。

引擎后果：

  • Spine 运行时 线性采样 时，边缘像素与透明区域（RGBA=0,0,0,0）插值，产生 黑边（Dark Fringing）
  • 如果透明区域为白色（RGBA=255,255,255,0），则产生 白边（Light Halo）
  • 角色运动时边缘 闪烁，在浅色/深色背景上交替出现黑/白线
  • 缩放动画时问题 更加明显（缩小时半透明像素占比增大）
📷 配图建议：截取一个部件 PNG 放大到 800%，标注边缘 1px 处出现的黑色/深色缝隙；另截取引擎运行时浅色背景下的角色关节处黑线

✅ Do — 使用拓补扩展 + 内容感知填充

在工具中设置 expandPx = 4px，选择 "内容感知" 填充模式：

  • 裁剪区域向外扩展 4px，扩展区域由 5 采样点平均 + 距离衰减 自动填充颜色
  • 扩展区域的 Alpha 从边缘向外 渐变衰减至 0，形成自然过渡
  • 线性采样时插值到的是 相近颜色 而非黑色/白色，消除 Fringing
  • 套索选区使用 Polygon Offset 算法沿法线方向均匀扩展，避免尖角异常
📷 配图建议：同一部件的拓补预览截图（工具右侧面板），绿色框 = 原始区域、蓝色区域 = 拓补扩展，可见扩展区颜色与原始区域平滑过渡

案例二：Atlas 排布超尺寸导致运行时崩溃

❌ Don't — 无节制添加部件 + 高扩展值

给一个机甲角色添加了 12 个部件，每个都用了 expandPx = 15px，原画尺寸 3000×4000px。

引擎后果：

  • Atlas 排布后 POT 尺寸被撑到 4096×4096（= 64MB RGBA VRAM）
  • 中低端移动设备 纹理上传超时，甚至 GPU OOM 崩溃
  • Spine 运行时首帧 卡顿 200ms+（纹理解码）
  • 即使 PC 端不崩溃，也会占用过多显存，影响同屏其他角色
📷 配图建议：工具 QA 面板截图，显示 "🔴 图集尺寸预警：预估 Atlas 尺寸 4096×4096 超过 2048×2048"，并标注 expandPx=15 的设置

✅ Do — 控制预算 + 导出前 QA

  • 遵守部件数量建议（人形 ≤ 7，机甲 ≤ 8，自定义 ≤ 15）
  • expandPx 控制在 2~6px，推荐默认值 4px
  • 原画尺寸如超过 2048px 的任一边，先 等比缩放 到 2048 以内再导入工具
  • 导出前点击"🔍 质检 QA"，确认 Atlas 预估尺寸在预算内
  • QA 通过后再执行导出，工具会在导出前自动执行一次 QA，如有 Error 会弹出确认
📷 配图建议：QA 面板截图显示 "✅ 全部通过 · 图集尺寸正常：预估 Atlas 1024×1024"

5️⃣ 常见问题排查 (FAQ)

Q1 拆分后部件边缘出现黑边/白边，引擎里更明显？

根因：这是 Spine 运行时 线性纹理采样（Bilinear Filtering）+ 预乘 Alpha（Premultiplied Alpha）共同作用的结果。当边缘像素的 Alpha=0 但 RGB 不是 0 时（直通 Alpha 模式），或 Alpha=0 且 RGB=0 时，插值结果分别产生白边或黑边。

完整解决方案：

  1. 工具端：确保 expandPx ≥ 4 + 填充模式 = 内容感知。扩展区域的 RGB 与角色边缘颜色相近，Alpha 渐变归零
  2. 导出时：检查 PNG 透明区域的 RGB 值。推荐做法是将透明区域的 RGB 设为 最近不透明像素的颜色（内容感知模式自动完成）
  3. Spine 端：确认图集打包时 premultiplyAlpha = true（Spine 默认）。如果手动制作 Atlas，需确保 Blend 模式与 PMA 一致
  4. 引擎端：
    • UE5：Spine Widget 的 BlendMode 设为 Translucent + PMA
    • Unity：SpineAtlasAsset → PMA 勾选；Shader 使用 One, OneMinusSrcAlpha
Q2 导入引擎后 Atlas 上某些部件位置偏移/错位？

根因：工具导出的 spine_data.json 中附件坐标使用的是 原图坐标系（左上角为原点，Y 轴向下），而 Spine 编辑器使用 骨骼坐标系（Y 轴向上）。工具已自动做了 Y 轴翻转：y = imgHeight - (originY + originH/2)。

排查步骤：

1. 打开 spine_data.json，检查每个 attachment 的 x/y/width/height
2. 确认 skeleton.height 与原图高度一致
3. 在 Spine 编辑器中创建新项目：
   a. 导入 Atlas（如使用合图）或逐个导入部件 PNG
   b. 使用 JSON 导入模式加载骨骼数据
   c. 如有偏移，手动调整 root bone 位置
4. 如使用独立 PNG 模式（非 Atlas），需在 Spine 中
   手动设置每个区域附件的位置和尺寸

提示：atlas_mapping.json 包含每个部件在 Atlas 中的精确 UV 坐标，可用于自定义加载器。

Q3 引擎中 Alpha 预乘（PMA）和直通（Straight）怎么选？会影响部件渲染吗？

核心概念：

  • 直通 Alpha（Straight）：RGB 和 Alpha 独立存储。RGBA(255, 0, 0, 128) 表示"50% 透明的纯红"
  • 预乘 Alpha（PMA）：RGB 已经乘以 Alpha。同样效果变为 RGBA(128, 0, 0, 128)

Spine 推荐：使用 预乘 Alpha（PMA），原因：

  1. PMA 在 线性采样 时不会产生错误的颜色插值（直通模式在半透明区域插值会引入黑色/白色）
  2. Spine 运行时默认使用 PMA 混合模式：src=GL_ONE, dst=GL_ONE_MINUS_SRC_ALPHA
  3. 如果部件 PNG 是直通 Alpha（工具默认导出格式），需要在 Atlas 打包时 勾选 premultiplyAlpha

引擎配置：

引擎设置位置推荐值
UE5 (spine-ue5)SpineWidget → BlendModeTranslucent + PMA
Unity (spine-unity)SkeletonRenderer → Advanced → PMA☑ Premultiply Alpha
Cocos Creatorsp.Skeleton → premultipliedAlphatrue
⚠️ 常见坑：如果引擎端开了 PMA 但 Atlas 没有预乘，角色会整体偏暗。反之如果 Atlas 已预乘但引擎用 Straight 混合，角色会变亮且边缘异常。两端必须一致。
