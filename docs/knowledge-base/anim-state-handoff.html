🚨 6. 常见踩坑案例

避坑指南：以下是动画交接中最高频的问题，严重时可导致整个角色重做。
🚨 案例 1：骨骼不匹配 高频

导入引擎后动画错乱、变形、T-Pose 残留。

典型场景：美术更新了骨骼结构（加了辅助骨骼），导出新动画。程序用旧骨骼导入新动画，导致全身错乱。

解决方案：① 骨骼变更必须通知所有相关方 ② 使用版本号管理骨骼（如 SK_Humanoid_v03）③ 引擎中统一使用共享 Skeleton Asset ④ 建立骨骼变更审批机制

🚨 案例 2：动画偏移 高频

角色播放动画后位置偏移/不回原点。

典型场景：Attack 动画中 Root 骨骼有位移数据残留。引擎中未关闭 Root Motion，角色每次攻击后向前滑。

解决方案：① 明确标注哪些动画需要 Root Motion ② 不需要 Root Motion 的动画，Root 骨骼建议在原点不动 ③ 在交接文档中增加 Root Motion 标注列

🚨 案例 3：状态机死循环 中频

两个状态互相跳转，动画抖动。

典型场景：Idle→Run 条件为 Speed>0，Run→Idle 条件为 Speed<1。Speed=0.5 时两个状态每帧互跳。

解决方案：① 添加 cooldown / 最小停留时间 ② 转换条件添加互斥逻辑 ③ 使用 ExitTime 而非纯条件触发 ④ 在状态转换条件表中逐条审查

🚨 案例 4：过渡闪烁 中频

状态切换瞬间出现闪烁/抖动。混合时间设置不当，差异极大的 Pose 直接混合。

解决方案：① 攻击→待机混合时间 0.2~0.3s ② 受击混合时间 0.05s ③ 避免两个差异极大的 Pose 直接混合 ④ 在状态机总览表中明确标注混合时间

🚨 案例 5：动画裁切 中频

动画播到一半被打断，看起来不自然。可打断标记设置有误。

解决方案：① 攻击动画在 ComboWindow 内才可打断 ② 技能前摇不可打断，后摇可打断 ③ 标注清楚每个动画的可打断窗口 ④ 在状态转换条件表中增加"可打断"列

🔄 8. 动画 Retarget 与骨架复用体系

🦴 8.1 骨架共享架构
核心策略：一套 Master Skeleton 覆盖 80% 以上人形角色，通过 Retarget 实现动画复用。
骨架类型适用范围骨骼数维护责任人
SK_Humanoid_Master所有人形角色（男/女/儿童通过比例缩放）85 骨骼TA Lead
SK_Monster_Biped二足怪物/Boss65 骨骼TA
SK_Monster_Quadruped四足生物72 骨骼TA
SK_Creature_Custom特殊形态（触手/多肢/飞行）按需定制TA + 绑定师
📐 8.2 Retarget 设置标准
引擎Retarget 方案关键设置注意事项
UE5IK RetargeterIK Rig 定义 → Chain Mapping → Retarget Pose 校准体型差异大时需手动调整 Chain 缩放
UnityHumanoid AvatarAvatar → Configure → Muscle 范围校准非标准骨骼需手动映射到 Avatar Bone
⚠️ Retarget 失败高频原因：① T-Pose/A-Pose 不一致（源与目标 Pose 不同）→ 修正 Retarget Pose；② 手指骨骼数不匹配 → 忽略或映射到最近骨骼；③ 武器插槽位置偏移 → 使用 Socket 相对坐标而非绝对坐标。
♻️ 8.3 动画复用优先级
动画类型复用策略节省工时
Locomotion (Walk/Run/Idle)全复用：所有人形角色共享一套 Locomotion~80%
基础动作 (Hit/Death/Stun)Retarget + 微调：共享后针对体型调整混合~60%
攻击/技能角色专属：SSR 原创，SR 可变体复用，R/N 共享R/N ~50%
表情/口型Morph Target 共享同一套 ARKit 52 Blend Shape~90%

📦 9. 动画压缩与内存优化策略

📊 9.1 压缩方案对比
方案压缩率精度损失适用场景引擎支持
无压缩 (Raw)1:1无仅调试阶段UE5 / Unity
Uniform Sampling2~4×极低短动画（<30帧）UE5
Keyframe Reduction5~10×低通用动画UE5 / Unity
ACL (Animation Compression Library)10~20×极低所有量产动画（推荐默认）UE5 内置
Optimal (Unity)5~15×低Unity 默认推荐Unity
✅ 推荐配置：UE5 项目统一使用 ACL 压缩（默认即可），误差阈值：平移 0.1mm、旋转 0.1°。Unity 项目使用 Optimal + Keyframe Reduction。
💾 9.2 动画内存预算
角色品级动画片段数压缩后总内存说明
SSR（主角）40~60 clips≤ 8 MB含全套技能/表情/交互
SR（精英）20~30 clips≤ 4 MB含基础战斗+特殊技能
R（普通）12~18 clips≤ 2 MB共享 Locomotion + 少量专属
N（杂兵）8~12 clips≤ 1 MB大量复用
