⚙️ 3. FBX 导出设置标准

⚙️ 3.1 通用导出设置
量产必读：所有工种通用的 FBX 导出设置。
参数标准值说明
FBX 版本2020兼容 UE5 / Unity 2022+
帧率30 FPS移动端标准；PC 可用 60 FPS
轴向Y-Up (Maya) / Z-Up (Max)引擎端自动转换
Scale Factor1.0保持原始比例
单位厘米 (cm)UE 默认单位
🦴 3.2 骨骼导出设置
参数设置
骨骼层级完整导出，不剥离
Deformation Bones Only❌ 关闭（保留辅助骨骼）
SmoothingNormals and Tangents
Blend Shapes如有面部表情则勾选
🏃 3.3 Root Motion 设置
核心：Root Motion 配置不一致是动画偏移的第一大原因。
项目说明
需要 Root Motion 的动画Run / Walk / Dash / Roll
不需要 Root Motion 的动画Idle / Attack / Skill / Hit / Death
Root 骨骼要求必须有一根名为 Root 或 root 的根骨骼
Root 骨骼位移Run/Walk 的位移数据写在 Root 骨骼上

📋 FBX 导出前核查

☐删除所有 Helper/Dummy 对象

🎮 4. 引擎导入 Checklist

🎮 4.1 Unreal Engine
UE 导入 UE 时的逐项检查。
#检查项操作
1导入类型选择Skeletal Mesh + Animation
2Skeleton 选择选择已有的共享 Skeleton Asset
3Root Motion勾选 Enable Root Motion（需要的动画）
4AnimBlueprint 创建基于导入的 Skeleton 创建 ABP
5Retarget如需跨骨架复用，设置 IK Retargeter
6Notify 添加在关键帧添加 AnimNotify（打击帧/音效帧/特效帧）
7压缩设置Animation → Compression → 根据平台选择
🔷 4.2 Unity
Unity 导入 Unity 时的逐项检查。
#检查项操作
1Animation TypeHumanoid / Generic
2Avatar 配置Configure Avatar → 确认骨骼映射
3Root Motion勾选 Bake Into Pose（不需要的动画）
4Animator Controller创建 AC 并配置状态机
5Animation Events在关键帧添加 Event（攻击判定/音效/特效）
6压缩设置Animation Compression → Optimal

🚨 6. 常见踩坑案例

避坑指南：以下是动画交接中最高频的问题，严重时可导致整个角色重做。
🚨 案例 1：骨骼不匹配 高频

导入引擎后动画错乱、变形、T-Pose 残留。

典型场景：美术更新了骨骼结构（加了辅助骨骼），导出新动画。程序用旧骨骼导入新动画，导致全身错乱。

解决方案：① 骨骼变更必须通知所有相关方 ② 使用版本号管理骨骼（如 SK_Humanoid_v03）③ 引擎中统一使用共享 Skeleton Asset ④ 建立骨骼变更审批机制

🚨 案例 2：动画偏移 高频

角色播放动画后位置偏移/不回原点。

典型场景：Attack 动画中 Root 骨骼有位移数据残留。引擎中未关闭 Root Motion，角色每次攻击后向前滑。

解决方案：① 明确标注哪些动画需要 Root Motion ② 不需要 Root Motion 的动画，Root 骨骼必须在原点不动 ③ 在交接文档中增加 Root Motion 标注列

🚨 案例 3：状态机死循环 中频

两个状态互相跳转，动画抖动。

典型场景：Idle→Run 条件为 Speed>0，Run→Idle 条件为 Speed<1。Speed=0.5 时两个状态每帧互跳。

解决方案：① 添加 cooldown / 最小停留时间 ② 转换条件添加互斥逻辑 ③ 使用 ExitTime 而非纯条件触发 ④ 在状态转换条件表中逐条审查

🚨 案例 4：过渡闪烁 中频

状态切换瞬间出现闪烁/抖动。混合时间设置不当，差异极大的 Pose 直接混合。

解决方案：① 攻击→待机混合时间 0.2~0.3s ② 受击混合时间 0.05s ③ 避免两个差异极大的 Pose 直接混合 ④ 在状态机总览表中明确标注混合时间

🚨 案例 5：动画裁切 中频

动画播到一半被打断，看起来不自然。可打断标记设置有误。

解决方案：① 攻击动画在 ComboWindow 内才可打断 ② 技能前摇不可打断，后摇可打断 ③ 标注清楚每个动画的可打断窗口 ④ 在状态转换条件表中增加"可打断"列
