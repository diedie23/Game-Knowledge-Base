🏷️ 1. 资产安全分级体系

核心原则：不是所有美术资产都需要同等保护。分级管理可以将有限的安全投入集中在最关键的资产上，避免"要么不管、要么过度管控"的两极化。
📊 1.1 四级安全分类
级别标签定义典型资产保护要求
🔴 绝密SECRET泄露将导致重大商业损失或法律风险未公开主角设定、核心玩法原画、市场营销素材（发布前）、IP 联动原始稿加密存储 + 水印 + 仅限授权人员访问 + 禁止截屏
🟡 机密CONFIDENTIAL泄露将影响竞争优势或外包管理量产中的角色模型/场景、技术美术方案、外包报价单水印 + 权限管控 + NDA 覆盖
🟢 内部INTERNAL仅限项目组内部使用过程稿、参考图收集、WIP 截图、内部会议素材内网访问 + 不外传
⚪ 公开PUBLIC已发布或授权对外使用已发布宣传图、公开版本截图、招聘展示用作品无特殊限制

📝 2. 外包 NDA 规范与签署流程

核心原则：NDA 不签，资产不发。这是铁律，没有例外。即使是"老合作伙伴"、"就发一张参考图"，都必须有有效的 NDA 覆盖。
📋 2.1 NDA 签署矩阵
合作方类型NDA 类型签署时间有效期审批人
长期 CP（年度框架）主协议 NDA + 项目补充协议框架合同签署时合同期 + 2 年脱密期法务 + 制作人
短期 CP（单批次）项目专属 NDA需求下发前项目结束 + 2 年脱密期APM + 法务
外部顾问/自由职业者个人 NDA首次接触项目信息前接触结束 + 3 年脱密期APM + 法务
CP 子外包（二包）严格禁止 or 需甲方书面同意——制作人

🔴 铁律：严禁二包。CP 未经甲方书面同意，不得将任何资产或需求转包给第三方。合同中必须写入"二包即解约 + 赔偿"条款。

📋 2.2 NDA 核心条款 Checklist

📝 NDA 必备条款（缺一不可）

☐保密范围定义：明确列出哪些信息属于保密信息（资产源文件、设定稿、技术方案、排期、报价等）
☐保密义务：不得向任何第三方披露、不得用于合同外用途、不得复制留存
☐脱密期：合作结束后 2~3 年内仍需保密
☐资产归还/销毁：合作结束后 7 个工作日内归还或销毁所有资产，并出具书面确认
☐禁止二包：未经甲方书面同意不得转包
☐违约赔偿：明确违约金金额或计算方式（建议合同总额 3~5 倍）
☐人员管控：CP 须提供接触项目资产的人员名单，人员变动需提前通知
☐作品展示限制：CP 不得将项目作品用于自身宣传/作品集，除非获得书面授权
☐设备与网络要求：CP 制作环境需满足基本安全要求（独立工位、禁止云盘同步等）

🖼️ 3. 水印工具使用规范

核心理念：水印不是为了"防君子"，而是为了泄密后的溯源追责。好的水印方案必须做到：① 不影响制作效率 ② 不影响视觉评审 ③ 泄密后能追溯到具体人员。
📊 3.1 水印类型矩阵
水印类型实现方式适用场景优点缺点
可见水印半透明文字/图案叠加外发参考图、评审截图、CP 下发的设定稿威慑力强，一眼可见影响视觉评审，可能被 P 掉
隐形水印DCT/DWT 频域嵌入🔴 绝密/🟡 机密级资产的源文件不影响视觉，难以去除需要专业工具提取
元数据水印EXIF/XMP 字段嵌入所有数字资产实现简单，可批量容易被清除
屏幕水印DLP 软件在屏幕叠加内部工作站、远程桌面防截屏/拍照影响工作体验

👋 5. 离职人员资产交接 SOP

核心原则：离职交接的目标不是"把文件拷过来"，而是确保① 资产零丢失 ② 权限零残留 ③ 知识零断层。一个关键美术离职如果交接不到位，可能导致数周的排期延误。

时间点执行动作责任人输出物

🔒 5.3 权限回收 Checklist

🔐 最后工作日必须完成（IT + APM 联合执行）

☐SVN / Perforce 账号：禁用账号（不是删除，保留审计记录）
☐JIRA / TAPD 账号：移除项目权限，任务重新指派
☐NAS / 共享盘：移除访问权限
☐企业邮箱：设置自动转发（转给接手人），30 天后停用
☐企业微信/飞书/钉钉：移出项目群组
☐VPN 账号：立即禁用
☐软件授权：回收 Maya/Max/PS/SD 等商业软件 License
☐云服务：移除 AWS/阿里云/腾讯云等账号权限
☐物理设备：回收工作电脑、移动硬盘、数位板等（IT 格式化后再分配）
☐门禁/工卡：注销门禁权限
🚨 高风险场景：如果离职人员是被辞退或竞业限制对象：
· 通知日当天即冻结全部系统权限（IT 在 HR 通知后 2 小时内完成）
· 在 IT 监督下清理个人设备中的项目资产
· 法务出面确认竞业限制条款

📌 8. 典型案例与 Do / Don't

🔴 案例 1：CP 员工跳槽带走源文件 真实高频

🎬 场景：CP-Alpha 的核心建模师跳槽到竞品项目的 CP。3 个月后，竞品游戏中出现了风格高度相似的角色。调查发现该建模师将源文件存在了个人移动硬盘中。

💡 事后复盘：
① NDA 中缺少"CP 员工离职需通知甲方"条款
② 未要求 CP 对员工设备进行管控
③ 源文件未嵌入隐形水印，无法提供侵权证据

🔧 预防措施：
① NDA 增加"CP 核心人员变动须提前 7 天书面通知甲方"
② 合同要求 CP 对参与人员的设备实施基本管控（禁止拷贝到个人设备）
③ 所有下发资产必须嵌入隐形水印

🟡 案例 2：离职美术的 Maya 工程丢失 中频

🎬 场景：资深角色美术离职后，接手人发现 SVN 上的 FBX 是导出文件，但 Maya 源文件（.ma）在离职人的本地电脑上，IT 已经格式化了电脑。Maya 工程中包含自定义 blendshape 和表情控制器，无法从 FBX 逆向还原。

💡 破局：交接清单中必须增加"源文件完整性检查"——不仅检查 SVN 上的导出文件，还要确认DCC 源文件（.ma/.max/.blend）已提交到版本库。

8.1 资产安全 Do / Don't

✅ Do（正确示范）

• NDA 先签，资产后发，没有例外

• 所有外发资产必加水印（可见 + 元数据）

• 权限最小化，按角色/工种分配

• 离职交接T-14 启动，留足缓冲

• DCC 源文件必须上传版本库

• 泄密事件证据先行，不打草惊蛇

❌ Don't（错误示范）

❌ "老合作伙伴了，NDA 走个形式" → 出事无法追责

❌ 微信直接发源文件 → 零审计、零水印

❌ 所有人都能访问所有目录 → 权限形同虚设

❌ 最后一天才开始交接 → 必定遗漏

❌ 只交付 FBX 不交源文件 → 后续无法修改

❌ 发现泄密先私下解决 → 错过最佳追责窗口
