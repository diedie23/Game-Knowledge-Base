1.3 制作跟进

🔴 首件制度是核心：80% 的批量返工源于首件审核不严格。首件必须由 Art Lead 亲自审核，审核通过后形成书面确认，作为后续批量制作的质量基准。

检查点时间检查内容形式
首件审核启动后 3~5 天完成 1 件成品，全面评审风格/质量/格式在线会议 + 标注截图
周进度同步每周固定时间完成数量、在制数量、问题清单进度表 + 截图
中期检查50% 进度时批量质量抽检、风格一致性、排期偏差抽查 3~5 件 + 进度报告
交付前预检交付前 2~3 天提前抽检最终成品，确认无重大问题在线预览
正式交付约定日期全量检查，出具验收报告Checklist + 验收报告
1.4 三轮验收制度
轮次检查重点参与人输出物
第一轮：技术验收格式、尺寸、命名、面数、UV、通道等硬性指标APM / TA技术问题清单
第二轮：美术验收风格、质量、细节、色彩、一致性Art Lead / 主美美术修改意见
第三轮：引擎验收引擎内实际表现、性能、LOD、材质效果TA / 程序引擎截图 + 性能数据

1.6 结算流程
步骤内容负责人输出
1. 验收签收全量资源通过验收，双方签字确认APM + CP验收签收单
2. 工作量核算统计实际交付数量、规格、品质等级APM工作量确认表
3. 费用计算按合同单价 × 实际数量，扣除不达标项APM + 财务费用明细表
4. 申请付款提交付款申请 + 验收材料APM付款申请单
5. 资料归档合同/验收单/沟通记录/源文件归档APM项目档案
结算注意事项：· 验收签收前不启动付款流程 · 修改超过 3 轮的资源，评估是否需要调整单价 · 保留完整的沟通记录和每版本文件，作为争议解决依据 · 建议分阶段付款（首件通过 30% + 中期 30% + 终验 40%）

🚨 2. 常见问题处理手册

2.1 资源不达标
问题典型表现立即处理长期预防
模型面数超标角色交付面数远超预算（如预算 15K，实际 25K）发送面数超标通知，标注具体数值和预算上限需求下发时附带面数检测脚本
PBR 材质效果差金属像塑料 / 布料像金属；SP 效果好引擎差异大提供标准 Checker Scene + PBR 数值参考表要求 CP 在引擎内截图而非 SP 内截图验收
命名目录不规范文件名含中文/空格/特殊字符；目录不一致随需求下发提供交付目录模板 ZIP使用批量重命名工具自检后再提交
风格不统一不同批次或不同美术师产出风格差异明显提供 5~10 张已审核标杆作品指定同批资源由同一美术师完成
2.2 延期处理

🔴 预防措施：① 排期预留 20% Buffer ② 设置中间检查点（50% 强制确认）③ 同类资源准备备选 CP

延期程度应对措施
1~3 天口头沟通原因，要求赶工 + 每日进度更新
3~7 天书面警告 + 评估是否影响项目里程碑 + 通知项目组
>7 天启动备选 CP + 评估合同违约条款 + 重新评估合作关系

2.4 CP 失联/不回复升级流程
时间动作
失联 1 天工作群 + 邮件 + 电话三渠道同时联系
失联 2 天联系 CP 对接人的直属上级

2.5 其他常见问题
问题处理方案
版权争议 / 素材来源不明合同中要求 CP 签署原创性声明；敏感资源进行相似度查重；禁止 AI 生成素材作为最终交付物
多 CP 协作冲突由 APM 统一分配任务边界；避免不同 CP 直接沟通；资源交接点建立衔接 Checklist
CP 更换核心人员合同约定核心人员变更需提前通知；新人员必须通过首件测试；评估对排期的影响

✅ 3. 按资源类型分类的验收 Checklist

使用说明：以下 Checklist 按资源类型分类，每个 Checklist 包含技术验收（硬性指标）和美术验收（主观评估）两部分。P0 项为必须通过项，任一 P0 项不通过即判定为不合格。
3.1 角色 3D 模型

🔧 技术验收（P0 全部必须通过）

☐[P0] 三角面数符合预算 — 主角 ≤30K / 精英 NPC ≤15K / 杂兵 ≤8K
☐[P0] 法线方向全部正确，无黑面/翻面
☐[P0] UV 展开无重叠（UV0），Texel Density 偏差 ≤15%
☐[P0] PBR 贴图套装齐全 — BaseColor / Normal / ORM
☐[P0] 贴图尺寸为 2^N（如 2048×2048）
☐[P0] 文件命名符合规范，无中文/空格/特殊字符
☐[P1] LOD 至少 3 级，切换距离合理
☐[P1] 骨骼蒙皮权重正确，极端 Pose 无穿模/塌陷
3.2 角色立绘（2D）

🎨 视觉 + 技术验收

☐[P0] 分辨率 ≥ 2048px（长边），色彩模式 sRGB
☐[P0] PSD 图层分组清晰 — 线稿/底色/阴影/高光/特效分层
☐[P0] PNG 导出含正确 Alpha 通道
☐[P0] 角色比例与设定画一致，头身比误差 ≤5%
☐[P0] 画风与项目基准统一
☐[P1] 光影方向统一（默认左上方 45°）
3.3 场景资产

🔧 技术验收

☐[P0] 面数符合预算 — 大物件 ≤5K / 中物件 ≤2K / 小物件 ≤500
☐[P0] 模块化拼接无缝隙、无法线错误
☐[P0] Lightmap UV 无重叠
☐[P0] 贴图分辨率匹配 — 大物件 2048 / 中物件 1024 / 小物件 512
☐[P1] Pivot Point 正确；碰撞体简化合理
3.4 UI 资源

🔧 技术验收

☐[P0] 命名规范 — ui_模块_元素_状态.png
☐[P1] 交互状态齐全 — Normal / Pressed / Disabled / Hover
3.5 动画资源

🔧 技术验收

☐[P0] 帧率 30FPS（3D）/ 12~24FPS（2D 帧动画）
☐[P0] Root Motion 与实际位移匹配
☐[P0] Loop 动画首尾帧无跳变
☐[P1] 无自穿插；重心符合物理规律
3.6 特效资源

🔧 技术验收

💡 5. APM 外包管理经验总结

5.1 核心原则
外包管理六字诀：清晰、前置、留痕
· 清晰 — 需求越清晰，返工越少。模糊需求 = 返工成本 × 3
· 前置 — 首件制度、规范前置、风险前置。后期补救的成本是前期预防的 5~10 倍
· 留痕 — 所有关键决策通过邮件确认。口头约定 = 没有约定
5.2 常见误区

❌ Don't — 常见误区

· "CP 是专业的，不用说太细"→ 每家 CP 标准不同，不说清就各做各的

· "首件差不多就行，批量再改"→ 批量返工成本是首件修改的 10x

· "群里说了就算数"→ 事后扯皮无据可查

· "只看最终效果，过程不管"→ 到最后才发现问题，补救困难

· "便宜的 CP 更省钱"→ 低价 CP 返工率高，综合成本反而更贵

· "一家 CP 做所有"→ 单点依赖风险大

✅ Do — 正确做法

· 提供详细规范 + 参考图 + 交付模板

· 首件必须严格达标后才批量

· 重要决定邮件确认

· 设置中间检查点 + 周进度同步

· 用 CP 评估表综合评分选择

· 核心资源至少 2 家 CP 备选
