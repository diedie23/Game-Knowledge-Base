面向游戏美术 APM 的外包全流程管理指南 — 需求下发、制作跟进、验收、结算

💡

1.1 需求下发

📋
核心原则：需求越清晰，返工越少。下发时必须做到"四有"——有规范、有参考、有排期、有验收标准。

检查点时间检查内容形式

⚠️
首件制度是核心：80% 的批量返工源于首件审核不严格。首件必须由 Art Lead 亲自审核，审核通过后形成书面确认，作为后续批量制作的质量基准。

1.4 结算

结算流程

步骤内容负责人输出
1. 验收签收全量资源通过验收，双方签字确认APM + CP验收签收单
2. 工作量核算统计实际交付数量、规格、品质等级APM工作量确认表
3. 费用计算按合同单价 × 实际数量，扣除不达标项APM + 财务费用明细表
4. 申请付款提交付款申请 + 验收材料APM付款申请单
5. 资料归档合同/验收单/沟通记录/源文件归档APM项目档案
✅
结算注意事项：
· 验收签收前不启动付款流程
· 修改超过 3 轮的资源，评估是否需要调整单价
· 保留完整的沟通记录和每版本文件，作为争议解决依据
· 建议分阶段付款（首件通过 30% + 中期 30% + 终验 40%）

二、常见问题处理手册

2.1 资源不达标

🔴 问题 A：模型面数超标
高频3D角色3D场景

典型表现

  • 角色模型交付面数远超预算（如预算 15K，实际 25K）
  • 场景单个资产面数过高，导致整体 Draw Call 爆炸

根因分析

  • CP 未认真阅读面数预算要求
  • 高模拓扑时未做减面优化
  • 不了解目标平台性能限制

处理方案

优先级动作
立即发送明确的面数超标通知，标注具体数值和预算上限
短期提供减面参考（哪些区域可减面，保持轮廓的优先级）
长期在需求下发时附带面数检测脚本（3ds Max / Maya 脚本）
🔴 问题 B：PBR 材质效果差
高频3D角色3D场景

典型表现

  • 金属看起来像塑料 / 布料看起来像金属
  • Substance Painter 中效果好，引擎中差异大
  • 法线贴图烘焙出现接缝 / 黑斑

处理方案

  • 提供标准 Checker Scene（固定灯光 + 参考球 + 对比材质）
  • 要求 CP 在引擎内截图而非 SP 内截图验收
  • 提供 PBR 数值参考表（金属 Metallic=1, Roughness=0.3~0.6 等）
🟡 问题 C：命名和目录不规范
中频所有类型

典型表现

  • 文件名包含中文 / 空格 / 特殊字符
  • 目录结构与规范不一致
  • 编号不连续 / 重复

处理方案

  • 随需求下发提供 交付目录模板 ZIP（空目录结构 + README）
  • 使用批量重命名工具自检后再提交
  • 首次违规口头提醒，累计 3 次写入质量评估
🟡 问题 D：风格不统一
高影响立绘角色

典型表现

  • 不同批次或不同美术师的产出风格差异明显
  • 配色偏离项目调色板
  • 细节精细度不一致

处理方案

  • 指定同一批资源由同一美术师完成
  • 提供 5~10 张已审核通过的标杆作品 作为风格基准
  • 每批次首件必须由 Art Lead 确认后才批量
  • 严重不统一时要求换人或终止合作

2.2 延期处理

🔴 CP 交付延期

预防措施

  • 排期预留 20% Buffer（如实际需 10 天，约定 12 天）
  • 设置 中间检查点，50% 进度时强制确认
  • 同类资源准备 备选 CP，可在紧急时分流

延期后应对

延期程度应对措施
1~3 天口头沟通原因，要求赶工 + 每日进度更新
3~7 天书面警告 + 评估是否影响项目里程碑 + 通知项目组
>7 天启动备选 CP + 评估合同违约条款 + 重新评估合作关系

延期沟通模板

📝 延期沟通邮件
主题：【紧急】[资源名称] 交付延期跟进 Hi [CP名称] 团队， 本批次资源原定 [日期] 交付，目前已逾期 [X] 天，请反馈以下信息： 1. 延期原因说明 2. 当前实际完成进度（百分比 + 截图） 3. 预计最终交付日期 4. 是否需要协调支持 请在今日内回复，感谢配合。 [你的名字] · APM

2.3 沟通失联

🔴 CP 失联/不回复

升级流程

时间动作
失联 1 天工作群 + 邮件 + 电话三渠道同时联系
失联 2 天联系 CP 对接人的直属上级
失联 3 天发送正式书面函件（邮件，抄送双方管理层）
失联 5 天启动应急预案：备选 CP 接手 + 评估合同违约
失联 7 天+终止合作 + 法务介入（如涉及预付款）
🚨
预防建议：
· 合同中明确约定"响应时效"条款（如 24 小时内必须回复）
· 每个 CP 至少 2 个联系人（对接人 + 其上级）
· 重要沟通全部通过邮件留档，群聊内容定期截图备份

2.4 其他常见问题

🟡 版权争议 / 素材来源不明
  • 合同中明确要求 CP 签署 原创性声明
  • 敏感资源（角色脸部/标志性元素）进行 相似度查重
  • 禁止使用 AI 生成素材作为最终交付物（除非合同明确允许）
  • 如发现抄袭，立即停止使用 + 要求 CP 承担全部法律责任
🟡 多 CP 协作冲突
  • 不同 CP 负责同一项目的不同模块时，由 APM 统一分配任务边界
  • 避免不同 CP 之间直接沟通（信息需通过 APM 中转）
  • 资源衔接点（如角色 CP 交模型 → 动画 CP 做动画）明确交接标准
  • 建立 资源交接 Checklist，防止中间环节信息丢失
🟡 CP 更换核心人员
  • 合同中约定 核心人员变更需提前通知
  • 新人员必须通过 首件测试 后才能接手批量任务
  • APM 需评估人员变更对排期的影响并及时上报

三、按资源类型分类的验收 Checklist

💡
使用说明：以下 Checklist 按资源类型分类，每个 Checklist 包含技术验收（硬性指标）和美术验收（主观评估）两部分。P0 项为必须通过项，任一 P0 项不通过即判定为不合格。

3.1 角色 3D 模型

🔧 技术验收（P0 全部必须通过）

☐[P0] 三角面数符合预算 — 主角 ≤30K / 精英 NPC ≤15K / 杂兵 ≤8K
☐[P0] 法线方向全部正确，无黑面/翻面
☐[P0] UV 展开无重叠（UV0），Texel Density 偏差 ≤15%
☐[P0] Lightmap UV（UV1）无重叠，Padding ≥2px
☐[P0] PBR 贴图套装齐全 — BaseColor / Normal / ORM
☐[P0] 贴图尺寸为 2^N（如 2048×2048）
☐[P0] 文件命名符合规范，无中文/空格/特殊字符
☐[P1] LOD 至少 3 级（LOD0/LOD1/LOD2），切换距离合理
☐[P1] 骨骼蒙皮权重正确，极端 Pose 无穿模/塌陷
☐[P1] Pivot Point 位于底部中心
☐[P2] 碰撞体简化合理，面数 ≤ 模型 10%

🎨 美术验收

☐[P0] 模型造型与设定原画一致，比例正确
☐[P0] PBR 材质在引擎标准灯光下表现准确（金属像金属、布料像布料）
☐[P1] 细节精度符合项目标准（纽扣/缝线/花纹等）
☐[P1] 与同项目其他角色风格统一
☐[P2] 360° 无死角检查通过

3.2 角色立绘（2D）

🔧 技术验收

☐[P0] 分辨率 ≥ 2048px（长边）
☐[P0] PSD 图层分组清晰 — 线稿/底色/阴影/高光/特效分层
☐[P0] PNG 导出含正确 Alpha 通道
☐[P0] 色彩模式 sRGB
☐[P1] 文件命名符合规范
☐[P1] 同角色多版本立绘尺寸/角度一致（便于切换）

🎨 美术验收

☐[P0] 角色比例与设定画一致，头身比误差 ≤5%
☐[P0] 配色与设定方案 HEX 值偏差 ΔE < 3
☐[P0] 画风与项目基准统一（眼睛/头发/皮肤上色手法）
☐[P1] 饰品/纹样/材质质感符合设定
☐[P1] 光影方向统一（默认左上方 45°）
☐[P2] 无版权风险（非抄袭、非 AI 直出）

3.3 场景资产

五、APM 外包管理经验总结

5.1 核心原则

🎯
外包管理六字诀：清晰、前置、留痕

· 清晰 — 需求越清晰，返工越少。模糊需求 = 返工成本 × 3
· 前置 — 首件制度、规范前置、风险前置。后期补救的成本是前期预防的 5~10 倍
· 留痕 — 所有关键决策通过邮件确认。口头约定 = 没有约定

5.2 常见误区

误区后果正确做法
❌ "CP 是专业的，不用说太细"每家 CP 标准不同，不说清就各做各的✅ 提供详细规范 + 参考图 + 交付模板
❌ "首件差不多就行，批量再改"批量返工成本是首件修改的 10x✅ 首件必须严格达标后才批量
❌ "群里说了就算数"事后扯皮无据可查✅ 重要决定邮件确认
❌ "只看最终效果，过程不管"到最后才发现问题，补救困难✅ 设置中间检查点 + 周进度同步
❌ "便宜的 CP 更省钱"低价 CP 返工率高，综合成本反而更贵✅ 用 CP 评估表综合评分选择
❌ "一家 CP 做所有"单点依赖风险大✅ 核心资源至少 2 家 CP 备选
