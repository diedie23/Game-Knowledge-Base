📝 1. 命名规范说明

💡 核心思想：采用“语义化 + 状态后缀”的结构化命名，彻底摒弃 `icon_A1` 等无语义的命名方式。

为满足团队规范，统一采用大驼峰式命名 (PascalCase)。手机端（iOS/Android）严格区分大小写，因此必须严格遵循以下字典中的大小写规范。

基础命名公式：

IconTab[部位首字母大写名称][状态后缀首字母大写]
  
示例：
IconTabBagNml       // 背包页签-正常态
IconTabEyeSel       // 眼睛页签-选中态
IconTabTopDis       // 上衣页签-禁用/锁定态

🔄 3. UI 状态后缀 (States)

状态中文状态后缀 (配置代码中拼装)示例含义说明

⚠️ 警告：严禁大小写混用出错（如 nml、NML），一律使用首字母大写的后缀词缀 Nml、Sel、Dis。

🛠️ 4. 开发对接指南

实施此规范后，程序和策划可无缝对接，自动化读取图标。

1. 策划配置表填法：配置表中只需填写英文标准词（如 top、eye）。

2. 前端代码实现逻辑（伪代码）：

// 1. 从策划表读取关键字
let keyword = config.categoryKey; // 例如 "top"

// 2. 首字母大写处理
let capitalizeKey = keyword.charAt(0).toUpperCase() + keyword.slice(1); // 变成 "Top"

// 3. 动态拼接获取当前需要的状态图
let normalImg = "IconTab" + capitalizeKey + "Nml";     // 结果: IconTabTopNml
let selectedImg = "IconTab" + capitalizeKey + "Sel";   // 结果: IconTabTopSel
let disabledImg = "IconTab" + capitalizeKey + "Dis";   // 结果: IconTabTopDis
