① 三种换色方案全景对比

不同的换色技术路线对美术的出图要求差异极大。先定方案再出图，是避免返工的第一条流程约定。

维度🅰 Mask+HSV（推荐）🅱 调色板替换 Palette Swap🅲 灰度底图+叠色
原理Mask 图标记换色区域，Shader 做 HSV/Tint 偏移索引色+LUT 表颜色映射灰度图 × 目标色（乘法混合）
美术提供原图 + Mask 遮罩图 + 配色 JSON索引色原图 + N 张 256×1 调色板灰度底图 + 可选 Mask + 细节叠加层
换色区域数3区/Mask（多 Mask 可无限扩展）取决于索引色数量（8~32色）需配合 Mask 多区域
画面品质⭐⭐⭐⭐⭐ 保留完整光影⭐⭐⭐ 适合像素风⭐⭐⭐ 简单纯色效果好
性能开销中（多 1~2 次纹理采样）低（1 次 LUT 采样）低（1 次乘法）
适用场景最主流，卡通/Q版角色像素风格、复古游戏纯色服装、简单场景
📌 推荐方案：绝大多数 2D UGC 游戏采用 方案🅰（Mask+HSV），兼顾效果、性能和扩展性。本文档以此方案为重点展开。

② RGBA 通道分配流程约定（核心）

🚨 流程约定：同一像素在 R/G/B 三个通道中只能有一个通道有值（通道互斥），否则会导致两个换色区域串色！
R 通道换色区域 1
如：头发
G 通道换色区域 2
如：上衣
B 通道换色区域 3
如：裤子/裙子
A 通道总开关
白=可染色 黑=不染色

通道值含义

值含义用途
255（白色）该像素完全受对应换色区域影响区域内部
0（黑色）该像素不受对应换色区域影响区域外部
1~254（中间值）半透明过渡，产生渐变混合效果边缘柔化（1~2px 羽化）

遮罩图绘制示意

          原图                        Mask 图

    ┌──────────────┐          ┌──────────────┐
    │   ★ 头发     │          │  R=255       │  ← 头发区域：R通道涂白
    │  ┌────────┐  │          │  ┌────────┐  │
    │  │  脸部  │  │          │  │ A=0    │  │  ← 脸部：Alpha 涂黑（不染色）
    │  └────────┘  │          │  └────────┘  │
    │   上衣       │          │  G=255       │  ← 上衣区域：G通道涂白
    │   裤子       │          │  B=255       │  ← 裤子区域：B通道涂白
    │   鞋子       │          │  A=0         │  ← 鞋子：Alpha 涂黑（不染色）
    └──────────────┘          └──────────────┘

遮罩图制作硬指标

要素规范
尺寸必须与基础原图完全一致（像素级对齐）
格式PNG（RGBA 32bit）或 TGA — 绝不允许 JPEG
压缩无损格式是流程约定，有损压缩会导致通道值杂色噪点
边缘处理区域边缘建议留 1~2px 渐变过渡，避免硬边锯齿
通道互斥同一像素 R/G/B 仅允许一个通道有值
非染色区域A 通道涂黑 (0)，无论 RGB 值都不会变色

③ 多 Mask 无限扩展算法

当角色的换色区域超过 3 个时，单张 Mask 的 RGB 三通道不够用。此时需启用多 Mask 图方案。

⚡ 核心公式（程序/TA 必知）：
通道索引 channelIndex = 区域编号 % 3（0=R, 1=G, 2=B）
Mask图编号 maskIndex = Math.floor(区域编号 / 3) 
区域 0（头发）  → Mask1.png 的 R 通道
区域 1（上衣）  → Mask1.png 的 G 通道
区域 2（裤子）  → Mask1.png 的 B 通道
─────────── 第 1 张 Mask 用满 ───────────
区域 3（鞋子）  → Mask2.png 的 R 通道
区域 4（手套）  → Mask2.png 的 G 通道
区域 5（配饰）  → Mask2.png 的 B 通道
─────────── 第 2 张 Mask 用满 ───────────
区域 6（斗篷）  → Mask3.png 的 R 通道
...以此类推（无上限）

替代方案：灰度分段（单通道多区域）

如果不想增加 Mask 图数量，可以用灰度值范围分段：

灰度值范围          对应区域
─────────────────────────────
0                   不换色
1 ~ 50              区域 1（头发）
51 ~ 100            区域 2（上衣）
101 ~ 150           区域 3（裤子）
151 ~ 200           区域 4（鞋子）
201 ~ 250           区域 5（配饰）
251 ~ 255           区域 6（特殊）
⚠️ 注意：灰度分段方案无法使用羽化过渡，边缘会有硬切锯齿。高品质项目建议使用多 Mask 方案。

④ 性能红线与规格限制

指标红线值推荐值说明
Mask 图尺寸与原图一致（2N）512×512必须与 Base 图像素级对齐
单角色 Mask 张数≤ 3 张1 张每多一张增加一次纹理采样
单 Mask 换色区域3 个（RGB三通道）—通道互斥流程约定
Base 图纯黑/白像素禁止 (0,0,0) 和 (255,255,255)HSV 饱和度 40~70%纯黑白 HSV 偏移无效
Mask 通道溢出0%（同像素多通道有值率）—溢出导致串色
调色板贴图尺寸256×1 固定—Palette Swap 方案专用
调色板采样模式Point（最近邻）—禁止线性插值
🔴 硬性红线：Base 图换色区域的任何像素，亮度 L < 3（纯黑）或 L > 252（纯白）的，HSV 色相偏移将完全失效。必须改用深灰 (10,10,10) / 浅灰 (245,245,245) 替代。

⑤ 灰度图明暗阈值限制

灰度叠色方案（方案🅲）中，灰度图的明暗值直接决定换色效果：

灰度范围效果建议
0 ~ 50叠色后几乎全黑❌ 避免使用
50 ~ 128叠色后偏暗，适合阴影区⚠️ 仅用于暗部
128 ~ 200叠色效果最自然✅ 推荐范围
200 ~ 255叠色后接近纯色，缺少层次⚠️ 仅用于高光
⚠️ 乘法特性：最终颜色 = 灰度值 × 目标颜色。灰度叠色只能让颜色变暗，想要提亮需要程序使用 HDR 值（>1.0）。

⑦ 图文红黑榜 Do's & Don'ts

❌ Don't — Mask 图用 JPEG 保存

[ JPEG 压缩伪影 → 通道值出现噪点杂色 ]

有损压缩会破坏通道精度，导致换色区域出现随机杂色噪点。

✅ Do — Mask 图用 PNG/TGA 无损保存

[ 干净的通道值 → 换色边缘清晰无杂色 ]

无损格式确保每个像素的通道值精确，换色效果纯净。

❌ Don't — Base 图换色区用纯黑/纯白

[ HSV=0,0,0 → 色相偏移无效 → 换任何颜色都是黑 ]

纯黑 (0,0,0) 和纯白 (255,255,255) 在 HSV 空间中饱和度为 0，偏移无效。

✅ Do — 换色区用中等饱和度颜色

[ 饱和度 40%~70% → 偏移后自然过渡 ]

中等饱和度的颜色在 HSV 偏移后效果最自然，光影层次保留完好。

❌ Don't — 同一像素 R+G 都有值

[ R=200, G=180 → 头发和上衣串色 ]

违反通道互斥流程约定，导致一个像素同时被两个区域染色，产生脏色。

✅ Do — 每像素严格只属于一个通道

[ R=255, G=0, B=0 → 该像素只属于头发区 ]

通道互斥：一个像素的 R/G/B 中只有一个允许有非零值。

⑨ 常见问题排查 FAQ

❓ 换色后出现杂色/噪点？
原因：Mask 图使用了 JPEG 或有损压缩格式。
解法：必须使用 PNG/TGA 无损格式导出 Mask 图。
❓ 换色边缘有明显锯齿？
原因：Mask 图边缘是硬切的 0/255 二值。
解法：边缘做 1~2px 的灰度渐变过渡（羽化）。
❓ 黑色/白色区域换不了色？
原因：纯黑 (0,0,0) 在 HSV 空间中 S=0，色相偏移无效。
解法：换色区域避免纯黑纯白，改用深灰 (10,10,10) / 浅灰 (245,245,245)。
❓ 两个区域换色互相串色？
原因：Mask 中同一像素多个通道同时有值，违反互斥流程约定。
解法：确保 R/G/B 三通道严格互斥，使用工具批量检测。
❓ 换色后光影消失变平？
原因：原图绘制时换色区域缺少明暗层次。
解法：保证换色区域有足够的光影细节（2~3 级明暗过渡）。
❓ Mask 图和原图对不上？
原因：导出尺寸不一致或存在亚像素偏移。
解法：使用同一画布分层导出，确保 1:1 像素对齐。

11.3 自动导出脚本（Photoshop CEP/UXP）
// Photoshop UXP 自动导出脚本（伪代码）
async function exportCharacterAssets(doc) {
  const exportGroup = doc.layers.find(l => l.name === '[export]');
  
  // 1. 导出 Base 图
  const baseLayers = exportGroup.layers.filter(l => l.name.includes('[base]'));
  await exportMergedPNG(baseLayers, `${charID}_base.png`);
  
  // 2. 导出 Mask 图（逐通道合成）
  const maskLayers = exportGroup.layers.filter(l => l.name.includes('[mask'));
  for (const mask of maskLayers) {
    await exportAsSingleChannel(mask, `${charID}_${mask.name}.png`);
  }
  
  // 3. 自动校验通道互斥
  const violations = await checkChannelExclusion(maskLayers);
  if (violations.length > 0) {
    alert(`❌ 通道互斥校验失败：${violations.length} 处冲突`);
    return;
  }
  
  // 4. 生成 config.json
  await generateConfigJSON(charID, maskLayers);
  
  // 5. 生成预览图（应用3套配色）
  await generatePreviews(charID, COLOR_SCHEMES);
}
✅ 工具化收益：从"手动拼合导出 ~30min/角色"降低到"一键导出 ~2min/角色"。且零人为错误（通道互斥自动校验、尺寸自动对齐）。
11.4 PSD 拆分常见错误

❌ Don't — 所有图层平铺在根目录

50+ 图层无分组、无命名规范。新人接手完全看不懂哪些是换色区域、哪些是固定区域。自动导出脚本无法识别。

✅ Do — 严格分组 + 前缀命名

按 [regions]/[fixed]/[export]/[reference] 分组。每个图层用前缀标记用途。任何人打开 PSD 都能 3 秒理解结构。

⑫ 引擎侧 Shader 实现方案（Unity/Cocos/自研）

📌 本章面向：TA（Technical Artist）、客户端程序。美术同学了解原理即可，重点关注"我出的图如何被 Shader 消费"。
12.1 Mask+HSV 方案 Shader 核心逻辑
// ═══ Mask+HSV 换色 Shader（HLSL/GLSL 通用伪代码）═══

// 输入
sampler2D _BaseTex;    // 基础原图
sampler2D _MaskTex;    // 遮罩图
float3 _HueShift;     // 3个区域的色相偏移值 (度数, -180~180)
float3 _SatScale;     // 3个区域的饱和度缩放 (0~2)
float3 _ValScale;     // 3个区域的明度缩放 (0~2)

float4 frag(v2f i) : SV_Target {
    float4 base = tex2D(_BaseTex, i.uv);
    float4 mask = tex2D(_MaskTex, i.uv);
    
    // 将 base RGB 转换到 HSV 空间
    float3 hsv = RGBtoHSV(base.rgb);
    
    // 按 Mask 通道权重应用偏移
    float totalWeight = mask.r + mask.g + mask.b;
    if (totalWeight > 0.01 && mask.a > 0.01) {
        // R 通道区域偏移
        hsv.x += _HueShift.x * mask.r;
        hsv.y *= lerp(1.0, _SatScale.x, mask.r);
        hsv.z *= lerp(1.0, _ValScale.x, mask.r);
        
        // G 通道区域偏移
        hsv.x += _HueShift.y * mask.g;
        hsv.y *= lerp(1.0, _SatScale.y, mask.g);
        hsv.z *= lerp(1.0, _ValScale.y, mask.g);
        
        // B 通道区域偏移
        hsv.x += _HueShift.z * mask.b;
        hsv.y *= lerp(1.0, _SatScale.z, mask.b);
        hsv.z *= lerp(1.0, _ValScale.z, mask.b);
    }
    
    // HSV 转回 RGB
    float3 result = HSVtoRGB(hsv);
    return float4(result, base.a);
}

12.3 Shader 变体与性能优化
优化策略做法收益
多 Mask 编译开关#pragma shader_feature _MASK_COUNT_1 _MASK_COUNT_2 _MASK_COUNT_3单 Mask 角色不采样多余贴图，节省 1~2 次纹理采样
Alpha 通道早退if (mask.a < 0.01) return base;非换色区域跳过 HSV 计算（ALU 节省 30%+）
预计算 LUTCPU 侧将 HSV 偏移预烘焙为 256×1 颜色 LUT运行时仅 1 次 LUT 采样代替 RGB↔HSV 转换，GPU ALU 降 70%
Batch-Friendly换色参数存入顶点色/实例化缓冲而非独立 Material保持合批不打断，DC 数不随角色数增长
⚠️ 常见坑：美术导出的 Mask 图被 Unity 默认识别为 sRGB（Gamma）空间 → 通道值被 Gamma 校正 → Shader 读到的值与原始不一致（128 变成 187）。必须把 Mask 贴图的 sRGB 设为 Off（Linear）。

13.3 自动校验脚本（Python 核心逻辑）
import numpy as np
from PIL import Image

def check_channel_exclusion(mask_path: str, tolerance: int = 5) -> list:
    """检查 Mask 图通道互斥性"""
    img = np.array(Image.open(mask_path))
    r, g, b = img[:,:,0], img[:,:,1], img[:,:,2]
    
    # 找到同时有多个通道超过容差的像素
    violations = []
    multi_channel = ((r > tolerance).astype(int) + 
                     (g > tolerance).astype(int) + 
                     (b > tolerance).astype(int)) > 1
    
    coords = np.argwhere(multi_channel)
    for y, x in coords[:100]:  # 最多报100个
        violations.append({
            'x': int(x), 'y': int(y),
            'R': int(r[y,x]), 'G': int(g[y,x]), 'B': int(b[y,x])
        })
    
    return violations

def check_brightness_safety(base_path: str, mask_path: str) -> list:
    """检查 Base 图换色区域的亮度安全"""
    base = np.array(Image.open(base_path).convert('RGB'))
    mask = np.array(Image.open(mask_path))
    
    # 换色区域 = Mask 中任一通道有值的像素
    active = (mask[:,:,0] > 5) | (mask[:,:,1] > 5) | (mask[:,:,2] > 5)
    
    # 计算亮度 (简化公式)
    luminance = 0.299 * base[:,:,0] + 0.587 * base[:,:,1] + 0.114 * base[:,:,2]
    
    # 找过暗/过亮的像素
    too_dark = active & (luminance < 3)
    too_bright = active & (luminance > 252)
    
    return {
        'dark_pixels': int(too_dark.sum()),
        'bright_pixels': int(too_bright.sum()),
        'total_active': int(active.sum())
    }

15.1 渐变换色（Gradient Color Swap）
技术方案实现原理效果性能开销
UV 渐变 Ramp根据像素在 Mask 区域内的 Y 坐标采样渐变色 Ramp 贴图头发从发根到发尾颜色渐变+1 次纹理采样

15.2 花纹贴花（Pattern Overlay）
方案原理适用场景美术提供
贴花纹理叠加在换色后的结果上叠加 Pattern 贴图（Multiply/Overlay 混合）服装花纹、迷彩、条纹额外的 Pattern 贴图 + Pattern UV 映射参数
通道内子区域用 Mask 灰度值的不同范围分出子区域，各子区域不同颜色同一件衣服上下两段不同色精确的灰度分段 Mask
