① 拓扑布线标准与面数预算

面数预算表

部件LOD0（近景/展示）LOD1（中景/战斗）LOD2（远景/大厅）
头部+面部≤ 8,000 tri≤ 4,000 tri≤ 1,500 tri
头发≤ 10,000 tri≤ 5,000 tri≤ 2,000 tri
躯干+上装≤ 12,000 tri≤ 6,000 tri≤ 2,500 tri
下装+腿部≤ 6,000 tri≤ 3,000 tri≤ 1,200 tri
手部+手套≤ 4,000 tri≤ 2,000 tri≤ 800 tri
鞋子+脚部≤ 3,500 tri≤ 1,500 tri≤ 600 tri
配饰/武器≤ 6,000 tri≤ 3,000 tri≤ 1,000 tri
总计≤ 50,000 tri≤ 25,000 tri≤ 10,000 tri
🚨 硬红线：LOD0 角色整体三角面数不得超过 50,000 tri（含所有配饰），否则同屏 30 角色场景下 GPU 渲染时间将突破 16ms 帧预算。

拓扑布线核心要求

面部拓扑（最优先）

  • 眼眶必须形成完整 Eye Loop（6~8 边环），保证眨眼 Blend Shape 平滑过渡
  • 口部必须形成完整 Mouth Loop（8~10 边环），保证口型 Morph Target 正确变形
  • 鼻翼沿线需平滑过渡，避免锐角三角面导致法线突变
  • 面部整体布线需遵循 肌肉走向原则：额肌→眼轮匝肌→颧肌→口轮匝肌→颏肌

关节拓扑

  • 膝盖/肘部弯曲区域：至少 3 环布线，确保 120° 弯曲不塌陷
  • 肩部：需 4~5 环过渡布线，覆盖抬臂/前伸/后展全方位变形
  • 指关节：每个关节至少 2 环，手掌掌心侧加密布线
  • 腰腹部：扭转区域需增加横向环线密度，支持躯干旋转

全局规则

规则标准说明
四边面率≥ 98%三角面仅允许出现在指尖、脚趾等极端末端
N-gon0 容忍五边面及以上完全禁止
法线方向统一朝外不允许翻面，背面剔除需保证可见面法线正确
重叠顶点0 容忍合并阈值 0.001 单位内的重叠点
孤立顶点/边0 容忍所有几何元素必须属于有效面
对称性建议角色躯干/四肢推荐镜像建模后调节细节

② PBR 贴图通道规范

PBR 贴图套件

贴图类型通道分配色彩空间推荐尺寸压缩格式
BaseColor (Albedo)RGB = 固有色, A = 透明度（不透明物体留白）sRGB2048×2048BC7 / ASTC 4×4
Normal MapRG = 切线空间法线 XY（B 通道由引擎重建）Linear2048×2048BC5 / ASTC 5×5
ARM (合并贴图)R = AO, G = Roughness, B = MetallicLinear2048×2048BC7 / ASTC 4×4
EmissiveRGB = 自发光颜色（黑色 = 无发光）sRGB1024×1024BC7 / ASTC 6×6
SubsurfaceColorRGB = 次表面散射颜色（皮肤/蜡质材质专用）sRGB1024×1024BC7（可选）
📌 ARM 通道复用原则：将 AO/Roughness/Metallic 合并到一张贴图的 R/G/B 通道，可减少 2 次纹理采样。移动端强制使用 ARM 合并，PC 端推荐使用。

数值红线

属性合法范围常见错误后果
BaseColor 亮度30~240 (sRGB)纯黑 (0) 或纯白 (255)破坏 PBR 能量守恒，渲染失真
Metallic0 或 1（非 0 即 1）0.5 等中间值现实中不存在"半金属"，物理不正确
Roughness0.05~0.95全 0（纯镜面）或全 1（无高光）0 导致反射过强，1 导致材质"死板"
Normal 强度默认 1.0，微调 0.5~1.5超过 2.0法线过强导致棱角突出、闪烁伪影
AO 范围0.2~1.0AO 全黑 (0)缝隙处过黑，与全局 AO 叠加产生"死黑"
🚨 铁律：禁止使用 Photoshop 「亮度/对比度」直接调整 ARM 贴图 — 这会破坏三通道独立的物理含义。每个通道必须单独调节。

Normal Map 格式约定

引擎法线格式G 通道方向说明
UE5DirectXG 朝下（-Y）UE 内置 Flip Green Channel 选项可辅助修正
UnityOpenGLG 朝上（+Y）默认 OpenGL，导入时确保不勾选翻转
自研引擎OpenGLG 朝上（+Y）如使用 DirectX 需在 Shader 内翻转

贴图命名规则

T_{角色代号}_{部件}_{贴图类型}.{ext}

示例：

T_Hero01_Body_Normal.tga         # 躯干法线
T_Hero01_Body_ARM.tga            # 躯干 AO+Roughness+Metallic
T_Hero01_Head_Emissive.tga       # 头部自发光
T_Hero01_Hair_BaseColor.tga      # 头发固有色

③ UV 展开与 Texel Density 规范

Texel Density (TD) 标准

Texel Density 表示每游戏单位（1m）上有多少像素覆盖，是保证不同部件纹理清晰度一致的核心指标。

区域TD 系数（相对基准）说明
面部2.0×最高优先级，玩家视线焦点区域
手部1.5×近景动画/武器持握经常特写
躯干/腿部1.0×（基准）TD 基准线
鞋底/内部0.5×不可见或极少可见区域可降低
⚠️ TD 一致性检查：同一角色身上不同部件的 TD 差异不得超过 3×（如面部 2.0× 与鞋底 0.5× 的差距 4× 已接近上限），否则视觉上会出现明显的清晰度断层。建议使用 Checker Map 全身检查。

UV 布局原则

原则说明检查方式
无翻转所有 UV 岛 winding 方向一致（逆时针），禁止翻转导致法线异常UV 编辑器中翻转岛显示为红色
接缝 = 硬边UV 接缝位置必须设置对应的硬边法线（Split Normals），否则法线贴图无法正确衔接渲染检查接缝处是否有明暗条纹
2px 间距UV 岛之间至少留 2px 边距（基于 2048 贴图），防止 Mipmap 采样时颜色溢出导出 UV 快照叠加 2px 网格检查
拉直四肢手臂/腿部 UV 沿肢体长轴方向拉直，保证条纹/织物纹理方向正确贴 Checker Map 查看条纹走向
对称利用对称部件（左右手套/鞋子）可 UV 重叠以节省空间，但需确认材质无不对称细节需求UV 编辑器中查看重叠率

UV Set 规范

  • UV0（必须）：主纹理映射，承载所有 PBR 贴图采样
  • UV1（推荐）：Lightmap UV，用于烘焙光照贴图（引擎自动生成或手动展开），不允许重叠
  • UV2（可选）：特殊效果层（溶解 Mask、流动 UV 等），按需创建

④ LOD 分级策略

LOD 切换距离与面数递减

LOD 级别屏幕占比切换距离（参考）面数比例贴图尺寸骨骼处理
LOD0> 30%0~5m100%（≤50K tri）2048²完整骨架
LOD115~30%5~15m50%（≤25K tri）1024²合并手指为拳头
LOD25~15%15~40m20%（≤10K tri）512²合并手臂/腿为单骨
LOD3（可选）< 5%> 40m8%（≤4K tri）256²仅保留脊柱+四肢根骨
📌 UE5 Nanite 适配：如项目使用 UE5 Nanite，静态物件（武器/配饰）可免除 LOD 制作。但 Skeletal Mesh 目前不支持 Nanite（至 UE 5.4），角色主体仍需手动 LOD。

LOD 生成规则

  • 优先减面区域：背部、腿部内侧、被服装遮挡的身体部位
  • 保护区域：面部轮廓线（Silhouette）、手指（LOD0~1 需保留）、关节弯曲区域
  • 递减方式：推荐 DCC 工具半手动减面（Maya Reduce / ZBrush Decimation Master），LOD2+ 可使用引擎自动 LOD
  • 切换过渡：启用 Dithered LOD Transition（屏幕空间抖动过渡），避免跳变

Mipmap 策略

贴图类型Mipmap 生成滤波方式注意事项
BaseColor自动生成Anisotropic 8×Alpha 通道需用 Premultiplied 避免边缘黑线
Normal Map自动生成Trilinear远距离 Mip 会导致法线减弱是正常现象
ARM自动生成Bilinear各通道物理含义不同，不可对 ARM 做模糊后处理
Emissive自动生成Bilinear小面积发光需确认远距离 Mip 不会丢失亮点

✅ 命名原则：前缀 L_ / R_ 区分左右，全部使用 PascalCase，与 UE Mannequin / Mixamo 兼容。自定义骨骼以 C_（Custom）前缀标识。

蒙皮权重规范

规则标准检查方法
最大影响骨骼数移动端 ≤ 4，PC ≤ 8DCC 工具中查看 Max Influences
权重归一化所有权重之和 = 1.0，误差 ≤ 0.001导出前执行 Normalize Weights
幽灵权重清除权重 < 0.01 必须清零DCC 工具 Prune Small Weights
对称蒙皮左右肢体蒙皮需镜像一致使用 Mirror Weights 后 Diff 检查
穿模检查全 ROM 动作下无穿模播放 T-Pose / A-Pose / 极端 Pose 测试集
🚫 蒙皮红线：禁止出现某顶点未绑定任何骨骼（"Free Vertex"）的情况 — 这会导致该顶点在原点产生尖刺。

IK 链设置标准

  • Foot IK：L_Foot / R_Foot 为末端，L_LowerLeg / R_LowerLeg 为中间节点，Pole Vector 朝前
  • Hand IK：L_Hand / R_Hand 为末端，L_LowerArm / R_LowerArm 为中间节点，Pole Vector 朝后
  • Spine IK：Hips → Spine → Spine1 → Spine2 → Neck 组成 5 节点 Spline IK
  • 所有 IK 链需在 DCC 中设置默认 Preferred Angle，避免引擎中翻转抖动

⑦ Do's & Don'ts 红黑榜

拓扑 & 建模

✅ Do's

  • 面部使用完整环线布线（Eye Loop / Mouth Loop）
  • 关节处保持足够环线密度（膝≥3环，肩≥4环）
  • 四边面率保持 ≥ 98%
  • 使用 Checker Map 验证布线密度均匀性
  • 对称建模后再做非对称细节调整

❌ Don'ts

  • 出现 N-gon（五边面及以上）
  • 关节弯曲处仅用 1~2 环（弯曲时必定塌陷）
  • 法线翻面（导致引擎中出现黑面）
  • 遗留重叠顶点和孤立边
  • 不检查就提交模型（至少跑一遍 MeshLint）

PBR 贴图

✅ Do's

  • BaseColor 亮度控制在 30~240 sRGB 范围
  • Metallic 只用 0 或 1，金属/非金属二值化
  • ARM 三通道独立调节，不做整体亮度调整
  • Normal Map 格式与目标引擎匹配（UE=DirectX / Unity=OpenGL）
  • 使用 Substance Painter 标准 PBR 验证球检查材质物理正确性

❌ Don'ts

  • BaseColor 中烘入光影信息（PBR 下由引擎计算光照）
  • Metallic 用中间值（如 0.5）— 物理不正确
  • Normal Map 存为 sRGB（必须是 Linear）
  • 对 ARM 贴图做模糊/锐化后处理
  • Emissive 贴图亮度值超过 10.0（Bloom 会严重过曝）

UV & 蒙皮 & 导出

✅ Do's

  • UV 接缝处设置硬边法线
  • UV 岛间距 ≥ 2px（基于 2048 贴图）
  • 导出前清除历史/冻结变换/删除辅助节点
  • 蒙皮后跑全 ROM 动画测试集检查穿模
  • FBX 版本统一为 2020.2

❌ Don'ts

  • UV 岛有翻转（winding 不一致）
  • Lightmap UV (UV1) 存在重叠
  • 蒙皮超过平台限制的骨骼影响数
  • 存在未绑定骨骼的 Free Vertex
  • Mesh 和动画打包在同一个 FBX 中导出

❓ 常见问题 FAQ

LOD0 面数已经到 48K，配饰再加就超标了怎么办？
方案一：将配饰做成独立 Actor/Component，使用自己的 LOD 策略，不计入角色主体面数。方案二：在 LOD0 中将配饰面数控制在 2K 以内，把"精细配饰"只在角色展示界面（非战斗）使用。
ARM 贴图中 Metallic 的边缘如何处理？
金属和非金属交界处应保持 硬边过渡（1~2 像素宽度），不要做渐变混合。在现实物理中，一个表面要么是金属要么不是，没有"渐变金属"的概念。边缘 1px 的过渡是为了避免锯齿。
Blender 导出的 FBX 在 UE5 中缩放不对？
Blender 默认单位是米，UE5 默认单位是厘米。导出时在 FBX Export 设置中将 Scale 设为 100，或在 Blender Scene Properties 中将单位改为 Centimeters 再导出。
Normal Map 导入后出现接缝黑线？
99% 的原因是 UV 接缝处没有设置硬边法线（Split Normals）。解决方案：在 DCC 中选择所有 UV 接缝边 → 设置为 Hard Edge → 重新导出。剩下 1% 可能是 Tangent Space 计算方式不匹配（MikkTSpace vs 其他）。
移动端真的只能 4 骨骼影响吗？
取决于 GPU。中低端 Android 设备的 Vertex Shader 寄存器有限，4 bone influences 是安全上限。UE5 移动端默认就是 4，Unity 也推荐 4。高端移动设备（如 A17 Pro / Snapdragon 8 Gen3）支持 8，但为了覆盖率建议统一 4。
什么情况下可以不做 LOD？
仅当：① 角色只在固定距离出现（如 UI 展示界面）；② 同屏最多 1~2 个角色且有充足性能余量；③ 使用 UE5 Nanite 且为 Static Mesh（注意 Skeletal Mesh 不支持）。其他所有情况都必须制作至少 LOD0 + LOD1。
