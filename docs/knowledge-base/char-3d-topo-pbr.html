① 拓扑布线标准与面数预算

🚧 待补充：本节将详细规定 3D 角色的拓扑布线标准，包括面部环线（Eye Loop / Mouth Loop）、关节拓扑（膝盖/肘部需 3 环以上）、四边面率等。

拓扑布线要求（待展开）

  • 面部：眼眶/口部需完整环线，支持 Blend Shape / Morph Target 表情驱动
  • 关节：膝盖/肘部/肩部至少 3 环布线，确保弯曲时不塌陷
  • 四边面率 ≥ 98%，三角面仅允许出现在极端末端（指尖/脚趾）
  • 禁止 N-gon（五边面及以上）
  • 顶点法线统一朝外，不允许翻面

② PBR 贴图通道规范

🚧 待补充：本节将详细规定每张 PBR 贴图的通道分配、格式和压缩策略。

PBR 贴图套件（框架已定）

贴图类型通道色彩空间推荐尺寸压缩格式
BaseColor (Albedo)RGB = 固有色, A = 透明度sRGB2048×2048BC7 / ASTC 4×4
Normal MapRG = 切线空间法线 XYLinear2048×2048BC5 / ASTC 5×5
ARM (合并贴图)R=AO, G=Roughness, B=MetallicLinear2048×2048BC7 / ASTC 4×4
EmissiveRGB = 自发光颜色sRGB1024×1024BC7 / ASTC 6×6
SubsurfaceColorRGB = 次表面散射颜色sRGB1024×1024BC7（可选）
📌 ARM 通道复用：将 AO/Roughness/Metallic 合并到一张贴图的 R/G/B 通道中，可减少 2 次纹理采样，是移动端的标准做法。

数值规范（待展开）

  • BaseColor 亮度范围：30~240（避免纯黑纯白破坏 PBR 能量守恒）
  • Metallic：非金属=0, 金属=1，避免中间值
  • Roughness：推荐 0.3~0.9 范围，全 0 镜面反射过强，全 1 无高光
  • Normal Map 强度：使用 OpenGL 格式（G 通道朝上），UE5 使用 DirectX 格式需翻转 G

蒙皮权重规范（待展开）

  • 每顶点最大影响骨骼数：移动端 ≤ 4，PC ≤ 8
  • 权重归一化：所有权重之和 = 1.0，误差 ≤ 0.001
  • 避免"幽灵权重"：权重 < 0.01 的需清除
  • 对称绑定：左右肢体蒙皮需镜像一致
