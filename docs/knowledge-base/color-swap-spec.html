🌓 3. 灰度图明暗阈值限制

灰度叠色公式：最终颜色 = 灰度值 × 目标颜色。灰度值直接决定换色效果。

⛔ 死区禁令：灰度 <15（死黑）或 >245（死白）为死区——换色后颜色不变。
灰度区间名称效果合规
0 ~ 14死黑恒黑，目标色不可见❌ 禁止
15 ~ 60深暗部偏暗，适合褶皱阴影⚠️ 谨慎
60 ~ 200安全区效果自然✅ 推荐
200 ~ 245高光区偏亮，限量使用⚠️ 少量
246 ~ 255死白丢失目标色❌ 禁止
3.1 Base 原图颜色限制（HSV 方案）

① 可换色区域像素 R,G,B 均不为 0 且均不为 255（纯黑纯白 HSV 偏移无效）

② 饱和度 ≥ 15%（灰色系偏移极差）

③ 明度在 20%~90%

颜色HSV效果合规

#808080 (纯灰)S=0%色相偏移无效❌

✅❌ 5. 图文红黑榜

5.1 Mask 通道互斥 → 换色串色

❌ Don't — 同一像素多通道有值

[此处插入图片：Mask 图中上衣和头发交界处 R=180,G=120 同时有值，引擎换色后产生脏混色]

后果：Shader 两次色相偏移叠加，产生脏混色——不可控的中间色。红发+蓝衣交界出现紫色污渍。

✅ Do — 通道严格互斥

[此处插入图片：R 通道 255→0 渐变，G 通道 0→255 反向渐变，R+G ≤ 255，换色后交界自然过渡]

方法：逐通道单独编辑，或使用自动 Mask 工具。边缘过渡在同一通道内渐变。验证 R+G+B ≤ 255。

5.2 灰度死黑 → 换色变黑洞

❌ Don't — 灰度图含大面积死黑（V<15）

[此处插入图片：灰度底图褶皱处纯黑(0,0,0)，叠色后无论选什么颜色都是黑色"黑洞"]

后果：0 × 颜色 = 0，褶皱永远黑色，角色像"阴间角色"。

✅ Do — 灰度 60~200 安全区间

[此处插入图片：灰度图最暗处 65、最亮处 195，叠色后各颜色自然显示明暗]

方法：中值 120~160。用 Photoshop 色阶工具将 Output 限制为 60~200。描边可保留 15~30。

❓ 7. 常见问题排查 FAQ

💬 Q1：换色后颜色偏暗 / 发灰？

原因 90%：Mask 图导入引擎时 sRGB 未关闭。Gamma 2.2 校正将通道值压缩到更低的范围，等效于给整张 Mask 叠了一层暗灰色。

修复：UE5 中选中 Mask → Details → sRGB = false；Unity 中选中 Mask → Inspector → sRGB (Color Texture) = OFF。

💬 Q2：Mask 边缘出现杂色噪点？

原因①：Mask 被保存为 JPEG（即使后缀改为 .png 也可能是 JPEG 编码）。用十六进制编辑器检查文件头：PNG 应为 89 50 4E 47，JPEG 为 FF D8 FF。

原因②：在 Photoshop 中使用了柔边橡皮擦/笔刷跨通道操作。

修复：重新从 PSD 中逐通道导出 Mask，确保无损 PNG 格式。

💬 Q3：Base 原图中皮肤/金属扣被意外染色了？

原因：Mask 图的 A（Alpha）通道未正确标记不染色区域。脸部/皮肤/金属扣件等不应参与换色的区域，Alpha 通道必须涂黑(0)。

修复：在 Photoshop 通道面板选中 Alpha 通道，用硬边笔刷将非换色区域涂为纯黑(0)，然后重新导出。
