📌 使用方法：① 将 UE4/UE5 Widget Blueprint 的层级树结构粘贴到左侧输入框（支持缩进/├/│ 等符号）；② 🆕 新功能：也可上传层级面板的截图，工具将自动识别并转换为文本格式。系统会自动解析并检查 18 项合规规则（命名规范、层级深度、节点数、锚点设置、动静分离、Visibility 等）。右侧可查看解析后的树状预览。

📝 输入 UMG 层级结构

每行一个节点，用缩进（空格/Tab/├─│）表示层级关系。
格式：节点名 (类型) 或 节点名 [类型]，例如：Btn_Close (Button)

🤖 AI 辅助识别（推荐）

📖 人工对照指南

显示截图并提供层级结构快速编写模板，对照截图手动输入。适合截图较复杂或 OCR 不准确的情况。

使用 Tesseract.js 在浏览器中本地识别文字（需联网加载模型，首次约 10s）。适合英文命名为主的层级树。

📖 快速编写模板 — 对照截图填写以下结构

📏 检查规则说明：以下是自动化检查工具覆盖的全部 18 项检查规则。每条规则标注了严重级别和检查逻辑，便于理解检查结果。

🏗️ 层级结构规则（4 条）

编号规则名称级别检查逻辑红线值
H-01最大嵌套深度错误遍历树统计最大深度深度 > 10 层报错；> 6 层报警告
H-02节点总数错误统计所有节点总数 > 300 报错；> 150 报警告
H-03空容器检测警告容器类节点无子节点检测到空容器报警告
H-04冗余嵌套警告容器仅包含一个同类型容器子节点检测到冗余嵌套报警告

📛 命名规范规则（6 条）

编号规则名称级别检查逻辑期望前缀
N-01Button 命名错误类型为 Button 的节点名称以 Btn_ 开头Btn_
N-02Text 命名错误类型为 Text Block 的节点以 Txt_ 开头Txt_
N-03Image 命名错误类型为 Image 的节点以 Img_ 开头Img_
N-04容器命名警告HBox→HB_ / VBox→VB_ / Overlay→OL_ / Canvas→CVS_ 等见命名规范表
N-05默认名残留错误匹配 Image_数字 / Button_数字 / TextBlock_数字 等模式—
N-06WBP 前缀警告根节点（User Widget）名称以 WBP_ 开头WBP_

⚡ 性能与结构规则（5 条）

编号规则名称级别检查逻辑
P-01ScrollBox 子节点过多错误ScrollBox / Scroll Box 的直接子节点 > 20 个
P-02合批打断风险警告Image-Text-Image 交替排列（同层级内）
P-03单容器子节点过多警告单个容器下直接子节点 > 30 个
P-04多层 Spacer 嵌套警告同一容器下连续出现 ≥ 3 个 Spacer
P-05Invalidation Box 内含高频元素错误Invalidation Box 下包含 ProgressBar / Timer 相关节点

🎯 最佳实践规则（3 条）

编号规则名称级别检查逻辑
B-01缺少背景遮罩建议弹窗类蓝图（名称含 Popup/Dialog/Modal）没有 BgMask/Mask 相关 Image
B-02缺少关闭按钮建议弹窗类蓝图没有 Btn_Close / Btn_Back 相关按钮
B-03SafeZone 缺失建议全屏界面（名称含 HUD/Screen/Main）未包含 SafeZone 节点
