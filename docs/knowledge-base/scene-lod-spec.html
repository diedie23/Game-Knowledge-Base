1️⃣ 模块化设计总则

💡 核心理念："一套模块 + 组合规则 = 无限场景"。标准化尺寸网格、连接点与材质规格，让关卡设计师像搭积木般拼装。

Deco 装饰D自由花瓶、路灯
Prop 道具PR自由箱子、桌椅

1.2 模块化"三一律"

📐 Three Unities：
① 统一 Grid：结构件对齐 100cm 主格 + 50cm 细格
② 统一 Pivot：结构件 Pivot 在左下角底部；Deco/Prop 在几何中心底面
③ 统一 Texel Density：同场景 TD 差异 ≤ ±15%

3️⃣ LOD 层级与面数预算

⛔ 红线：单模块 LOD0 ≤ 50,000 三角面。LOD 级数至少 3 级，建议 4 级。

3.1 LOD 标准

LOD屏占比面数比距离用途
LOD0>30%100%0~15m全精度

LOD3<3%≤10%80m+Impostor

6️⃣ 碰撞体与 NavMesh

碰撞类型面数性能适用
Box12⭐⭐⭐⭐⭐方形墙壁/地板
Convex Hull≤256⭐⭐⭐⭐不规则凸形

  UCX_Wall_Medieval_01_00   ← Convex #0
  UBX_Wall_Medieval_01_01   ← Box #1
  USP_Wall_Medieval_01_02   ← Sphere #2
⚠️ NavMesh 规则：地板斜面坡度 ≤ 45°，台阶高度 ≤ 40cm，相邻地板高差 ≤ Agent Step Height。

7️⃣ Do/Don't 红黑榜

✅ Do — 正确做法

[此处插入图片：模块对齐、UV 连续、LOD 完整的正确示例]

• 所有结构件对齐 100cm Grid
• Pivot 统一在左下角底部
• LOD0~LOD3 完整四级链
• 使用 Trim Sheet 减少材质数
• 碰撞体简化为 Box / Convex
• 接缝处顶点焊接 + UV 连续

❌ Don't — 错误做法

[此处插入图片：模块错位、Z-fighting、LOD 缺失的反面示例]

• Pivot 位置随意 → 对齐困难
• 不同 Grid 混用 → 缝隙/穿模
• 仅 LOD0 无低模 → GPU 过载
• 每个模块独立材质 → Draw Call 爆炸
• Complex Collision → 物理卡顿
• 硬切 LOD → 视觉 Pop 闪烁

✅ Do — Texel Density 一致

[此处插入图片：TD Checker 全场景绿色=一致]

• 所有模块 TD 在 10.24 px/cm ±15% 以内
• Checker 工具验证：全场景无红色区域

❌ Don't — TD 差异过大

[此处插入图片：TD Checker 花花绿绿=不一致]

• 地板 20 px/cm，墙壁 5 px/cm → 观感撕裂
• 玩家能直观感受到清晰度落差

9️⃣ 常见问题 FAQ

❓ 启用 Nanite 后还需要手动 LOD 吗？
Nanite 自动处理静态网格 LOD，但 仅限静态、不透明物体。半透明/骨骼/可破坏物体仍需手动 LOD。建议混合策略：静态用 Nanite，动态保留手动 LOD 链。
❓ Trim Sheet 和 Texture Atlas 有什么区别？
Trim Sheet：长条形 UV 区域，适合线性重复元素（墙边、管线），UV 可沿 U 轴 Tiling。
Atlas：矩形格子分区，每个模块占一格，UV 不可 Tiling。Trim Sheet 灵活度更高，但 UV 拆分要求更精细。
❓ 碰撞体面数的 1% 规则如何验证？
UE5：Show → Collision 可视化碰撞体；Statistics 面板查看碰撞面数。
Unity：Physics Debugger 显示碰撞网格。自动化脚本可在 CI 中检查 collision tri / render tri 比值。
