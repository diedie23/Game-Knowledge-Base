7️⃣ Do/Don't 红黑榜

✅ Do — 正确做法

[此处插入图片：模块对齐、UV 连续、LOD 完整的正确示例]

• 所有结构件对齐 100cm Grid
• Pivot 统一在左下角底部
• LOD0~LOD3 完整四级链
• 使用 Trim Sheet 减少材质数
• 碰撞体简化为 Box / Convex
• 接缝处顶点焊接 + UV 连续

❌ Don't — 错误做法

[此处插入图片：模块错位、Z-fighting、LOD 缺失的反面示例]

• Pivot 位置随意 → 对齐困难
• 不同 Grid 混用 → 缝隙/穿模
• 仅 LOD0 无低模 → GPU 过载
• 每个模块独立材质 → Draw Call 爆炸
• Complex Collision → 物理卡顿
• 硬切 LOD → 视觉 Pop 闪烁

✅ Do — Texel Density 一致

[此处插入图片：TD Checker 全场景绿色=一致]

• 所有模块 TD 在 10.24 px/cm ±15% 以内
• Checker 工具验证：全场景无红色区域

❌ Don't — TD 差异过大

[此处插入图片：TD Checker 花花绿绿=不一致]

• 地板 20 px/cm，墙壁 5 px/cm → 观感撕裂
• 玩家能直观感受到清晰度落差

9️⃣ 常见问题 FAQ

❓ 启用 Nanite 后还需要手动 LOD 吗？
Nanite 自动处理静态网格 LOD，但 仅限静态、不透明物体。半透明/骨骼/可破坏物体仍需手动 LOD。建议混合策略：静态用 Nanite，动态保留手动 LOD 链。
❓ Trim Sheet 和 Texture Atlas 有什么区别？
Trim Sheet：长条形 UV 区域，适合线性重复元素（墙边、管线），UV 可沿 U 轴 Tiling。
Atlas：矩形格子分区，每个模块占一格，UV 不可 Tiling。Trim Sheet 灵活度更高，但 UV 拆分要求更精细。
❓ 碰撞体面数的 1% 规则如何验证？
UE5：Show → Collision 可视化碰撞体；Statistics 面板查看碰撞面数。
Unity：Physics Debugger 显示碰撞网格。自动化脚本可在 CI 中检查 collision tri / render tri 比值。

🎯 制作人关注：场景模块化关键决策

💡 制作人经验："场景模块化做得好 = 关卡设计师可以无限组合出新地图而几乎不需要额外美术资源。但做得不好 = 每张图都是'定制品'，资源复用率 < 30%，场景团队永远人力紧缺。"

🚨 预研期必须锁定的约束

  • Grid 尺寸：一旦进入量产，主格尺寸不可更改（会导致所有已有模块无法对齐）
  • 同屏三角面总预算：确定目标硬件后倒推 → 单帧可渲染总面数 → 分配给场景/角色/特效各占比
  • Nanite vs 传统 LOD：二选一影响整个管线，不可量产中期切换
  • 光照方案：烘焙 vs 全动态 vs 混合 → 决定模块的 Lightmap UV 是否需要制作

🌐 10. HLOD 与 World Partition 规范

大世界场景中，传统手动 LOD 链无法满足数十平方公里的 Streaming 需求。HLOD (Hierarchical LOD) 和 World Partition 是 UE5 大世界核心技术栈，Unity 则通过 Addressables + DOTS Streaming 实现类似方案。

10.1 HLOD 层级策略

10.2 World Partition / Streaming 规范
配置项UE5 World PartitionUnity Addressables建议值
Grid Cell 尺寸WorldPartition → CellSize手动按场景 Chunk 划分128m × 128m（室外）/ 64m（室内）
加载距离DataLayer → LoadingRangeAddressables.LoadSceneAsync玩家可视距离 + 1 Cell 预加载缓冲
卸载距离加载距离 × 1.2（避免抖动）同理加载距离 + 20%

⛔ 红线：

• 单 Cell 内 Actor 数量 ≤ 500 个（含 Instanced Static Mesh）

• 单 Cell 加载耗时 ≤ 100ms（中端设备 SSD 基准）

• 加载/卸载距离差值 ≥ 20%，防止边界抖动反复加载

11.2 Python 校验脚本核心逻辑
# scene_asset_validator.py — 场景资产 CI 校验入口
import sys, json, re
from pathlib import Path

NAMING_REGEX = re.compile(r'^SM_(Floor|Wall|Pillar|Ceiling|Stair|Deco|Prop)_[A-Z]\w+_\d{2,3}$')
MAX_LOD0_TRIS = 50000
MIN_LOD_LEVELS = 3
MAX_COLLISION_RATIO = 0.01

def check_asset(asset_path: str, meta: dict) -> list[str]:
    errors = []
    
    # SC-005: 命名合规
    name = Path(asset_path).stem
    if not NAMING_REGEX.match(name):
        errors.append(f"[SC-005] 命名不合规: {name}")
    
    # SC-001: LOD 完整性
    lod_count = meta.get("lod_count", 0)
    if lod_count < MIN_LOD_LEVELS:
        errors.append(f"[SC-001] LOD 不足: {lod_count} 级 (需 ≥{MIN_LOD_LEVELS})")
    
    # SC-002: LOD0 面数
    lod0_tris = meta.get("lod0_tris", 0)
    if lod0_tris > MAX_LOD0_TRIS and not meta.get("hero_asset", False):
        errors.append(f"[SC-002] LOD0 面数超标: {lod0_tris} (上限 {MAX_LOD0_TRIS})")
    
    # SC-003: 碰撞面数比
    collision_tris = meta.get("collision_tris", 0)
    if lod0_tris > 0 and collision_tris / lod0_tris > MAX_COLLISION_RATIO:
        ratio = collision_tris / lod0_tris * 100
        errors.append(f"[SC-003] 碰撞面数比过高: {ratio:.1f}% (上限 1%)")
    
    # SC-008: Complex Collision
    if meta.get("has_complex_collision", False):
        errors.append(f"[SC-008] 禁止使用 Complex Collision")
    
    return errors

if __name__ == "__main__":
    changed = get_git_diff_files("SM_*.fbx")
    all_errors = []
    for f in changed:
        meta = load_asset_meta(f)
        all_errors += check_asset(f, meta)
    if all_errors:
        print("\n".join(all_errors))
        sys.exit(1)
11.3 UE5 Editor Utility 批量扫描工具
# UE5 Python — 场景资产批量合规扫描
import unreal

def scan_scene_assets():
    """扫描 /Game/Environment/ 下所有 StaticMesh 的 LOD、碰撞、Nanite 合规性"""
    registry = unreal.AssetRegistryHelpers.get_asset_registry()
    assets = registry.get_assets_by_path("/Game/Environment", recursive=True)
    
    report = {"errors": [], "warnings": [], "passed": 0}
    
    for asset_data in assets:
        asset = asset_data.get_asset()
        if not isinstance(asset, unreal.StaticMesh):
            continue
        
        name = asset.get_name()
        
        # LOD 检查
        lod_count = asset.get_num_lods()
        if lod_count < 3:
            report["errors"].append(f"[SC-001] {name}: LOD {lod_count} 级 (需≥3)")
        
        # LOD0 面数检查
        lod0_tris = asset.get_num_triangles(0)
        if lod0_tris > 50000:
            report["errors"].append(f"[SC-002] {name}: LOD0={lod0_tris} tri (上限 50K)")
        
        # Nanite 适用性检查
        if asset.nanite_settings.enabled:
            # 检查是否有透明材质
            for mat_idx in range(asset.get_num_sections(0)):
                mat = asset.get_material(mat_idx)
                if mat and mat.blend_mode != unreal.BlendMode.OPAQUE:
                    report["errors"].append(
                        f"[SC-011] {name}: Nanite 启用但含非 Opaque 材质")
        
        report["passed"] += 1
    
    unreal.log(f"扫描完成: {report['passed']} 通过, "
               f"{len(report['errors'])} Error, {len(report['warnings'])} Warning")
    return report

12.3 数据驱动决策模型

✅ 复用率 ≥ 70% 时

• 维持当前模块化策略
• 定期清理引用 ≤1 的闲置模块
• 考虑扩展 Kit 变体增加多样性
• 季度回顾频率可降为月度

❌ 复用率 < 50% 时

• 触发模块化专项治理 Sprint
• 分析定制资产产生原因（需求驱动/规范缺失？）
• 安排 TA 驻场辅导关卡设计师
• 增加 Kit Review 频率至每周
• PM 升级为风险项，纳入周报
