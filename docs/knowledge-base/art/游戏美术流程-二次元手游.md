# 二次元手游 — 美术制作流程总结

> 适用范围：二次元 / 日系 / 卡通渲染手游，如原神、崩坏系列、明日方舟、蓝色档案、胜利女神等类型，核心特征是角色驱动、卡通渲染（Cel-Shading / NPR）、Live2D / Spine 大量使用、卡牌立绘和皮肤是核心付费点。

---

## 二、APM 视角关注重点

| 关注维度 | 具体要点 |
|:---:|:---:|
| **角色生产节奏** | 二次元手游按卡池/版本节奏出角色，通常每 2~3 周 1 个新角色，APM 需建立角色生产流水线 |
| **立绘质量把控** | 立绘是玩家付费的核心驱动力，质量标准极高，需专人 Art Lead 逐张审核 |
| **Live2D/Spine 动态** | 动态立绘已成标配，需关注呼吸/眨眼/待机/触摸等交互动画的自然度 |
| **NPR 渲染一致性** | 3D 模型在不同光照/角度下需保持二次元观感，Face SDF / 描边粗细需逐角色调整 |
| **皮肤/时装管理** | 皮肤是核心收入，需管理好皮肤排期、品质分级（S/A/B 级）、返利策略的美术匹配 |
| **多语言适配** | 日/中/韩/英 多语种 UI 文本长度差异大，需预留弹性空间 |
| **版权与合规** | 二次元重视 IP 保护，角色设计需避免与已有 IP 过度相似，联动 IP 需版权方逐帧审核 |
| **社区美术运营** | 壁纸/头像/表情包等二创素材官方需持续供给，APM 需纳入排期 |

## 三、外包管理难点与常见风险

### 3.1 典型外包分工

二次元手游外包占比通常 40%~70%：

| 外包内容 | 典型方式 | 占比 | 备注 |
|:---:|:---:|:---:|:---:|
| 角色立绘 | 按角色/按皮肤 | 高 | 核心产出，质量要求最高 |
| Live2D 制作 | 按角色 | 中 | 需与立绘同步 |
| 3D 角色建模 | 按角色 | 中 | NPR 调整通常内部完成 |
| 场景原画/3D | 按场景 | 中 | 模块化程度看项目 |
| 特效 | 按技能/角色 | 低~中 | 风格统一难度大 |
| UI | 按界面 | 低 | 通常内部为主 |
| CG/PV | 按条 | 按需 | 关键 PV 外包给专业动画公司 |

### 3.2 常见风险

| 风险 | 表现 | 严重度 | 应对策略 |
|:---:|:---:|:---:|:---:|
| **画师风格差异** | 不同画师的二次元风格差异明显 | 🔴 高 | 指定画师 + Style Guide + 首件审核 + 配色监修 |
| **立绘返修率高** | 二次元用户对立绘要求极高 | 🔴 高 | 3 轮审核（草图→线稿→上色）+ 社区风评前测 |
| **Live2D 不自然** | 运动僵硬、物理穿模、表情诡异 | 🟡 中 | 标准动作参考 + 物理参数模板 + 反复调试 |
| **NPR 适配困难** | 外包模型在项目渲染管线下效果差 | 🔴 高 | NPR Checker Scene + 要求引擎内截图提交 |
| **排期与版本冲突** | 卡池上线日期固定，外包延期影响收入 | 🔴 高 | 每角色提前 6~8 周启动 + 备选画师 |
| **泄露风险** | 新角色/皮肤上线前被泄露 | 🔴 高 | 分阶段透露 + 水印 + NDA 严格执行 |
| **配色争议** | 外包配色与角色设定不符 | 🟡 中 | 精确 HEX 配色表 + 监修机制 |

## 四、各阶段交付物清单（Deliverables）

### 阶段 ① 世界观 & 角色企划

| 交付物 | 格式 | 说明 |

| NPR 技术方案 | 文档 + Demo | 渲染管线 + Shader 说明 |

### 阶段 ② 角色概念设计

| 交付物 | 格式 | 说明 |

### 阶段 ③ 2D 立绘制作

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|

### 阶段 ④ 3D 角色制作

## 五、验收标准建议

### 5.1 角色立绘验收

| 检查项 | 标准 | 优先级 |
|:---:|:---:|:---:|
| 分辨率 | ≥ 2048px（长边） | P0 |
| 图层规范 | PSD 图层分组清晰，线稿/底色/阴影/高光/特效分层 | P0 |
| 角色比例 | 与设定画一致，头身比 ±5% 误差 | P0 |
| 配色准确 | 与配色方案 HEX 值偏差 ΔE < 3 | P0 |
| 画风统一 | 眼睛/头发/皮肤上色手法与项目基准一致 | P0 |
| 细节完成度 | 饰品/纹样/材质质感符合设定 | P1 |
| 大破立绘 | 与常态立绘为同一角度，方便切换 | P1 |
| 文件大小 | PSD ≤ 500MB / PNG ≤ 20MB | P2 |

### 5.2 Live2D 验收

| 检查项 | 标准 |
|:---:|:---:|
| 分层数 | ≥ 30 层（标准品质） |
| 呼吸动画 | 自然起伏，周期 3~4 秒 |
| 眨眼 | 自然频率，含半闭/全闭 |
| 头部运动 | X/Y/Z 三轴旋转流畅 |
| 触摸反应 | 头/胸/手 3 个触摸区域有不同反应 |
| 物理模拟 | 头发/饰品/衣摆摆动自然，无穿模 |
| 嘴部口型 | 支持 3~5 个元音口型 |
| 文件大小 | 模型 + 贴图 ≤ 10MB |

### 5.3 3D NPR 角色验收

| 检查项 | 标准 |
|:---:|:---:|
| 面数 | ≤ 15K（手游）/ ≤ 30K（PC） |
| 描边 | 粗细均匀，转角无断裂 |
| 阴影分界 | 明暗交界线清晰锐利 |
| Face SDF | 面部阴影在旋转时形状自然 |
| 发色渐变 | 头发高光 / 暗部过渡符合二次元审美 |
| 多角度检查 | 360° 无穿模 / 无法线错误 |

### 5.4 特效验收

| 检查项 | 标准 |
|:---:|:---:|
| 风格匹配 | 日系二次元风格，非写实 |
| 节奏感 | 前摇→爆发→残留 节奏清晰 |
| 打击反馈 | 顿帧/缩放/震屏效果到位 |
| 帧动画 | 手绘帧 ≥ 8 张/秒 |
| 性能 | 单特效 Draw Call ≤ 10 |

## 七、二次元特色注意事项

### 7.1 角色设计的"商业性"

- 角色外观直接决定卡池收入，美术质量 = 商业价值
- 需提前与运营/市场部门对齐角色定位（人气向/实用向/情怀向）
- 立绘发布后社区反馈极快，需预留紧急修图的时间窗口

### 7.2 NPR 渲染的特殊性

- 不遵循物理正确的 PBR 规则，追求"好看"优先于"正确"
- 每个角色的描边粗细/阴影形状可能需要单独调整
- Face SDF 需要美术手动绘制，无法完全自动化

### 7.3 二创与社区运营

- 二次元游戏的社区生态依赖同人二创
- 官方需提供高清立绘/壁纸/透明底素材
- 角色生日/节日需要专属素材，APM 需纳入年度排期
