# 2D 角色换色 —— 美术出图资源提供规范

> 本文档面向 2D 美术，梳理"角色换色"场景下需要提供给程序的资源清单、制作规范与注意事项。
>
> **文档版本**：v1.0 | **更新日期**：2026-03-31 | **维护人**：美术组

---

## 一、换色方案概览与资源需求对照

不同的换色技术路线，对美术的出图要求差异很大。先了解方案再出图，可以避免返工。

| 方案 | 原理简述 | 美术需要提供的资源 | 适用场景 |
|------|----------|-------------------|----------|
| **① 调色板替换（Palette Swap）** | 用索引色/LUT 表做颜色映射 | 原图 + 调色板贴图（256×1） | 像素风格、复古游戏 |
| **② 遮罩分区 + HSV/Tint** | 用 Mask 图标记换色区域，Shader 做色相/叠色偏移 | 原图 + Mask 遮罩图 | **最主流方案**，卡通/Q版角色 |
| **③ 灰度底图 + 叠色** | 原图去色只保留明暗，程序乘以目标色 | 灰度底图（+ 可选 Mask） | 简单场景、纯色服装 |
| **④ 通道混合染色** | 利用 RGB 通道矩阵运算混合 | 原图 + Mask 遮罩图 | 高品质需求、美术预设配色 |
| **⑤ 多套贴图直出** | 每种颜色出一套图 | N 套完整贴图 | 颜色种类少、品质要求极高 |

> **推荐方案**：绝大多数 2D UGC 游戏采用 **方案②（遮罩分区 + HSV/Tint）**，兼顾效果、性能和扩展性。以下重点展开。

## 二、方案②详解：遮罩分区换色（重点）

### 2.1 美术需要提供的资源清单

```
📁 角色资源包
├── character_base.png        ← ① 基础原图（带完整色彩和细节）
├── character_mask.png        ← ② 换色遮罩图（RGBA 四通道）
├── palette_config.json       ← ③ 配色方案表（可选，美术预设的推荐配色）
└── preview_thumbnail.png     ← ④ 换色效果预览缩略图（可选）
```

### 2.2 资源①：基础原图（Base Texture）

| 要素 | 规范 |
|------|------|
| **格式** | PNG（带 Alpha 通道） |
| **尺寸** | 与正常角色出图一致（2^N，如 512×512） |
| **色彩要求** | 正常绘制完整色彩，包含光影、褶皱等细节 |
| **注意** | 换色区域的颜色**不要用纯黑(0,0,0)或纯白(255,255,255)**，否则 HSV 偏移无效 |
| **饱和度** | 换色区域建议使用**中等饱和度**的颜色（40%~70%），偏移后效果最自然 |

### 2.3 资源②：换色遮罩图（Mask Texture）—— 最关键

#### 通道定义规范

遮罩图的 **RGBA 四个通道** 各自承担不同职责：

```
┌─────────────┬──────────────────────────────────┐
│   通道       │           含义                    │
├─────────────┼──────────────────────────────────┤
│ R（红色）    │ 换色区域 1（如：头发）             │
│ G（绿色）    │ 换色区域 2（如：上衣）             │
│ B（蓝色）    │ 换色区域 3（如：裤子/裙子）        │
│ A（Alpha）   │ 总开关 —— 白=可染色，黑=不染色     │
├─────────────┼──────────────────────────────────┤
│ 如需更多区域  │ 可使用灰度值分段（见下方扩展方案）  │
└─────────────┴──────────────────────────────────┘
```

#### 通道值含义

| 值 | 含义 |
|----|------|
| **255（白色）** | 该像素**完全受**对应换色区域影响 |
| **0（黑色）** | 该像素**不受**对应换色区域影响 |
| **中间值（如 128）** | 半透明过渡，产生渐变混合效果（用于边缘柔化） |

#### 遮罩图绘制示意

```
          原图                      Mask 图

    ┌──────────────┐          ┌──────────────┐
    │   ★ 头发     │          │  R=255       │  ← 头发区域：R通道涂白
    │  ┌────────┐  │          │  ┌────────┐  │
    │  │  脸部  │  │          │  │ A=0    │  │  ← 脸部：Alpha 涂黑（不染色）
    │  └────────┘  │          │  └────────┘  │
    │   上衣       │          │  G=255       │  ← 上衣区域：G通道涂白
    │   裤子       │          │  B=255       │  ← 裤子区域：B通道涂白
    │   鞋子       │          │  A=0         │  ← 鞋子：Alpha 涂黑（不染色）
    └──────────────┘          └──────────────┘
```

#### 遮罩图制作要求

| 要素 | 规范 |
|------|------|
| **尺寸** | 必须与基础原图**完全一致** |
| **UV 对齐** | 必须与原图像素级对齐，不能有偏移 |
| **格式** | PNG（RGBA 32bit）或 TGA |
| **压缩** | **不要使用有损压缩**（如 JPEG），会导致通道值出现杂色噪点 |
| **边缘处理** | 区域边缘建议留 1~2px 的渐变过渡，避免硬边锯齿 |
| **区域互斥** | 同一像素在 R/G/B 三个通道中**只能有一个通道有值**（不能同时属于两个区域） |
| **非染色区域** | A 通道涂黑（0），无论 R/G/B 是什么值都不会变色 |

## 三、方案①详解：调色板替换（Palette Swap）

适用于**像素风格**游戏，原理是用一张小贴图做颜色映射。

### 3.1 美术需要提供的资源

```
📁 角色资源包
├── character_indexed.png     ← ① 索引色原图（颜色数量受限）
└── palettes/
    ├── palette_default.png   ← ② 默认调色板（256×1 像素）
    ├── palette_fire.png      ← ③ 火焰配色调色板
    └── palette_ice.png       ← ④ 冰霜配色调色板
```

### 3.2 索引色原图制作规范

| 要素 | 规范 |
|------|------|
| **颜色数量** | 控制在 **8~32 色** 以内（像素画的常规范围） |
| **颜色唯一性** | 需要被独立换色的区域，其颜色的 **R 通道值必须唯一** |
| **避免渐变** | 不要使用大面积的平滑渐变，每种颜色应为纯色色块 |
| **记录色表** | 必须导出一份**颜色对照表**，标明每个颜色对应的部位 |

### 3.3 调色板贴图制作规范

| 要素 | 规范 |
|------|------|
| **尺寸** | **256×1** 像素（一行，256 列） |
| **格式** | PNG（无压缩/无抗锯齿） |
| **映射规则** | 第 X 个像素的颜色 = 原图中 R 值为 X 的像素的替换色 |
| **透明像素** | 调色板中 Alpha=0 的位置表示"保持原色不变" |
| **采样模式** | 程序需设置为 **Point（最近邻）** 采样，禁止线性插值 |

### 3.4 调色板示意

```
原图中：头发 R=32, 上衣 R=115, 裤子 R=214

调色板贴图（256×1）：
位置:  [0] ... [32]    ... [115]   ... [214]   ... [255]
颜色:  透明    棕色(头发)  蓝色(上衣)  灰色(裤子)   透明

换成"火焰配色"时：
位置:  [0] ... [32]    ... [115]   ... [214]   ... [255]
颜色:  透明    红色(头发)  橙色(上衣)  暗红(裤子)   透明
```

## 四、方案③详解：灰度底图 + 叠色

最简单的方案，适合纯色面积大、细节需求低的场景。

### 4.1 美术需要提供的资源

```
📁 角色资源包
├── character_grayscale.png   ← ① 灰度图（只有明暗，无色彩）
├── character_mask.png        ← ② 分区 Mask（同方案②，可选）
└── character_detail.png      ← ③ 细节叠加层（可选，如花纹/图案）
```

### 4.2 灰度图制作规范

| 要素 | 规范 |
|------|------|
| **色彩** | 完全去色，只保留明暗关系（亮面→暗面的灰度过渡） |
| **中间灰** | 基础亮度建议在 **128~200** 之间（过暗叠色后太黑，过亮则没有暗部） |
| **细节** | 褶皱、阴影、高光全部用灰度表现 |
| **原理** | 程序做 `最终颜色 = 灰度值 × 目标颜色`，所以灰度=1(白)时显示纯目标色 |

### 4.3 注意事项

- 灰度叠色方案**只能让颜色变暗**（乘法特性），想要提亮需要程序使用 HDR 值（>1.0）
- 如果角色有多个需要独立换色的区域，仍然需要配合 **Mask 遮罩图**
- 适合做"纯色T恤换色"这类简单需求，复杂图案不适用

## 五、扩展：超过 3 个换色区域怎么办？

当 RGBA 四通道不够用时（需要 4 个以上独立换色区域），有以下扩展方案：

### 方案 A：灰度分段（单通道多区域）

用一个通道的**不同灰度值区间**表示不同区域：

```
灰度值范围          对应区域
─────────────────────────────
0                   不换色
1 ~ 50              区域 1（头发）
51 ~ 100            区域 2（上衣）
101 ~ 150           区域 3（裤子）
151 ~ 200           区域 4（鞋子）
201 ~ 250           区域 5（配饰）
251 ~ 255           区域 6（特殊）
```

**优点**：一张单通道图支持 6+ 个区域
**缺点**：不支持渐变过渡（边缘必须硬切），需要美术严格控制灰度值

### 方案 B：多张 Mask 图

```
mask_1.png → R=头发, G=上衣, B=裤子, A=开关
mask_2.png → R=鞋子, G=手套, B=配饰, A=开关
```

**优点**：每个区域仍可渐变过渡
**缺点**：多一张贴图，多一次采样，性能略有开销

### 方案 C：UV 多象限

将不同换色区域的 UV 放到不同象限（第1~第4象限），程序根据象限判断区域。

**优点**：不需要额外 Mask 图
**缺点**：需要美术重新规划 UV，只适合特定引擎

## 六、美术与程序对接检查清单

### 出图前确认

- [ ] 与程序确认采用哪种换色方案（Palette / Mask+HSV / 灰度叠色）
- [ ] 确认需要支持几个独立换色区域
- [ ] 确认遮罩图的通道分配定义（哪个通道对应哪个部件）
- [ ] 确认贴图尺寸和格式要求

### 出图时检查

- [ ] 基础原图换色区域无纯黑/纯白（HSV 方案）
- [ ] 遮罩图与原图尺寸完全一致、像素对齐
- [ ] 遮罩图使用**无损格式**导出（PNG/TGA），不要 JPEG
- [ ] 同一像素不同时归属多个换色区域（通道互斥）
- [ ] A 通道正确标记了不参与换色的区域（脸部/皮肤/金属扣等）
- [ ] 遮罩图边缘处理干净，无杂色溢出
- [ ] 调色板贴图（如适用）为 256×1 像素，Point 采样

### 交付时提供

- [ ] 基础原图 `.png`
- [ ] 遮罩图 `.png`（附通道定义说明）
- [ ] 配色方案表 `.json`（推荐配色组合）
- [ ] 换色区域说明文档（标注哪些部位可换色、哪些不可）
- [ ] 效果预览图（至少 3 套配色的预览）

## 七、常见踩坑与避免

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 换色后出现杂色/噪点 | Mask 图用了 JPEG 或有损压缩 | **必须使用 PNG/TGA 无损格式** |
| 换色边缘有明显锯齿 | Mask 图边缘是硬切的像素 | 边缘做 1~2px 的灰度渐变过渡 |
| 黑色/白色区域换不了色 | 纯黑 HSV 色相偏移无效 | 换色区域避免纯黑纯白，用深灰/浅灰替代 |
| 两个区域换色互相串色 | Mask 中同一像素多个通道都有值 | 确保 R/G/B 通道互斥 |
| 换色后光影消失/变平 | 原图绘制时阴影不够，或灰度图亮度过低 | 保证换色区域有足够的明暗层次 |
| 皮肤/脸部意外被染色 | A 通道没有正确涂黑 | 仔细检查 Alpha 通道的非染色区域标记 |
| 调色板换色颜色错乱 | 原图中两种颜色的 R 值相同 | 确保索引色的 R 通道唯一 |
| Mask 图和原图对不上 | 导出尺寸不一致或有偏移 | 使用同一画布分层导出，确保 1:1 对齐 |

## 八、最佳实践总结

```
                    美术出图工作流
                    
  ┌─────────────────────────────────────────┐
  │  1. 与程序确认换色方案和区域数量           │
  │              ↓                           │
  │  2. 正常绘制基础原图                      │
  │     （注意：换色区域不用纯黑/纯白）        │
  │              ↓                           │
  │  3. 在原图基础上，新建图层绘制 Mask 遮罩   │
  │     R/G/B = 不同换色区域                  │
  │     A = 总开关（黑=不染色）               │
  │              ↓                           │
  │  4. 检查通道互斥、边缘过渡、像素对齐       │
  │              ↓                           │
  │  5. 分别导出 Base.png 和 Mask.png         │
  │     （同一画布，确保尺寸一致）             │
  │              ↓                           │
  │  6. 输出配色方案表 + 效果预览图            │
  │              ↓                           │
  │  7. 交付程序 + 附带通道定义说明文档         │
  └─────────────────────────────────────────┘
```

### 核心原则

1. **先定方案再出图**：不同技术路线对资源的要求完全不同
2. **Mask 是关键**：遮罩图质量直接决定换色效果好坏
3. **无损格式是铁律**：Mask 图绝对不能用有损压缩
4. **通道互斥**：一个像素只能属于一个换色区域
5. **保留明暗层次**：换色区域要有足够的光影细节，否则换色后一片死板
6. **充分沟通**：通道定义、区域划分、配色范围都需要美术与程序达成共识
