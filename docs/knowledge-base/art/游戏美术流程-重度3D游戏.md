# 重度 3D 游戏 — 美术制作流程总结

> 适用范围：MMORPG、开放世界、3A 级射击游戏、竞技对战等大型项目，核心特征是资源量巨大、流程复杂、多 CP 并行、质量要求极高。

## 二、APM 视角关注重点

| 关注维度 | 具体要点 |
|----------|----------|
| **资源体量管控** | 角色 100+ / 场景模块 500+ / 特效 200+，必须建立资源台账（Asset Tracker），每周更新进度 |
| **多 CP 并行管理** | 角色 CP、场景 CP、特效 CP 可能 3~5 家同时推进，需统一规范 + 接口人制度 |
| **PBR 管线一致性** | 不同 CP 的 PBR 理解差异大，必须提供 Material Guideline + 参考球 + Checker Scene |
| **LOD & 性能预算** | 每类资产有三角面预算 / 贴图尺寸上限 / Draw Call 配额，APM 需跟踪实际 vs 预算 |
| **里程碑卡点** | Alpha / Beta / RC 每个节点有明确的资源完成率要求（如 Alpha 60%、Beta 95%） |
| **版本管理** | 必须使用 Perforce / SVN，APM 负责分支策略 + 资源锁定 + 冲突解决流程 |
| **审核前置** | 每批资源 first-pass 审核 → 批量制作 → final 审核，避免方向性返工 |
| **跨部门协同** | 美术 ↔ 程序（技术限制）/ 美术 ↔ 策划（需求变更）/ 美术 ↔ QA（Bug 验证） |

## 三、外包管理难点与常见风险

### 3.1 典型外包分工

重度 3D 游戏外包占比通常 50%~80%：

| 外包内容 | 典型分包方式 | 备注 |
|----------|-------------|------|
| 角色建模 | 按角色数量 / 按批次 | 高模 + 低模 + 贴图 打包 |
| 场景资产 | 按场景模块 / 按资产类别 | 模块化组件优先 |
| 动画 | 按动作数量 | 动捕 vs 手 K 分开报价 |
| 特效 | 按特效数量 / 按技能 | 需提供引擎内 Shader |
| 原画概念 | 按张数 | 前期集中，后期零散 |
| UI 制作 | 按界面数量 | 含交互态 |

### 3.2 常见风险

| 风险 | 表现 | 严重度 | 应对策略 |
|------|------|--------|----------|
| **PBR 质量不达标** | 金属度/粗糙度数值不准确，材质在引擎中表现与 SP 预览差异大 | 🔴 高 | 提供 Checker Scene + 标准灯光环境，要求在引擎内截图验收 |
| **面数超标** | 单模型三角面远超预算 | 🔴 高 | 下发时明确面数上限 + 集成工具自动检测 |
| **UV 不规范** | UV 岛过碎 / 比例不均 / 未对齐 Texel Density | 🟡 中 | 提供 UV Checker 贴图 + Texel Density 标准值 |
| **骨骼/权重问题** | 蒙皮穿插 / 权重涂抹不均 | 🟡 中 | 提供标准 Pose 测试 + 极端动作测试 |
| **批次间风格不统一** | 不同批次/不同美术师产出风格差异 | 🔴 高 | 首件制度 + 定期 Review + 指定 Art Lead |
| **交付延期** | CP 产能不足 / 返工导致延期 | 🔴 高 | 预留 20% Buffer + 里程碑中间检查点 + 备选 CP |
| **资源版本混乱** | 交付版本不是最新 / 未标注修改内容 | 🟡 中 | 统一交付包格式 + 变更日志 + Perforce 直提 |
| **数据安全** | 资源泄露 / CP 将资源用于其他项目 | 🔴 高 | NDA + 项目专用交付通道 + 定期审计 |

## 四、各阶段交付物清单（Deliverables）

### 阶段 ① 前期策划
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 美术规范总纲 | PDF（50~100 页） | 含风格定义 / 技术规范 / 命名规范 / 目录结构 |
| 渲染管线说明 | PDF + 工程文件 | Shader 清单 + Master Material 说明 |
| 性能预算表 | Excel | 各类资产三角面/贴图/Draw Call 预算 |
| Asset Tracker 模板 | Excel / TAPD | 全量资源跟踪表 |

### 阶段 ② 概念设计
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 角色概念原画 | PSD + PNG | 三视图 + 配色方案 + 材质标注 |
| 场景概念 Key Visual | PSD + PNG | 2~3 个关键场景气氛图 |
| 武器/道具概念 | PSD + PNG | 含尺寸比例参考 |
| 特效风格板 | PPT / 视频 | 元素色板 + 参考视频链接 |
| UI Style Guide | Figma / Sketch | 组件库 + 色彩系统 + 字体规范 |

### 阶段 ③ 白盒验证
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 白盒关卡 | 引擎场景文件 | 含尺度标注 + 游玩路径 |
| 角色白模 | FBX | 含碰撞体 + 基础骨骼 |
| Benchmark 报告 | PDF | 渲染管线性能基线数据 |

### 阶段 ④ 高模 & 中模
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 高模 | ZBrush(.ZTL) / FBX | 含 SubTool 分层 |
| 中模（游戏用） | FBX | 符合面数预算 + 正确法线 |
| 低模（LOD1~3） | FBX | 按 LOD 级别减面 |
| UV 展开 | FBX 内置 | 2 套 UV（贴图用 UV0 + 光照用 UV1） |

### 阶段 ⑤ 贴图与材质
| 交付物 | 格式 | 说明 |
|--------|------|------|
| PBR 贴图套装 | TGA / PNG | BaseColor / Normal / ORM（AO+Rough+Metal） |
| 自发光贴图 | TGA / PNG | 如有 |
| Substance 源文件 | .spp / .sbs | 可追溯修改 |
| 材质效果截图 | PNG | 标准灯光环境下正面/侧面/背面 |

### 阶段 ⑥ 绑定 & 动画
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 绑定文件 | FBX + Maya(.ma) | 含骨骼层级 + 控制器 |
| 蒙皮权重 | FBX | 极端 Pose 测试通过 |
| 动画文件 | FBX（每动作单独） | Idle / Walk / Run / Attack... |
| 动画状态机文档 | 文本 / 引擎蓝图 | 状态转换关系 |
| 面部 Morph Target | FBX | 52 个 ARKit Blend Shape（如需） |

### 阶段 ⑦ 特效 & 光照
| 交付物 | 格式 | 说明 |
|--------|------|------|
| 粒子特效 | 引擎资产 | Niagara / Cascade |
| 特效贴图 | TGA / EXR | 含 Flipbook / 噪声图 |
| 光照设置文件 | 引擎场景 | 含 LightMap 烘焙参数 |
| 后处理配置 | 引擎资产 | LUT + PP Volume |

### 阶段 ⑧ 集成 & 优化
| 交付物 | 格式 | 说明 |
|--------|------|------|
| LOD 配置 | 引擎资产 | 各级别切换距离 |
| 性能报告 | PDF / Excel | GPU Time / Draw Call / VRAM |
| 压缩配置 | 引擎设置 | ASTC / ETC2 / BC7 参数 |
| 最终资源包 | 引擎项目 | 可直接打包 |

### 阶段 ⑨ 里程碑
| 里程碑 | 资源完成率 | 关键标准 |
|--------|-----------|----------|
| Alpha | 60% | 核心角色 + 1 个完整场景 + 基础 UI 可跑通 |
| Beta | 95% | 全量内容就位 + 性能达标 |
| RC | 100% | Final Polish + 提审素材齐全 |

## 五、验收标准建议

### 5.1 角色模型验收

| 检查项 | 标准 | 优先级 |
|--------|------|--------|
| 三角面数 | 符合预算（如主角 ≤ 30K，NPC ≤ 15K） | P0 |
| 法线方向 | 全部外翻，无黑面 | P0 |
| UV Texel Density | 全身 TD 偏差 ≤ 15% | P0 |
| PBR 数值准确性 | 金属/非金属 Metallic 为 0 或 1，Roughness 合理 | P0 |
| 蒙皮权重 | 极端 Pose 无穿插 / 无明显塌陷 | P0 |
| 命名规范 | 骨骼 / 网格 / 材质球全部英文命名 | P1 |
| LOD | 至少 3 级 LOD，最低级 ≤ 原面数 20% | P1 |

### 5.2 场景资产验收

| 检查项 | 标准 |
|--------|------|
| 模块化接缝 | 拼接无缝隙、无法线错误 |
| Lightmap UV | UV1 无重叠，Padding ≥ 2px |
| Pivot Point | 底部中心（地面物体）/ 几何中心（悬挂物体） |
| 碰撞体 | 简化碰撞体，面数 ≤ 模型 10% |
| 贴图分辨率 | 大物件 2048 / 中物件 1024 / 小物件 512 |

### 5.3 动画验收

| 检查项 | 标准 |
|--------|------|
| 帧率 | 30FPS |
| 根运动 | Root Motion 与实际位移匹配 |
| 循环衔接 | Loop 动画首尾无跳变 |
| 穿插检测 | 无自穿插（武器/披风/头发） |
| 重心 | 符合物理重心，无漂浮感 |

### 5.4 特效验收

| 检查项 | 标准 |
|--------|------|
| Overdraw | 单特效 Overdraw ≤ 屏幕 30% |
| 粒子数 | 峰值粒子数 ≤ 预算 |
| 生命周期 | 特效结束后无残留粒子 |
| 匹配度 | 与概念设定一致 |

## 六、关键时间节点参考

| 阶段 | 典型工期 | 备注 |
|------|----------|------|

| **总计** | **12~36 个月** | 重度项目标准周期 |
