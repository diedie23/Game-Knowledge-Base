📊 1. 换色方案概览与资源需求对照

核心原则：不同的换色技术路线，对美术的出图要求差异很大。先了解方案再出图，可以避免返工。
方案原理简述美术需提供的资源适用场景
① 调色板替换用索引色/LUT 表做颜色映射原图 + 调色板贴图（256×1）像素风格、复古游戏
② 遮罩分区 + HSV/Tint用 Mask 图标记区域，Shader 色相偏移原图 + Mask 遮罩图最主流方案，卡通/Q版
③ 灰度底图 + 叠色原图去色只保留明暗，乘以目标色灰度底图（+ 可选 Mask）简单场景、纯色服装
④ 通道混合染色利用 RGB 通道矩阵运算混合原图 + Mask 遮罩图高品质需求
⑤ 多套贴图直出每种颜色出一套图N 套完整贴图颜色少、品质极高

🎯 2. 方案②详解：遮罩分区换色（重点）

2.1 美术需要提供的资源清单
📁 角色资源包
├── character_base.png        ← ① 基础原图（带完整色彩和细节）
├── character_mask.png        ← ② 换色遮罩图（RGBA 四通道）
├── palette_config.json       ← ③ 配色方案表（可选但推荐）
└── preview_thumbnail.png     ← ④ 换色效果预览缩略图（可选）
2.2 基础原图（Base Texture）
要素规范
格式PNG（带 Alpha 通道）
尺寸与正常角色出图一致（2^N，如 512×512）
色彩要求正常绘制完整色彩，包含光影、褶皱等细节
注意换色区域不要用纯黑(0,0,0)或纯白(255,255,255)，否则 HSV 偏移无效
饱和度换色区域建议使用中等饱和度 40%~70%
2.3 换色遮罩图（Mask Texture）— 最关键
⛔ 通道互斥铁律：同一像素在 R/G/B 三个通道中只能有一个通道有值，其余必须为 0。A 通道独立控制染色开关（白=可染 · 黑=禁染）。
通道含义
R（红色）换色区域 1（如：头发）
G（绿色）换色区域 2（如：上衣）
B（蓝色）换色区域 3（如：裤子/裙子）
A（Alpha）总开关 — 白=可染色，黑=不染色
2.4 遮罩图制作要求

🔴 铁律 #1：尺寸必须与基础原图完全一致，像素级对齐，差 1px 也打回。

🔴 铁律 #2：格式为 PNG（RGBA 32bit）或 TGA，绝对禁止 JPEG（有损压缩导致通道噪点）。

🔴 铁律 #3：区域边缘建议留 1~2px 渐变过渡，避免硬边锯齿。非染色区域 A 通道涂黑(0)。

🎮 3. 方案①详解：调色板替换（Palette Swap）

适用于像素风格游戏，原理是用一张小贴图做颜色映射。

3.1 资源清单
📁 角色资源包
├── character_indexed.png     ← ① 索引色原图
└── palettes/
    ├── palette_default.png   ← ② 默认调色板（256×1）
    ├── palette_fire.png      ← ③ 火焰配色
    └── palette_ice.png       ← ④ 冰霜配色
3.2 制作规范
要素规范
颜色数量控制在 8~32 色以内
颜色唯一性需要被独立换色的区域，其 R 通道值必须唯一
避免渐变不要使用大面积平滑渐变，每种颜色应为纯色色块
调色板尺寸256×1 像素，PNG 无压缩/无抗锯齿
采样模式程序需设置为 Point（最近邻）采样，禁止线性插值

🌓 4. 方案③详解：灰度底图 + 叠色

最简单的方案，适合纯色面积大、细节需求低的场景。公式：最终颜色 = 灰度值 × 目标颜色

要素规范
色彩完全去色，只保留明暗关系
中间灰基础亮度建议在 128~200 之间
细节褶皱、阴影、高光全部用灰度表现
⚠️ 注意：灰度叠色只能让颜色变暗（乘法特性），想提亮需要 HDR 值(>1.0)。如需多个独立换色区域，仍需配合 Mask 遮罩图。

🔧 5. 扩展：超过 3 个换色区域

方案区域数优点缺点
A. 灰度分段6+仅 1 张图不支持渐变过渡，边缘必须硬切
B. 多张 Mask（推荐）6+支持渐变+1 次采样，性能略有开销
C. UV 多象限4不需要额外 Mask 图需要重新规划 UV，只适合特定引擎

⚠️ 7. 常见踩坑与避免

问题原因解决方案
换色后出现杂色/噪点Mask 图用了 JPEG 或有损压缩必须使用 PNG/TGA 无损格式
换色边缘有明显锯齿Mask 图边缘是硬切的像素边缘做 1~2px 灰度渐变过渡
黑色/白色区域换不了色纯黑 HSV 色相偏移无效换色区域避免纯黑纯白
两个区域换色互相串色Mask 中同一像素多个通道都有值确保 R/G/B 通道互斥
换色后光影消失/变平原图绘制时阴影不够保证换色区域有足够明暗层次
皮肤/脸部意外被染色A 通道没有正确涂黑检查 Alpha 通道的非染色区域标记
调色板换色颜色错乱原图中两种颜色的 R 值相同确保索引色的 R 通道唯一
Mask 图和原图对不上导出尺寸不一致或有偏移使用同一画布分层导出

✅ 六大核心原则

1. 先定方案再出图 — 不同技术路线对资源要求完全不同

2. Mask 是关键 — 遮罩图质量直接决定换色效果好坏

3. 无损格式是铁律 — Mask 图绝对不能用有损压缩

4. 通道互斥 — 一个像素只能属于一个换色区域

5. 保留明暗层次 — 换色区域要有足够的光影细节

6. 充分沟通 — 通道定义、区域划分需美术与程序达成共识

❌ 常见反面案例

· Mask 图使用 JPEG → 通道噪点脏色

· 同一像素多通道有值 → 换色串色

· 换色区域用纯黑/纯白 → HSV 偏移无效

· Mask 与原图尺寸不一致 → 对不齐

· 不提供配色表 → 程序瞎猜颜色范围

· 灰度图有大面积死黑 → 叠色后黑洞
