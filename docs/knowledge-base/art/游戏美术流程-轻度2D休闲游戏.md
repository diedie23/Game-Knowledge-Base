## 二、APM 视角关注重点

| 关注维度 | 具体要点 |
|:---:|:---:|
| **迭代节奏** | 休闲游戏 1~2 周一个版本，美术排期必须与开发同步甚至提前一步 |
| **风格一致性** | 资源量小但风格跑偏影响大，需在概念阶段锁定 Style Guide 并严格执行 |
| **分辨率适配** | 需覆盖 16:9 / 18:9 / 20:9 等多比例，UI 安全区必须提前定义 |
| **资源复用率** | 休闲游戏利润率低，鼓励通过换色/换皮/组合方式复用资源，降低制作成本 |
| **商店素材质量** | App Store / Google Play 截图和 Icon 直接影响下载转化率，需专人负责 |
| **动效节奏** | Juice 是休闲游戏的灵魂，需在批量制作阶段预留动效工作量 |
| **包体控制** | 目标 50~100MB 以内，图集合并和压缩是必须项 |

## 三、外包管理难点与常见风险

### 3.1 典型外包使用场景

轻度休闲游戏通常外包占比低（20%~40%），主要外包内容：
- 角色/道具立绘（量少但要求风格统一）
- UI 图标批量制作
- 节日/活动主题皮肤

### 3.2 常见风险

| 风险 | 表现 | 应对策略 |
|:---:|:---:|:---:|
| **风格偏差** | 外包画风与项目不统一，返工 2~3 轮 | 提供完整 Style Guide + 参考图至少 5 张 + 首件确认 |
| **沟通过度** | 小量外包沟通成本反而超过内部制作 | 评估是否值得外包，单件沟通成本 > 制作成本时改为内做 |
| **交付格式错误** | PNG 无 Alpha / 尺寸非 2^N / 色彩空间错误 | 提供《资源交付规范》+ 使用检测工具自检 |
| **排期碎片化** | 外包方因量小不优先排期 | 合并需求批次下发，避免零散单件 |
| **版权合规** | 外包使用了网图素材 | 合同中明确原创声明 + 查重机制 |

## 四、各阶段交付物清单（Deliverables）

### 阶段 ① 立项 & 风格定义

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| 竞品美术分析报告 | PDF/PPT | 至少 5 款竞品截图 + 风格关键词提取 |
| 美术风格指南 v0.1 | PDF | 风格关键词、配色板、参考图、禁忌项 |
| 调色板 | PNG / ASE | 主色 + 辅色 + 强调色 |

### 阶段 ② 概念设计

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| 核心角色概念图 | PSD + PNG | 2~3 版方案，含正面/侧面 |
| 核心道具概念图 | PSD + PNG | 关键道具 3~5 件 |
| UI Layout 草稿 | Figma / PSD | 主要界面流程草图 |
| 配色方案确认表 | PDF | 标注 HEX + 用途 |

### 阶段 ③ 原型验证

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| Placeholder 资源包 | PNG | 临时占位图，尺寸与最终一致 |
| 美术方向确认邮件 | 文本 | 玩法验证通过后确认美术方向 |

### 阶段 ④ 批量制作

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| 角色/道具立绘 | PNG（带 Alpha） | 按命名规范，尺寸 2^N |
| UI 切图 | PNG | 9宫格标注 + 命名规范 |
| Spine 动画文件 | JSON + Atlas + PNG | 含骨骼 + 动画数据 |
| 帧动画精灵图 | PNG Sprite Sheet | 标注帧数 + 帧率 |
| 资源清单表 | Excel / CSV | 全量资源索引 |

### 阶段 ⑤ 集成联调

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| 引擎内截图确认 | PNG | 各分辨率下的实机截图 |
| 适配问题清单 | 文本 / TAPD | 需修正项列表 |

### 阶段 ⑥ 打磨优化

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| Juice 动效列表 | 文本 | 每个交互点的反馈动效定义 |
| 优化后资源包 | ZIP | 图集合并 + 压缩后最终版 |
| 性能测试报告 | PDF | 包体大小 / Draw Call / 内存 |

### 阶段 ⑦ 上线 & 运营

| 交付物 | 格式 | 说明 |
|:---:|:---:|:---:|
| App Icon | PNG 1024×1024 | iOS / Android 各尺寸 |
| 商店截图 | PNG | 5~8 张，按商店规范 |
| 宣传视频 | MP4 | 15~30 秒 |
| 节日皮肤/活动资源 | PNG / Spine | 按活动排期交付 |

## 五、验收标准建议

### 5.1 通用验收标准

| 检查项 | 标准 | 优先级 |
|:---:|:---:|:---:|
| 文件格式 | PNG，带正确 Alpha 通道 | P0 |
| 画布尺寸 | 2^N（如 512×512） | P0 |
| 命名规范 | `类型_名称_编号_变体.png` | P0 |
| 风格一致 | 与 Style Guide 无明显偏差 | P0 |
| 色彩准确 | 主色与调色板 ΔE < 5 | P1 |
| 边缘干净 | 无杂色像素、无半透明残留 | P1 |
| 文件大小 | 单图 ≤ 500KB（压缩前） | P2 |
| 图层规范 | PSD 源文件图层命名清晰 | P2 |

### 5.2 动画验收标准

| 检查项 | 标准 |
|:---:|:---:|
| 帧数 | 与需求一致 |
| 帧率 | Spine 30FPS / 帧动画 12~24FPS |
| 循环衔接 | Loop 动画首尾帧无跳帧 |
| 骨骼命名 | 英文命名，层级清晰 |
| 导出完整性 | JSON + Atlas + PNG 三件齐全 |

### 5.3 UI 验收标准

| 检查项 | 标准 |
|:---:|:---:|
| 切图精度 | 无多余透明像素 |
| 9宫格标注 | 拉伸测试无变形 |
| 交互状态 | Normal / Pressed / Disabled 齐全 |
| 字体 | 标注字号 / 字重 / 颜色 HEX |

## 六、关键时间节点参考

| 阶段 | 典型工期 | 备注 |
|:---:|:---:|:---:|
| 风格定义 | 2~3 天 | 可与策划并行 |
| 概念设计 | 3~5 天 | 含 2 轮修改 |
| 原型验证 | 3~7 天 | 与开发同步 |
| 批量制作 | 1~3 周 | 取决于资源量 |
| 集成联调 | 3~5 天 | 含适配调整 |
| 打磨优化 | 3~5 天 | 含性能优化 |
| 商店素材 | 2~3 天 | 提审前完成 |
| **总计** | **4~8 周** | 轻度游戏标准周期 |
