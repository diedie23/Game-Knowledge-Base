🏗️ 1. 外包协同总则与策略选型

"外包不是'甩锅'——它是产能杠杆。用好了让 20 人团队拥有 100 人的产出；用不好就是花了 100 人的钱，产出还不如 20 人自己做。"

— 某大型手游制作人

1.1 外包策略选型矩阵

资产类型推荐模式适用场景风险等级
核心角色（主角/主要NPC）内部制作品质和风格把控要求极高—
量产角色（小怪/路人）驻场外包风格已锁定、需频繁沟通迭代🟡 中
场景资产（Kit/道具）远程外包标准化程度高、可批量验收🟢 低
概念原画/Key Visual内部 + 顾问风格探索期、需要创意能力—
动画/特效混合模式核心战斗内部、过场/环境外包🟡 中
UI 切图/图标远程外包规格明确、批量生产🟢 低

1.2 外包占比健康区间

项目阶段外包产能占比说明
预研期≤ 10%风格未锁定，外包难以参与创意工作
Alpha 前20~40%标准件开始量产，CP 可介入
量产高峰50~70%内部团队转为品质把控+核心资产，CP 承接主要产能
收尾/维护期30~50%逐步收缩外包规模，知识回流内部

🏅 2. CP 准入评估与分级管理

2.1 准入评估维度（百分制）

评估维度权重评估要点合格线
作品品质35%历史案例与本项目风格匹配度、技术表现力≥ 70 分
产能稳定性20%团队规模、核心人员稳定性、并发项目数≥ 60 分
沟通协作20%响应速度、文档规范度、问题主动暴露意识≥ 65 分
交付准时率15%历史项目的 On-Time Delivery 率≥ 80%
安全合规10%保密协议签署、文件管理规范、防泄露措施必须合格

2.2 CP 分级体系

等级综合评分权限配额
S 级≥ 90可承接核心角色外包、参与预研、获得长期框架合同不限
A 级80~89可承接量产角色/主要场景，每批次 ≤ 20 件月产能 60%
B 级70~79可承接标准场景道具/UI/图标，每批次 ≤ 50 件月产能 40%
C 级60~69仅承接最标准化的批量资产，需加强监控月产能 20%
⚠️ 降级/淘汰机制：连续 2 个月交付合格率 < 70% → 降一级；降至 C 级后仍不达标 → 暂停合作 3 个月后重新评估。

📋 3. 需求下发规范（Briefing Package）

"外包做出来不像，80% 的原因不是 CP 能力不行，而是 Briefing 没写清楚。给模糊需求就别期待精确产出。"

— 某 3A 外包经理

3.1 Briefing Package 必备要素

📦 需求包完整度检查（缺一不可）

01风格锚点图：≥ 3 张已确认的风格参考（含"正确范围"和"超出底线"的边界示例）
02技术规格表：面数上限/贴图尺寸/骨骼数/文件格式/命名规范/目录结构（引用内部标准文档）
03设计文档：角色背景/功能需求/关键设计意图/情绪关键词
04三视图/正侧背：已审核通过的定稿设计图（非草图）
05验收标准清单：量化指标 + 主观评分维度 + 评分权重
06里程碑计划：分阶段交付时间点 + 每阶段交付物 + 审核反馈周期
07沟通机制：对接人/审核人/响应时效/周会安排/紧急联系方式
08合同条款摘要：付款节点/修改轮次/超出范围变更计费

3.2 常见 Briefing 缺陷与后果

Briefing 缺陷后果修复成本
无风格锚点、仅给"参考图"CP 理解偏差导致多轮返工平均 +3 轮修改（成本翻倍）
技术规格含糊（"差不多就行"）面数/贴图超标，后期才发现需重做单资产重做成本 = 首次制作 ×1.5
无里程碑拆分（一次性交付）发现问题太晚、修改成本极高后期修改成本 = 早期 ×4~8
验收标准主观化（"好看就行"）验收扯皮、CP 无法自检每次验收消耗 3~5 天反复沟通

4.1 标准四阶段模型

全套文件/自检报告

M1 白模/草稿低模白模 / 线稿草图大比例/剪影/造型语言/体量感主美确认✓签约款 20%
M2 高模/精稿高模 / 精修稿（含正侧背）细节品质/风格还原度/设计意图主美+制作人确认✓阶段款 30%
M3 引擎效果引擎内截图 / 材质球 / LOD引擎还原度/性能达标/材质正确TA 技术验收✓ + 主美视觉验收✓阶段款 30%

🚨 红线：禁止跳过里程碑！"为了赶进度直接交终稿"是最常见的翻车原因——问题积累到 M4 才暴露，修改成本是 M1 阶段的 8 倍以上。

⭐ 5. 资产验收标准与评分体系

5.1 验收评分维度

评分维度权重满分标准不合格线
风格还原度30%与风格锚点图高度一致，无偏差感< 60 分
设计还原度25%与设计稿 1:1 还原，关键特征完整< 60 分
技术规格达标25%面数/贴图/骨骼/命名 100% 达标任一项超红线
细节品质15%材质表现/细节丰富度/专业完成度< 50 分
文件规范度5%命名/目录/图层/源文件整理达标< 60 分

5.2 评分等级与处理

总分等级处理方式
≥ 90优秀直接通过，记录到 CP 档案加分项
80~89合格通过，可能有少量微调（不计修改轮次）
70~79有条件通过需修改后二次审核，消耗 1 次修改轮次
60~69不合格退回修改，消耗修改轮次，需书面说明改进计划
< 60严重不合格退回重做或终止该资产合作，触发 CP 降级预警

5.3 客观指标自动化检测（CI 集成）

🤖 可自动化检测的项目（工具完成，无需人工）

A1文件命名是否符合规范（正则匹配）
A2面数是否在红线内（FBX 解析）
A3贴图尺寸/格式/色彩空间是否正确
A4骨骼数量是否在预算内
A5是否存在 N-gon / 重叠顶点 / 法线翻转
A6UV 是否越界 / Texel Density 偏差率
A7目录结构是否符合模板
💡 最佳实践：CI 自动检测在 CP 提交时即运行，不合格直接打回并生成详细报告，减少人工审核负担 60% 以上。

🔄 6. 修改轮次管控与变更流程

6.1 标准修改轮次预算

资产类型含内修改轮次超出计费说明
概念原画3 轮每轮 +15% 合同额每轮修改不超过总面积 30%
3D 角色2 轮每轮 +20% 合同额大改（改造型/换风格）不计入
场景模块2 轮每轮 +15% 合同额规格调整不算修改轮次
UI/图标2 轮每轮 +10% 合同额功能变更导致的改动不计入

6.2 "修改" vs "变更" 的边界定义

类型定义计入修改轮次？需审批？
精调不改变设计方向，仅微调细节/颜色/比例❌ 不计否
修改在原设计框架内修改局部内容（< 30% 改动量）✅ 计入否
变更改变设计方向/新增功能需求/修改技术规格❌ 不计（另付费）✅ 需 APM 审批 + 变更单
重做整体推翻重新制作❌ 不计（另起合同）✅ 需制作人审批
⚠️ 制作人经验："分不清'修改'和'变更'是外包管理最大的灰色地带。必须在合同和 Briefing 中明确边界定义，否则 CP 认为你在白嫖改动，你认为 CP 在碰瓷加钱。"

🔒 8. 知识产权与安全合规

8.1 保密等级与对应措施

保密等级适用资产要求措施
机密核心主角/关键剧情/未公开玩法独立加密网盘传输、禁止外网传播、水印追溯、驻场制作
内部量产角色/场景/标准资产企业网盘传输、NDA 覆盖、CP 不得对外展示
一般通用 Kit/图标/UI 元素标准传输方式、基本 NDA

8.2 知识产权归属条款要点

  • 工作成果归属：合同约定所有交付物的著作权、改编权归甲方所有，CP 不得留存或二次使用
  • 展示权：CP 展示合作案例需获得书面授权，且在产品正式发布后方可展示
  • 源文件留存：项目结束后 CP 须在 30 天内删除所有源文件并提供删除证明
  • 泄露追责：合同明确泄露的违约金条款（通常为合同总额的 3~10 倍）

8.3 安全操作 Checklist

🔐 外包安全自检

01NDA（保密协议）是否已签署且在有效期内
02文件传输是否通过指定安全渠道（禁止微信/QQ直传）
03参考图/设计稿是否加上项目水印
04CP 人员是否已签署个人保密承诺书
05CP 是否有独立的项目工作区（物理隔离或逻辑隔离）
06已完成资产的访问权限是否按需收缩

🟢🔴 10. Do's & Don'ts 红黑榜

✅ Do — 正确做法

  • 下发需求前确保 Briefing Package 8 项要素全齐
  • 用风格锚点图代替口头描述"我想要的感觉"
  • 严格执行里程碑分阶段审核，越早发现问题修改越便宜
  • 审核反馈用截图标注 + 量化描述，不说"这里不对"
  • 合同明确修改 vs 变更的边界定义和计费规则
  • 定期做CP 绩效评估，用数据驱动合作关系管理
  • 保持审核反馈时效承诺——别让 CP 等你比你等 CP 更久

❌ Don't — 错误做法

  • ❌ "先做出来看看再说"——没有方向就开始制作 = 100% 返工
  • ❌ 用微信口头传需求——无书面记录 = 验收时互相扯皮
  • ❌ 跳过里程碑直接要终稿——M4 发现问题修改成本 ×8
  • ❌ 审核时只说"不好看"——CP 不知道怎么改 = 无效沟通
  • ❌ 为赶进度降低验收标准——技术债 + 品质劣化不可逆
  • ❌ 所有 CP 一视同仁——不分级 = 把关键资产交给不靠谱的人
  • ❌ 忽视保密管理——泄露一次就是产品级灾难
