# APM 个人成长路线图

> **适用阶段**：全阶段 | **优先级**：中 | **负责人**：周八
>
> 本文档描述从初级 APM 到资深 APM 的能力模型、学习路径与阶段性自我评估清单。

> 💡 **核心理念**：APM 的成长不是线性的，每个阶段都有一个**关键突破点**。突破了就上台阶，卡住了就是瓶颈。

---

## 1. APM 能力模型

### 1.1 硬技能矩阵

| 技能领域 | 初级 | 中级 | 高级 | 资深 |
|---------|------|------|------|------|
| **项目管理方法论** | 了解敏捷基础 | 熟练 Scrum/Kanban | 混合方法论灵活运用 | 可定制团队方法论 |
| **美术管线理解** | 知道基本流程 | 理解各工种管线细节 | 能优化管线效率 | 能设计全新管线 |
| **成本核算** | 能做基础报价 | 熟练人天模型 | 能做年度预算规划 | 能做 ROI 分析与战略决策 |
| **工具链** | 会用 TAPD/Jira | 能配置自动化规则 | 能设计工具链方案 | 能推动工具研发 |
| **引擎基础** | 知道 UE/Unity 是什么 | 能在引擎中查看资产 | 理解渲染管线与性能 | 能与 TA 讨论技术方案 |
| **数据分析** | 能做 Excel 报表 | 能搭建看板/仪表盘 | 能从数据中发现问题 | 数据驱动决策 |
| **风险管理** | 能识别明显风险 | 能评估风险等级 | 能制定应对策略 | 能建立预警体系 |

## 2. 成长阶段划分

### 2.1 四阶段定义

| 阶段 | 经验 | 核心能力 | 典型职责 | 管理范围 |
|------|------|---------|---------|---------|
| 🟢 **初级 APM** | 0~2 年 | 执行力、基础沟通 | 跟踪任务进度、整理会议纪要、维护排期表 | 协助管理 1 个工种 |
| 🟡 **中级 APM** | 2~5 年 | 独立管理、数据能力 | 独立管理一个版本/模块、成本核算、外包管理 | 独立管理 10~20 人 |
| 🟠 **高级 APM** | 5~8 年 | 战略规划、机制设计 | 多版本/多项目管理、团队建设、流程优化 | 管理 20~50 人 |
| 🔴 **资深 APM** | 8 年+ | 行业影响力、组织能力 | 美术中台建设、跨项目资源调配、培养后继 | 管理 50+ 人或多项目 |

### 2.2 各阶段关键突破点

| 突破 | 标志 |
|------|------|
| 初级→中级 | 第一次独立管理一个版本并按时交付 |
| 中级→高级 | 第一次在没人指导下处理项目危机并化解 |
| 高级→资深 | 第一次建立了可复用的管理体系/工具 |

## 5. 常见成长瓶颈与突破

### 5.1 "执行到管理"的瓶颈

| 表现 | 突破方法 |
|------|---------|
| 事事亲力亲为，不敢授权 | 练习"教会别人做"而非"自己做" |
| 关注细节但失去全局视角 | 每周花 1 小时看全局进度 |
| 被动接受需求 | 主动提出优化建议 |

### 5.2 "管理到领导"的瓶颈

| 表现 | 突破方法 |
|------|---------|
| 能管事但不会管人 | 学习教练技术、1on1 方法 |
| 能解决问题但不能预防 | 建立机制和流程 |
| 有管理经验但没有行业影响力 | 写文章/做分享/建知识库 |

## 6. 优秀 APM 的共性特征

| 特征 | 说明 |
|------|------|
| 📊 **数据驱动** | 用数据说话，不凭感觉 |
| 🔮 **前瞻性** | 能在问题发生前预判 |
| 🤝 **信任基石** | 各方都信任你的判断和承诺 |
| 📝 **文档化思维** | 重要的事一定落纸面 |
| ⚡ **快速响应** | 坏消息不过夜 |
| 🌱 **持续学习** | 不断扩展知识边界 |
| 😊 **正能量** | 高压下依然能稳住团队情绪 |
