⚙️ 1. 美术任务状态机

1.1 标准状态流

待审核

* 审核不通过 → 修改中 → 再次提审；可从进行中挂起或取消

💡 核心原则：状态流转必须单向推进，禁止跳过审核环节直接标记完成。
1.2 状态定义
状态英文 Key说明进入条件
待排期Backlog需求已确认，等待排入 Sprint评审通过
已排期Planned已分配到 SprintAPM 排期
进行中In Progress美术正在制作美术领取任务
待审核In Review提交审核美术完成后提审
审核通过Approved主美/TA 审核通过审核人操作
审核不通过Rejected需要修改审核人打回
修改中Revising根据审核意见修改自动流转
已完成Done任务关闭所有审核通过
搁置/挂起On Hold等待外部依赖APM 操作
已取消Cancelled需求取消APM/策划操作
1.3 TAPD 配置方法
TAPD → 项目设置 → 工作流配置 → 自定义状态
1. 添加上述状态
2. 配置状态转换权限
3. 设置必填字段（如审核意见）
1.4 Jira 配置方法
Jira → 项目设置 → Workflows → 编辑工作流
1. 添加 Status
2. 配置 Transitions
3. 设置 Conditions/Validators/Post Functions

2.2 审核人自动指派
审核阶段审核人触发条件
视觉审核主美状态变为"待审核"
技术审核TA任务标签含"性能相关"
最终审核APM所有前置审核通过
2.3 TAPD 自动化规则配置
# 规则1: 新任务自动指派
触发条件: 任务创建且module=Character
执行动作: 
  - 自动指派给: 角色组长-张三
  - 自动添加关注人: APM-周八

# 规则2: 审核通过自动流转
触发条件: 状态变为"审核通过"
执行动作:
  - 状态流转到: 已完成
  - 通知: 任务创建者

# 规则3: 超时自动预警
触发条件: 状态为"进行中"超过5天
执行动作:
  - 发送提醒给: 负责人 + APM

状态变更关注人企微/飞书"[标题] 状态更新为 [新状态]"
审核打回制作人企微/飞书 + 邮件"你的任务 [标题] 被打回，原因：[意见]"
