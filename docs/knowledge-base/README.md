### 📌 两篇核心文档的区分与导航

许多同学初次进入知识库时，会分不清这两篇文档的区别。以下是一张对比表帮助你"对号入座"：

| 维度 | 🔄 **美术生产管线总览** | 📅 **美术排期与里程碑管理** |
|:---:|:---:|:---:|
| **一句话定位** | 📍 **生产地图** — "路怎么走" | ⏱️ **调度手册** — "走多快、什么时候到" |
| **核心视角** | 🔧 **工艺维度**：每个工序做什么、产出物是什么、验收标准是什么 | 📅 **时间维度**：怎么排期、怎么切 Sprint、怎么识别进度偏差 |
| **内容覆盖** | 按游戏类型（轻度2D/重度3D/二次元）拆解的全工序制作流程 + 外包风险 + 交付标准 | 不分游戏类型的 Sprint 排期模板、里程碑分解、燃尽图跟踪、风险预警 SOP |
| **典型读者问题** | "一个3D角色从原画到引擎集成要经过哪些工序？" | "这个 Sprint 应该排多少任务？燃尽图偏差 20% 该怎么办？" |
| **最佳阅读时机** | 🟢 项目立项 / 新人入职 / 管线定义阶段 | 🟡 每个 Sprint 规划 / 里程碑复盘 / 进度出现偏差时 |
| **文档类型** | 按类型横向对比的参考手册（查阅型） | 含操作模板和决策流程的实操指南（执行型） |

> 🔗 **关联阅读建议**：先读「管线总览」建立全景认知，再读「排期管理」掌握落地操作。两篇互为上下游：管线总览告诉你"有哪些工序和交付物"，排期管理告诉你"如何把这些工序排进时间线并跟踪交付"。
