🚨 1. 性能红线与规格限制

⛔ 红线 = 不可违反的硬性指标。任何提交物违反以下红线将被直接打回，不进入美术评审环节。

1.1 贴图尺寸与内存预算

资源类型最大尺寸推荐尺寸内存预算（RGBA 未压缩）引擎压缩后（ASTC 6×6）
角色全身图 Base2048×20481024×102416 MB / 4 MB~1.28 MB / ~320 KB
单部件（头/手/鞋等）512×512256×2561 MB / 256 KB~80 KB / ~20 KB
表情/面部特写256×256128×128256 KB / 64 KB~20 KB / ~5 KB
精灵图 Atlas4096×40962048×204864 MB~5 MB
缩略图/图标128×12864×6464 KB~5 KB

🔴 铁律 #1：所有贴图尺寸必须为 2 的幂次（POT）或 4 的整数倍（ASTC/ETC2 压缩要求）。非 POT 贴图在移动端 GPU 上会被 padding 到最近的 POT，浪费显存。

🔴 铁律 #2：单个角色（含所有部件贴图）总显存 ≤ 4 MB（ASTC 压缩后），超出将触发 LOD 降级。

🔴 铁律 #3：如果部件不需要透明（如纯色背景块），禁止使用 RGBA，改用 RGB 格式——省 25% 显存。

🧍 2. 素体安全区与关节 Padding 规范

素体（Nude Body）是所有部件的定位基准层。换装系统依赖素体坐标进行拼接，因此素体的规格直接决定整个换装系统的质量。

2.1 素体安全区定义

安全区（Safe Zone）= 素体上被服装覆盖时绝对不能留空的区域。即使服装完全遮挡，素体该区域也必须绘制完整的皮肤纹理，防止穿模时露出空白。
关节部位安全区范围（素体坐标系）Padding 预留说明
颈部下巴底线 → 锁骨线8 px高领/低领切换时可见
肩部颈根 → 上臂 1/312 px袖口位置不确定，预留大
腰线肋骨底 → 髋骨顶16 px上下装衔接最高频穿模区域
肘部上臂 2/3 → 前臂 1/38 px弯曲动画时外露
腕部前臂末端 → 手掌根6 px手套/袖口交界
膝盖大腿 2/3 → 小腿 1/310 px裤脚/长靴交界
踝部小腿末端 → 脚背6 px鞋口高度不统一
⚠️ Padding 计算公式：Padding = max(动画最大形变量, 部件交界偏差) + 4px 安全余量。以 1024×1024 画布为基准，其他尺寸等比缩放。

2.2 素体与部件的坐标锚定

所有部件必须与素体共享同一画布原点 (0, 0)，部件图与素体图画布尺寸完全一致。

🔴 铁律：禁止"裁切后单独导出部件"——部件 PNG 的画布大小必须等于素体画布大小。引擎通过 Alpha 通道裁切透明区域，美术不得手动裁切。

📚 3. 图层绝对层级顺序（Z-Order 铁律）

2D 换装系统的渲染顺序完全由 Z-Order 整数值决定。Z 值越大越靠前（离摄像机越近）。

⛔ 此层级表为全项目强制标准。任何部件的 Z-Order 不得与此表冲突，自定义部件只能使用预留的 Gap 区间。
Z=100前景特效层（粒子/光晕）不参与换装
Z=90前发 / 刘海遮挡面部上半
Z=85帽子 / 头饰可覆盖前发
Z=80面部表情层（眼/眉/嘴）独立切换表情
Z=75面部底层（脸型/腮红）底色层
Z=70眼镜 / 面具叠在面部之上
Z=60上装（外套/披风）最外层服装
Z=55上装（内衣/T恤）被外套遮挡
Z=50手臂 / 手套与上装同级或略低
Z=45身体素体（躯干）基准层
Z=40下装（裤子/裙子）腰线与上装衔接
Z=35腿部素体被裤子覆盖
Z=30鞋子 / 袜子踝部衔接
Z=20后发 / 长发在身体后方
Z=15翅膀 / 背饰身体最后方
Z=10尾巴最底层装饰
Z=0阴影 / 投影地面层
⚠️ Gap 预留原则：相邻层级之间预留 5 ~ 10 的 Gap（如 90、85、80…），便于后续插入新部件层。自定义部件只能使用两个标准层之间的空位值（如 Z=87 介于前发和帽子之间）。

✅❌ 5. 图文红黑榜

5.1 关节 Padding 穿模问题

❌ Don't — 腰线无 Padding，上下装硬接

[此处插入图片：角色弯腰动画帧，上装与下装之间出现 2~5px 的肉色/透明缝隙，腰部素体完全空白未绘制]

引擎后果：角色做弯腰、跑步等动画时，上装和下装之间暴露空白间隙（黑缝/透明缝），玩家看到角色"腰部断裂"。在低端设备上由于浮点精度误差，缝隙更明显。

根因：素体腰部区域未绘制皮肤纹理，上装和下装的边缘像素刚好对齐，没有预留重叠区域。

✅ Do — 腰线 16px Padding，互相覆盖

[此处插入图片：同一弯腰动画帧，上装底部延伸 16px 进入下装区域，下装顶部延伸 16px 进入上装区域，素体腰部完整绘制皮肤。无论动画如何形变都无缝隙]

正确做法：

① 素体的腰部安全区完整绘制皮肤纹理（即使 100% 被服装遮挡也要画）

② 上装底部向下延伸 16px 超过腰线

③ 下装顶部向上延伸 16px 超过腰线

④ 上下装在腰线处形成 32px 的重叠区域，彻底杜绝穿模

5.2 图层 Z-Order 错乱

❌ Don't — 前发与帽子 Z-Order 冲突

[此处插入图片：角色戴帽子时，刘海穿透帽子檐部显示在帽子前方，形成诡异的"头发在帽子外面"效果]

引擎后果：前发和帽子使用了相同的 Z 值（如都是 Z=90），渲染引擎的排序不确定（Z-Fighting），在不同帧出现闪烁——前发时而在帽子前方，时而在后方，产生"z-fighting 闪烁"。

根因：美术自行定义 Z-Order，未遵守全局层级表，导致两个部件争抢同一层级。

✅ Do — 严格遵守全局 Z-Order 层级表

[此处插入图片：同一帽子场景，帽子 Z=85 正确渲染在前发 Z=90 下方（帽子在头发后面），或帽子设计为 Z=92 压在前发上方，效果清晰无闪烁]

正确做法：

① 所有部件严格使用第 3 章层级表的 Z 值

② 帽子固定 Z=85，前发固定 Z=90

③ 如果设计上帽子需要在前发之上，通过"帽子前片"子层级（Z=92）解决，禁止修改基础层级

④ 新增部件使用 Gap 区间，提交 TA 审批后写入层级表

5.3 部件画布尺寸不一致

❌ Don't — 各部件裁切为不同尺寸单独导出

[此处插入图片：头发图 256×128、上装图 512×384、鞋子图 128×96——三个部件尺寸各异，在引擎中拼合时位置全部偏移]

引擎后果：每个部件的锚点坐标不一致，引擎拼合时需要额外的偏移矩阵校正。一旦某个部件的偏移值出错，该部件在角色身上的位置完全错乱。而且不同尺寸的贴图无法合入同一 Atlas。

✅ Do — 所有部件统一画布尺寸导出

[此处插入图片：所有部件都是 1024×1024 的画布，部件内容在对应位置绘制，其余区域为透明。叠加后完美对齐]

正确做法：所有部件共享素体画布尺寸（如统一 1024×1024）。部件在画布中的位置与素体像素级对齐。引擎通过 Alpha 通道裁切透明区域，无需任何手动偏移。

❓ 7. 常见问题排查

💬 Q1：部件在引擎中位置偏移了怎么办？

症状：帽子飘在角色头顶上方 / 鞋子沉到地面以下。

排查步骤：

① 检查该部件的 PNG 画布尺寸是否等于素体画布尺寸。这是 90% 偏移问题的原因——美术手动裁切了透明区域。

② 在 Photoshop 中打开素体和该部件，使用"图像 → 画布大小"对比，必须完全一致。

③ 如果尺寸一致但仍偏移，检查引擎中的 Pivot / Anchor Point 是否被手动修改过。

修复：重新以素体画布导出部件，确保画布左上角坐标 (0,0) 对齐。

💬 Q2：角色做动画时关节处出现缝隙/穿模？

症状：跑步时腰部闪出肤色间隙 / 弯臂时肘部断裂。

排查步骤：

① 打开 PSD，选中上装图层，检查底部边缘是否延伸超过腰线 ≥ 16px。

② 打开素体图层，确认腰部安全区有完整皮肤纹理（而非空白/透明）。

③ 在引擎中播放目标动画，逐帧检查最大形变帧是否仍被 Padding 覆盖。

修复：增大对应关节的 Padding 值，并确保素体安全区无遗漏。

💬 Q3：角色贴图在手机上模糊/发虚？

症状：PC 上清晰的角色贴图，在手机上边缘模糊、颜色偏暗。

排查步骤：

① 检查贴图在引擎中的 Texture Group 是否设为 UI（2D 角色应使用 UI 或 Character 组，避免被全局 LOD 降级）。

② 检查 Compression Settings：ASTC 6×6 是质量和性能的最佳平衡；如果被设为 ASTC 8×8 或更大块会模糊。

③ 确认贴图尺寸为 POT——非 POT 贴图在部分 GPU 上会被自动缩放。

④ 检查引擎的 Texture Streaming 是否将该贴图降级了 Mip Level。

修复：确保 POT 尺寸 + 正确的 Texture Group + 适当的 ASTC 块大小。
