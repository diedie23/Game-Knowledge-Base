配置项推荐值说明
Tile Size256 × 256每个 VT Tile 的纹素分辨率
Tile Count根据地形面积计算覆盖完整地形

💡 关键优势：RVT 将地形多层混合材质的 Pixel Shader 开销从每帧计算变为"按需烘焙到虚拟纹理"，Draw Call 降为 1（整个地形），对大世界场景至关重要。

9️⃣ Do/Don't 红黑榜

✅ Do — 正确做法

• 使用程序化工具生成地形基底（World Machine/Gaea）
• 地表材质层数 ≤ 12 层（移动端 ≤ 4）
• 草地使用 Landscape Grass Type 自动管理
• 植被使用 HISM（Hierarchical Instancing）
• 开启 RVT 减少地形 Draw Call
• 地形边缘平滑过渡到不可见区域
• 预研期锁定地形精度和材质层数

❌ Don't — 错误做法

• 手动逐株放置草地（效率极低 + 无法管理）
• 地形材质超过 12 层（Shader 爆炸）
• 使用 8-bit 高度图（台阶感明显）
• 同屏植被实例超过 50K（GPU 过载）
• 大树启用 WPO（顶点位移开销大）
• 移动端使用实时水面反射
• 忽略地形 Texture Streaming 配置

🔟 常见问题 FAQ

❓ 地形材质超过 4 层后移动端性能骤降？
这是因为每增加一层 Landscape Layer，Pixel Shader 复杂度近乎线性增长。移动端建议：① 使用 RVT 将多层混合预烘焙到虚拟纹理（Shader 变 1 次采样）；② 使用 Texture Array 减少 Sampler 数；③ 远距离自动切换为 Base Map（单张预混合贴图）。
❓ 程序化植被和手动放置应如何配比？
80% 程序化 + 20% 手动是推荐比例。草地、小灌木、碎石 100% 程序化；中型树按密度规则程序化后手动微调；大型标志性树和 Hero 植被手动放置。
❓ Nanite 能用于植被吗？
UE5.3+ 开始实验性支持 Nanite 植被，但有限制：① WPO 不完全支持（风摆效果受限）；② 半透明叶片不支持（需用 Masked + Dithered 替代 Alpha Blend）；③ 推荐仅对岩石类静态物体使用 Nanite，树木暂用传统 LOD + HISM。

📊 11. PM视角：地形植被质量度量看板

地形和植被是场景中面积最大、资产量最多的部分，PM 需要量化监控其制作进度、性能达标情况和资产健康度。
