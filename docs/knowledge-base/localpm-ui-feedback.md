## 详细描述

### 🔴 问题 1：任务名称列宽度不足 & 标题字号偏小

**现状**：
- 甘特图左侧"任务名称"列的可用宽度较窄，长任务名（如"【UGC小游戏】【外围系统】【公告】公告系统"）无法完整显示
- 列标题"任务名称"的字号偏小，层级感不足

**期望**：
- 将任务名称列宽度扩展至甘特图左侧约 **40%-45%** 的面板宽度（当前约为 30%），确保 3 级层级的任务名可完整展示
- 列标题"任务名称"字号至少提升到 **16px / 1rem**（建议加粗），与下方内容形成明确的视觉层级

**建议方案**：
```
优化前：任务名称列 ≈ 360px，标题 12px
优化后：任务名称列 ≈ 500-560px（可拖拽调整），标题 16px bold
```

- 支持列宽**拖拽自适应**（用户可手动调整左右分割比例）
- 长标题支持 **hover tooltip** 显示完整内容
- 考虑增加**文字折行**选项（非截断省略）

---

### 🔴 问题 2：团队成员面板 — 名字截断 & 颜色点含义不明

**现状**：
- 左侧团队成员列表中，部分成员姓名被截断显示（如"张…"、"刘心…"、"陈浩…"），无法辨别具体是谁
- 每个成员名字右侧有不同颜色的小圆点（蓝、紫、粉等），但没有任何图例说明这些颜色代表什么含义

**期望**：
- 姓名应完整显示，至少显示 **4 个汉字**（覆盖绝大部分中文姓名）
- 颜色点必须有明确的**图例/Tooltip 说明**

**建议方案**：

#### 2.1 名字显示优化
| 方案 | 说明 |
|------|------|
| A. 扩宽面板 | 将成员面板最小宽度从当前 ~120px 调整到 160-180px |
| B. 折叠头像 | 头像缩小为 32x32，为文字腾出空间 |
| C. Tooltip | 截断时 hover 显示完整名字 + 职位 |

#### 2.2 颜色点说明
- 在面板顶部增加**颜色图例**（Legend），明确每种颜色的含义
- 可能含义建议（需确认）：
  - 🔵 在线 / 🟢 空闲 / 🟡 忙碌 / 🔴 离线
  - 或：不同工种/角色的颜色标识（UX设计 / UI设计 / Layout 等）
- 鼠标悬停在颜色点上时显示 Tooltip 解释

---

### 🔴 问题 3："AI建议"按钮点击无响应（Bug）

**现状**：
- 界面右上角的"🤖 AI 建议"按钮点击后完全没有反应，无加载状态、无弹窗、无报错
- 用户无法判断是功能未实现还是接口异常

**复现步骤**：
1. 打开甘特图视图
2. 点击右上角"AI 建议"按钮
3. 无任何响应

**期望**：
- 如果功能**已实现**：修复 Bug，确保点击后有响应（loading → 结果展示）
- 如果功能**未实现**：按钮应显示为 **disabled 状态**，并 Tooltip 提示"即将上线"
- 建议：点击后至少有 **loading 状态动画**，避免用户反复点击

**建议补充**：
- AI 建议如果是基于当前排期数据生成的，建议在弹出面板中展示：
  - 📊 资源负载均衡建议
  - ⚠️ 延期风险预警
  - 🔄 任务重新排期建议
  - 📈 团队效率分析

### 🟡 问题 4：用户头像样式单一，缺乏多样性

**现状**：
- 所有用户头像都是相同的圆角矩形 + 纯色背景 + 2 字缩写
- 不同用户之间缺乏视觉区分度，特别是当团队成员较多时容易混淆

**期望**：
- 提供多种头像样式以增强辨识度和个性化

**建议方案**：

| 样式方案 | 说明 | 实现难度 |
|----------|------|----------|
| 渐变背景色 | 为每个用户自动生成独特的渐变色 | 低 |
| 自定义头像上传 | 允许用户上传个人照片 | 中 |
| 预设图标库 | 提供 20+ 种预设图标/表情供选择 | 中 |
| AI 生成头像 | 基于用户名生成独特的像素艺术头像 | 高 |
| 工种图标 | 不同工种显示不同的职业图标（如画笔/代码/设计等） | 低 |

**推荐优先实现**：
1. ✅ **渐变背景色**（每个用户的背景色使用 HSL 色环分配，保证色差足够）
2. ✅ **工种图标角标**（右下角叠加一个小的工种标识）
3. 🔜 自定义头像上传

---

### 🔴 问题 5：已完成任务颜色辨识度差 + 状态变更交互复杂

**现状**：
- 甘特图中"已完成"状态的任务条颜色与其他状态视觉差异不明显
- 底部资源负载条中的数字（1, 3, 3, 2, 2）颜色区分度不够
- 更改任务状态的交互步骤过多（需要多次点击才能完成状态变更）

**期望**：

#### 5.1 颜色优化
| 状态 | 建议颜色 | 说明 |
|------|----------|------|
| 未开始 | `#6B7280`（灰色） | 低饱和度，表示未激活 |
| 进行中 | `#3B82F6`（蓝色） | 高饱和度，吸引注意力 |
| 已完成 | `#10B981`（绿色）✅ | **应改为绿色**，表示成功完成 |
| 延期 | `#F59E0B`（橙色） | 警告色 |
| 阻塞 | `#EF4444`（红色） | 危险色 |

#### 5.2 状态变更交互优化
| 优化方案 | 说明 | 减少步骤 |
|----------|------|----------|
| **右键快捷菜单** | 在甘特条上右键 → 直接选择目标状态 | 3→2 步 |
| **拖拽变更** | 拖拽到完成区域自动标记完成 | 3→1 步 |
| **单击切换** | 点击状态图标直接在预设顺序间切换（未开始→进行中→已完成） | 3→1 步 |
| **批量操作** | 框选多个任务 → 一次性变更状态 | N×3→2 步 |

**推荐最佳实践**：
- 实现**点击状态 badge 直接切换**（参考 Linear、Notion 的交互方式）
- 甘特条上增加**快捷操作按钮**（hover 时显示）
- 增加**键盘快捷键**（如 `1=未开始, 2=进行中, 3=已完成`）
