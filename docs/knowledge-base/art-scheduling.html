🎯 1. 美术排期的特殊性

1.1 与程序排期的核心差异
维度💻 程序排期🎨 美术排期
任务可量化程度功能点明确，可拆解为函数/模块视觉效果主观性强，"好看"难量化
迭代特征Bug 修复有明确终止条件创意迭代无明确收敛点，存在反复修改
依赖关系模块间依赖相对清晰强依赖策划需求冻结、引擎支持、参考图确认
并行度前后端可并行同一角色的 原画→建模→贴图→绑定 串行依赖
风险来源技术难题、性能瓶颈需求变更、审美分歧、外包质量波动

✅ Do — 充分考虑创意迭代不确定性

在工期评估中预留 20%~30% 探索 Buffer，将"风格确认"作为前置里程碑。

❌ Don't — 把美术排期当程序排期来做

按照"功能点拆解"的思路拆美术任务，忽略审美主观性和串行依赖。

📐 排期计算公式

实际排期 = 基准人天 × 复杂度系数 × 团队校准系数 + 审核时间(2~3d) + 外部依赖等待

复杂度系数：S 级 = 1.5x ｜ A 级 = 1.0x ｜ B 级 = 0.8x ｜ C 级 = 0.6x
团队校准系数：基于上 3 个 Sprint 的「预估/实际」比值（见下一节详解）

🔴 核心红线：禁止对所有资产使用统一人天评估。S 级角色 vs C 级角色的工期差异可达 6~8 倍，统一评估 = 必然爆雷。

🎯 2C. 排期校准方法论

💡 核心理念：排期不是「拍脑袋」，而是基于历史数据持续校准的过程。每个团队都有自己的「速度常数」，找到它就能大幅提升排期准确率。
2C.1 速度系数计算法

📐 团队速度系数（Velocity Factor）

VF = 实际耗时总和 ÷ 预估耗时总和（取最近 3 个 Sprint 的均值）

解读：
• VF = 1.0 → 预估完美准确
• VF = 1.3 → 团队实际耗时比预估多 30%（你的预估偏乐观，下次排期应 ×1.3）
• VF = 0.8 → 团队实际耗时比预估少 20%（预估偏保守，可适当压缩）

💡 资深 APM 经验分享

经过 3~4 个 Sprint 的校准后，团队排期准确率通常能从 60% → 85%+。关键在于：
① 分工种统计 — 不同工种的 VF 差异巨大，不可混用
② 分等级统计 — S 级资产的 VF 往往比 C 级高 30~50%（S 级更难预估）
③ 趋势追踪 — VF 如果逐月递减，说明团队在成长；如果递增，说明有系统性问题

⚡ 3B. 关键路径优化策略

💡 核心问题：角色管线 120 天的串行依赖是大多数项目的瓶颈。以下 5 种策略可在不牺牲质量的前提下缩短关键路径 20~40%。
3B.1 五大优化策略一览
策略原理节省时间适用场景风险
🔀 分批启动S 级角色提前 2 周启动，与预研期并行~15d所有项目低（风格已锁定即可）
✂️ 阶段重叠建模完成 80% 时启动贴图（而非等 100%）~10d团队配合默契时中（需要沟通成本）
📦 模块化拆分头/身/武器拆分给不同建模师并行~12dS/A 级复杂角色中（接缝匹配）
🤖 工具加速AI 辅助原画变体 / 自动拓扑 / 批量贴图~8d批量同类资产低（需 TA 支持）
🏭 外包前置B/C 级角色全部外包，内部专注 S/A 级~20d有成熟 CP 资源中高（质量管控）
3B.2 策略①详解：分批启动时间线

🔀 分批启动 — 让 S 级角色「跑在前面」

传统方式：等风格定稿 → 所有角色统一启动原画 → 串行到底
优化方式：预研期 1~2 周即可锁定 S 级角色风格 → 提前启动 S 级原画 → 与预研其他工作并行

角色等级启动时机说明
S 级（3~5 个）预研中期（风格锁定后）提前 2~3 周启动，不等全部风格定稿会
A 级（8~10 个）预研结束（风格正式定稿）第一批启动
B/C 级（其余）Alpha 前 4 周A 级进入建模后，释放原画产能
3B.3 策略②详解：阶段重叠法

✂️ 阶段重叠 — 80% 完成即可触发下游

原理：下游工序并不需要等上游 100% 完成。例如建模完成 80%（大型已定，细节微调中）时，贴图师就可以开始 UV 展开和基础色块。

重叠节点上游完成度下游可启动内容注意事项
原画 → 建模70%（三视图 + 色板确认）开始搭建大型细节可能微调
建模 → 贴图80%（拓扑定版 + UV 初版）开始铺色块 + 材质测试UV 若有修改需返工
贴图 → 绑定90%（基础贴图完成）开始骨骼绑定几乎无风险
⚠️ 重叠法风险管控：① 重叠节点需双方口头确认「当前状态是否足够下游启动」② 返工概率 ~15%，但节省的时间远超返工成本 ③ 仅推荐团队内协作，不适用于外包（沟通链路长，返工成本高）。
3B.4 优化前后对比
场景传统串行优化后节省
S 级角色（单体）~60d~42d（分批+重叠+模块化）-30%
20 个角色（团队并行）~120d（关键路径）~85d（分批+外包前置）-29%

🔴 核心红线：关键路径优化不等于「压缩工期」。优化 = 消除等待浪费 + 提高并行度。任何优化方案的前提是不降低质量标准。如果质量下降了，说明优化过头了。

形态📡 信号🔍 诊断🎯 行动

📉 实际线中段断崖下跌批量关闭任务可能是虚假进度（批量标完成）抽查验收质量
🚨 案例：燃尽图断崖式下跌
Sprint 最后两天，燃尽图从 60% 剩余直接跌到 0%，但 Review 时发现多数资产质量不达标。
根因：团队成员在 Sprint 末期批量标记任务完成以"凑进度"，缺乏每日更新进度的规范约束。
破局：① 实施抽查机制，APM 随机抽取 20% 已完成任务进行验收 ② 将"每日站会更新进度"纳入团队规范

🅲️ 接受延期质量不可妥协里程碑推后，影响上线计划

🔴 核心红线：红色预警建议在 4 小时内上报制作人/总监，禁止 APM 独自决策。

🚨 案例：红色预警处理不及时
APM 发现进度偏差已超 30%，但抱有"赶一赶能追回来"的侥幸心理，延迟 3 天才上报，导致错失最佳补救窗口。
破局：① 建立自动化预警机制（看板工具自动检测进度偏差 > 25% 时触发通知） ② 将"4h 上报"写入团队章程，延迟上报纳入绩效考核
5.4 量化预警指标体系

📐 进度健康度公式（每日自动计算）

健康度 = (已完成任务点数 ÷ 理论应完成点数) × 100%

理论应完成 = Sprint 总点数 × (已过天数 ÷ Sprint 总天数)
健康度 ≥ 90% = 🟢 绿色 ｜ 75%~89% = 🟡 黄色 ｜ < 75% = 🔴 红色

指标名计算公式阈值数据来源
Sprint 燃尽偏差率(实际剩余 - 理想剩余) ÷ Sprint 总量 × 100%>15% 黄色 / >25% 红色TAPD 看板
里程碑完成率已交付资产数 ÷ 里程碑目标数 × 100%里程碑前 3 天 < 80% = 红色资产管理系统
阻塞任务占比被阻塞任务数 ÷ 进行中任务数 × 100%>20% = 黄色 / >40% = 红色每日站会记录
外包延期率延期交付批次 ÷ 应交付批次 × 100%>30% = 红色（考虑更换 CP）外包交付记录
需求变更频率本 Sprint 新增/变更需求数>3 条 = 黄色 / >5 条 = 红色变更申请单
💡 APM 周报模板：每周五下午用以上 5 个指标生成一页「排期健康度仪表盘」，发送给制作人和主美。数据驱动的汇报比感性描述更有说服力。

🔄 5B. 多版本并行排期（长线运营）

💡 适用场景：项目进入运营期后，通常 2~3 个版本并行推进（当前版本收尾 + 下版本量产 + 下下版本预研）。如何管理多条时间线并行不打架，是资深 APM 的核心能力。

5B.3 多版本并行的排期要诀

📋 五条黄金法则

  • 人力不跨三线 — 单人同时只参与 ≤ 2 条线，否则上下文切换成本爆炸（效率降低 30~40%）
  • 明确优先级 — 当多线冲突时：Line A Bug Fix > Line B 里程碑资产 > Line C 探索任务
  • Buffer 独立 — 每条线的 Buffer 独立计算，禁止「借用 C 线 Buffer 补 B 线」
  • 切换需公告 — 人力从一条线切换到另一条线，APM 必须提前 3 天通知 + 更新排期表
  • 预研产出限时 — Line C 的预研探索设置明确截止日（如 2 周），到期无论结果如何都进入决策
⚠️ 常见踩坑：制作人临时从 Line B 抽调 3 人去支援 Line A 紧急 Bug，导致 Line B 里程碑延期。
正确做法：① 抽调前评估 Line B 影响 ② 如抽调 > 2 天，Line B 排期必须同步调整 ③ 被抽调人员明确回归日期

💣 7. 常见问题与避坑指南

7.1 排期十大雷区
🚨 避坑指南：以下是 APM 排期中最常见的 10 个致命错误，每一条都可能导致里程碑 miss。
#❌ 雷区✅ 正确做法
1不给 Buffer — 预估工期 = 实际排期应加 15~20% Buffer
2忽略串行依赖 — 原画没定稿就排建模严格按依赖链排序
3不区分复杂度 — 所有角色统一工期按 S/A/B/C 分级评估
4需求未冻结就排期 — 策划还在改就开始做等需求 Sign-off 后再排期
5低估沟通成本 — 外包沟通每天额外 0.5~1h在排期中预留沟通时间
6不跟踪中间产出 — 只看最终交付设置中间检查节点
7过度乐观 — 总觉得"应该能赶上"基于历史数据评估
8缺少兜底方案 — 没有 Plan B每个关键路径准备备选方案
9忽略审核时间 — 主美审核/QA走查需要时间预留 2~3 天 审核周期
10不做复盘 — Sprint 结束不回顾排期准确率每个 Sprint 回顾 + 校准系数
7.2 APM 排期黄金法则
💡 排期公式：排期 = 预估工时 × 1.2（Buffer 系数）+ 审核时间 + 外部依赖等待时间
💬 Q1：美术同学总觉得排期太紧怎么办？

答：① 让美术参与排期（自己评估的工期更有承诺感） ② 用历史数据证明预估准确性 ③ 区分复杂度——S 级资产给足时间，C 级资产快速周转。

💬 Q2：策划频繁改需求导致排期失控怎么办？

答：① 推动需求冻结机制——Alpha 后原画需求不再变更 ② 每次变更必须走变更评审流程，评估对排期的影响 ③ 用数据说话——统计需求变更导致的额外工期，定期向管理层汇报。

💬 Q3：外包延期了怎么补救？

答：① 预留 20% Buffer 应对延期 ② 同类资源准备备选 CP ③ 延期 1~3 天口头沟通，3~7 天书面警告，>7 天启动备选 CP + 评估合同违约。详见《外包全流程管理指南》。
