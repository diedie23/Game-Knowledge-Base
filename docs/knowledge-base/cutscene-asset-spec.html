📋 1. 概述与适用范围

本规范定义了游戏项目中所有 过场演出（Cutscene / Cinematic） 资产的制作标准、交接流程、版本管理和质量门禁，覆盖实时演出（引擎内 Sequencer）与离线预渲染（CG/PV）两大类型。

适用场景

场景类型引擎典型案例
主线剧情过场UE5 Sequencer / Unity Timeline章节转场、BOSS 登场、结局动画
角色必杀技演出引擎实时大招释放 → 特写镜头 → 慢动作回放
抽卡/获得演出引擎实时 + 后期合成SSR 角色抽取 → 全屏演出 → 立绘展示
PV / CG离线渲染（Maya/Blender → AE/Nuke）宣传 PV、版本更新 CG
剧情对话演出引擎内轻量 SequenceNPC 对话 → 表情/动作切换 → 镜头微运动
过场转场（Loading）引擎实时 / 预渲视频地图加载过渡动画
💡 核心原则：演出是玩家情感体验的"放大镜"——所有规范的终极目标是保障叙事节奏感和技术稳定性的统一。

📋 5. 分镜文件交接 SOP

5.1 分镜文件规范

要素必填说明
镜头编号✅按时间序编号，如 Shot_001 ~ Shot_N
景别标注✅ECU / CU / MS / FS / LS / ELS（参见第7节）
运镜方式✅固定 / 推 / 拉 / 摇 / 移 / 跟 / 升降 / 环绕
时长标注✅每镜时长（秒），总计时长
台词/旁白✅对应镜头的台词文本，标注语种
表情/动作描述✅角色情绪关键词 + 参考动作
音效/BGM 指示⬜建议使用的音效类型或 BGM 段落
特效指示⬜需要配合的粒子/光效/屏幕后处理
转场方式✅硬切 / 溶解 / 淡入淡出 / 划 / 自定义
参考素材链接⬜动画参考视频 / 电影片段 URL

5.2 交接流程

步骤负责人动作产出时限
1策划/导演提交分镜需求单到 TAPD需求单（含脚本 + 演出意图）—
2分镜师绘制分镜板SB 文件 + PDF 导出3 工作日/分钟
3导演 + 策划分镜评审会评审记录 + 修改意见评审后 1 日内反馈
4分镜师修改并锁定版本终版 SB（版本号标记 Final）2 工作日
5演出 TA创建引擎 Layout Sequence粗动 + 相机预设2 工作日/分钟
6全组Layout 审查（对照分镜板）确认节奏 → 进入精调审查当日
✅ 交接核心原则：分镜板是全组"合约"——未通过评审的分镜不得进入引擎制作阶段。任何分镜修改必须走变更流程。

🏗️ 6. Level Sequence 组织架构

6.1 Master / Sub-Sequence 分层设计

CUT_MS_C01_001_Master (Master Sequence)
│
├── Subscenes Track ─────────────────────────
│   ├── CUT_MS_C01_001_Cam      # 相机轨道（所有镜头切换/运镜）
│   ├── CUT_MS_C01_001_Anim     # 角色动画轨道（骨骼动画 + IK）
│   ├── CUT_MS_C01_001_FX       # 特效轨道（Niagara + 后处理）
│   ├── CUT_MS_C01_001_Audio    # 音频轨道（Wwise Event + Attenuation）
│   └── CUT_MS_C01_001_Light    # 灯光轨道（灯光动画 + 渐变）
│
├── Event Track ─────────────────────────────
│   ├── GameplayEvent_Start      # 演出开始事件（禁用输入/HUD隐藏）
│   ├── GameplayEvent_End        # 演出结束事件（恢复控制/触发后续逻辑）
│   └── SkipPoint_Markers[]      # 可跳过检查点标记
│
└── Fade Track ──────────────────────────────
    ├── FadeIn (0~0.5s)          # 入场淡入
    └── FadeOut (end-0.5s~end)   # 结束淡出

6.2 分层职责与锁定规则

子 Sequence负责人锁定策略说明
_Cam演出导演 / Layout 师Layout 完成后锁定镜头切换、FOV、运镜曲线
_Anim动画师精调完成后锁定角色骨骼动画、Additive 层
_FX特效师Polish 完成后锁定粒子系统、后处理体积
_Audio音频设计师音频集成完成后锁定Wwise Event 绑定、音量曲线
_Light灯光师 / TA与 FX 同步锁定关键灯光 + 氛围光动画
_Master演出 Lead终验收后锁定统筹时间线、Event Track
🚫 禁止行为：在他人锁定的子 Sequence 中直接修改——必须通过 TAPD 提解锁申请或在 Perforce 中请求锁文件转移。

6.3 Shot Section 规范

每个 Master Sequence 中使用 Shot Section 划分镜头，每个 Shot 对应分镜板一帧/一组：

规则说明
Shot 命名Shot_{3位序号}_{景别缩写}，如 Shot_001_CU
过渡帧相邻 Shot 重叠 2~4 帧用于转场混合
标记点每个 Shot 起始帧添加 Marker（颜色按景别区分）
空镜处理无角色的纯环境镜头仍需保留 Shot 容器

8.3 性能优化 Checklist

  • 演出场景使用独立 Streaming Level，演出结束后立即 Unload
  • 角色仅保留演出所需骨骼（非全身 Full-body 时剔除下半身）
  • 粒子系统设置 MaxCount + Auto-Deactivate
  • 相机视锥外物体启用 Visibility Culling
  • 光照烘焙 LightMap 用于静态物体，动态光仅照关键角色
  • 后处理体积限定在 Sequence 时间范围内激活
  • 音频资源使用 Stream 模式加载（非 Resident）
  • 低端机降级方案：关闭 DOF / Motion Blur / Volumetric Fog

🔐 9. 多人协作与锁规范

9.1 Perforce Workspace 规则

规则说明
独占锁所有 .uasset（Sequence / AnimSequence）默认使用 Exclusive Checkout

✅ 11. 质量验收标准

11.1 视觉质量评分卡

维度权重评分标准（1~5 分）不合格阈值
镜头节奏25%镜头时长合理、切换流畅、节奏感强<3 分
角色表演25%动作自然、表情到位、无穿模/滑步<3 分
光影氛围20%光照层次分明、色调统一、无漏光<3 分
特效/音画同步15%音效同步、特效节奏匹配、无延迟<3 分
技术稳定性15%无闪退/掉帧/加载卡顿/黑屏<4 分（一票否决）
🎯 门禁线：加权总分 ≥ 3.5 分方可通过，任意维度 < 不合格阈值则整体驳回。

11.2 常见驳回原因 Top 10

#问题类别修复难度
1角色穿模（手/武器穿入身体/场景）动画中
2镜头抖动/急停（曲线关键帧异常）相机低
3音画不同步（音效延迟 >3帧）音频低
4低端机掉帧到 <20fps性能高
5转场黑屏/闪烁技术中
6角色表情僵硬/嘴型不匹配动画中
7灯光/阴影穿帮（浮空阴影/漏光）光照低
8字幕超出安全区 / 时序错乱UI低
9跳过功能失效（按钮无响应/逻辑不触发）逻辑中
10演出后游戏逻辑异常（操控恢复失败）逻辑高

💣 12. 常见问题与踩坑记录

Case 1：Master Sequence 丢失引用

现象：提交后其他人 Sync 发现 Master Sequence 中子 Sequence 引用断裂。
根因：子 Sequence 文件被移动到新路径，但 Master 中的 Soft Reference 未更新。
解法：① 移动资产前先在 Master 中 Remove Track → 移动文件 → 重新 Add Track；② 使用 Fix Redirectors 命令修复残留重定向器。

Case 2：低端机演出崩溃

现象：某必杀技演出在 Mali-G57 GPU 设备上 100% 崩溃。
根因：粒子 Ribbon 的材质使用了 4 层 Texture Sample，超出低端 Shader Model 限制。
解法：① 建立低配材质变体（≤ 2 层采样）；② 在 Device Profile 中设置粒子 Scalability 自动降级。

Case 3：音画同步问题

现象：对话演出中嘴型与配音不匹配，延迟约 5~8 帧。
根因：Audio Track 中 Wwise Event 的 Pre-Roll 未与 Animation Notify 对齐。
解法：① 在 Audio Sequence 中使用 Section Start Offset 预偏移；② 统一规定音频 Event 触发点为嘴型动画起始帧 -2 帧。

Case 4：多平台字幕安全区

现象：iPad Pro 上字幕被刘海/Home Indicator 遮挡。
根因：字幕 Widget 未考虑 Safe Area Padding。
解法：字幕 Widget 内嵌 SafeZone 节点，底部预留 ≥ 5% 屏幕高度。

Case 5：Perforce 锁死无法提交

现象：动画师 Checkout 了 _Anim 子 Sequence 后请假，其他人无法修改。
解法：① Admin 强制解锁 + Shelve 该用户未提交变更；② 流程改进：设置 4 小时锁定超时自动通知机制。
