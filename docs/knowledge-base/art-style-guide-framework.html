🎯 1. 风格指南的战略价值

"风格指南不是'限制创意的牢笼'，而是'让 50 个人画出来像 1 个人画的'工具。没有风格指南的项目，最终会变成'每个美术都在画自己心中的游戏'。"

— 某百人美术团队制作人

1.1 为什么需要风格指南

维度有风格指南无风格指南
视觉一致性玩家感受到统一的世界观和品质感不同角色/场景风格割裂，像"拼贴画"
团队协作效率新人 3 天上手出图，返工率 ≤10%反复口头沟通，返工率 30%~50%
外包管控CP 有明确标准可参照，验收有据可依验收全凭主观感受，甲乙双方扯皮不断
品牌识别度游戏具备独特的视觉 DNA，截图即可辨认泯然众人，无法建立视觉记忆点
迭代可控新版本/新角色自然延续已有风格每次新内容都可能"画风突变"

⚓ 3. 风格锚点体系（Style Anchor System）

"风格锚点就是团队的'北极星'——当任何人对'这个对不对'产生疑问时，只需要拿出锚点图对比，答案立刻清晰。"

— 美术总监经验

3.1 锚点类型与层级

锚点层级类型数量作用谁来定
L0 总锚世界观 Key Visual1~2 张定义整个游戏的视觉调性和情绪基调美术总监 + 制作人
L1 类型锚角色锚/场景锚/UI锚每类 2~3 张定义该类型资产的风格标准主美
L2 子类锚各阵营/种族/环境的风格差异样板每子类 1~2 张在统一大风格下展现差异化组长级美术

3.2 锚点图的制作要求

  • 完成度：锚点图必须是完全精修的高品质稿，不能是草图或概念探索稿
  • 标注：每张锚点图需附带设计说明文档（解释为什么这样设计、哪些元素是"必须保持"的、哪些可以灵活变化）
  • 覆盖性：锚点图应尽可能覆盖代表性场景（如角色锚要覆盖不同体型/性别/阵营）
  • 边界样本：除了"标准答案"，还应提供边界案例——"这是能接受的极限" vs "这已经超出风格范围"

3.3 锚点图对比使用法

场景使用方法判定标准
美术自检制作完成后将成品与锚点图并排对比色调/比例/材质表达是否在同一视觉语言体系内
评审会投屏时锚点图常驻在侧边作为参照偏离锚点需说明设计意图并获得主美确认
外包指导锚点图作为外包合同附件的核心参考CP 交付物与锚点图的风格偏差度打分

🌈 4. 色彩体系设计

4.1 色彩体系的四层结构

层级内容示例决策者
L1 基调色定义整体色温倾向和明暗基调"暖色调偏冷灰" / "高饱和明快" / "去饱和厚重感"美术总监
L2 主题色各阵营/系统/场景的标识色火系=暖红橙、冰系=冷蓝青、暗系=紫黑主美 + 策划
L3 调色板具体资产可使用的颜色范围每个阵营 5~8 个核心色 + 禁用色主美
L4 色值规范精确的 HSV / Hex 数值和使用比例主色 60% / 辅色 30% / 点缀色 10%美术组长

4.2 色彩约束规则

🎨 色彩规范要点

01饱和度上限：除 UI 图标外，游戏内资产的饱和度 S ≤ 80%（HSV），避免"塑料感"
02黑白极值限制：BaseColor 明度 V 不低于 10%、不高于 95%（留 5% 空间给光照）
💡 对应 sRGB 数值约 30~240，详见 《3D角色PBR规范 §2 数值红线》
03冷暖对比规则：同一画面中冷暖色面积比应控制在 3:7 或 7:3，避免 5:5 平分（无主次）
04禁用色清单：明确列出"绝对不可在该项目中出现的颜色"（如写实军事题材禁止荧光粉）
05品阶区分：不同品质/稀有度角色通过色彩丰富度（而非饱和度爆炸）来区分层次

📐 5. 造型语言与比例规范

5.1 体型比例标准

角色类型头身比风格倾向适用项目类型
写实/次世代7~8 头身真实人体比例3A 大作 / 开放世界
二次元/日系6~7 头身微拉长四肢 + 大眼二次元手游 / ARPG
韩系时尚7~8 头身超模比例 + 精致五官MMO / 时装游戏
Q版/可爱2~3 头身大头小身 + 圆润体型休闲 / 派对游戏
欧美卡通3~5 头身夸张体型差异 + 几何化合家欢 / 冒险

5.2 造型"形状语言"规则

核心概念：所有角色/场景设计都建立在三大基础形状的组合之上——圆形（亲和/柔软）、方形（稳重/力量）、三角形（危险/速度）。风格指南需明确项目的形状语言偏好。
角色定位主导形状造型特征设计要点
英雄/主角方 + 三角肩宽/V字身材/三角轮廓正面剪影需一眼识别
萌系/辅助圆圆润体态/大头/柔和曲线强调亲和力和可爱感
反派/Boss三角 + 尖锐尖角/不对称/压迫性剪影剪影需传递威胁感
NPC/路人简化几何降低细节密度，保持背景感不抢主角视觉焦点

5.3 剪影测试标准

🚨 红线规则：所有角色设计在纯黑剪影状态下必须可辨识。如果两个角色的剪影高度相似（去色后无法区分），则必须回炉重做造型差异。这是角色设计的"生死线"。
  • 辨识度测试：10 个角色缩小到 64px 高度的剪影 → 随机测试者应能正确识别 ≥ 80%
  • 远距测试：游戏实际远景（如团战场景）中角色缩小后仍可辨认
  • 动态剪影：角色标志性动作/待机的运动剪影也需有差异化

💡 6. 材质/光影/渲染风格定义

6.1 渲染风格光谱定位

风格类型光影表达材质细节边缘处理代表作
照片写实 PBR物理准确光照 + GI高精度纹理 + 微表面细节自然过渡TLOU2, RDR2
风格化 PBRPBR + 色彩夸张简化纹理 + 风格化粗糙度微描边原神, 塞尔达
赛璐璐/二次元硬边阴影 2~3 层纯色块 + 渐变明确描边崩铁, 绝区零
手绘/绘本手绘笔触阴影纸质/水彩纹理叠加手绘线条Okami, Hades
低多边形/极简平面光照或无阴影纯色/低噪点硬边几何Monument Valley
💡 关键决策：风格指南中必须明确项目位于这个光谱的哪个位置，以及允许的偏移范围。例如"以风格化 PBR 为主，允许在特效/UI 上偏向赛璐璐方向，但角色/场景不允许出现照片写实质感"。

6.2 光影规则定义

规则项需要定义的内容示例
主光方向全项目统一的光源方向约定"永远从左上 45° 打光"（保证角色立绘一致）
阴影层数阴影分几层、过渡方式"2 层阴影：亮面 → 1 影（软过渡）→ 2 影（硬边）"
阴影色相偏移阴影区域的色温变化方向"暖光源 → 阴影偏冷蓝紫" / "冷光源 → 阴影偏暖"
高光形态高光的形状和面积"角色高光为圆润形、面积控制在 5%以内"
描边/Outline是否有描边、描边宽度和颜色规则"角色 1.5px 深色描边，颜色为暗面色 × 0.3"
AO 强度环境遮蔽的可见度"角色 AO 50% 强度、场景 AO 70% 强度"

6.3 材质表达规则

  • 金属：定义金属的高光清晰度、反射率范围、是否允许环境反射
  • 布料：定义褶皱密度、阴影软硬度、是否有次表面散射
  • 皮肤：定义肤色范围、SSS 强度、高光锐度
  • 头发：定义高光条（Angel Ring）宽度、透明度处理方式
  • 宝石/发光体：定义发光区域大小、Bloom 强度范围

🔒 7. 风格一致性管控机制

"风格一致性不是靠'每张图都让主美看一遍'来保证的——那只能管 5 个人的团队。50 人团队需要制度化的管控机制。"

— 美术管理经验

7.1 四级审核体系

审核层级审核节点审核人关注重点通过率目标
L1 自检提交前美术本人Checklist 逐项自查100% 完成自检
L2 组审提交时组长工艺合规 + 风格对标首次通过率 ≥ 70%
L3 主美审里程碑主美整体风格一致性 + 品质首次通过率 ≥ 85%
L4 制作人审关键节点制作人/美术总监是否符合游戏调性和品牌定位一票否决权

7.2 风格偏差评分机制

评分维度权重评分标准（1~5分）
色调一致性25%与锚点图色温/饱和度的匹配程度
比例正确性20%头身比/面部比例是否在规范范围内
材质表达20%金属/布料/皮肤是否符合项目材质规则
线条/笔触15%描边/笔触是否符合项目美术语言
细节密度10%装饰/纹理密度是否与同品质角色一致
剪影辨识度10%去色后是否可与其他角色区分
⚠️ 打分规则：总分 ≥ 4.0 通过；3.5~4.0 有条件通过（需小改）；< 3.5 打回重做。任一维度 ≤ 2 分则不论总分直接打回。

7.3 风格培训体系

  • 入职培训（Day 1~3）：风格指南精讲 + 临摹锚点图练习
  • 月度分享会：主美带领全组对标最新锚点，分析好/坏案例
  • 外包培训包：每个 CP 签约后发放风格培训视频 + 电子版 Style Guide + 测试稿评审
  • Red Flag 案例库：累积典型的"风格偏离"案例，作为反面教材持续更新

🔄 8. 风格迭代与版本管理

8.1 何时需要迭代风格指南

触发条件迭代范围审批级别
新大版本/新地图上线可能新增 L2 子类锚（新阵营/新区域）主美审批
玩家反馈大量"不协调"排查风格规则是否需要修正制作人决策
技术升级（如引擎换代）渲染风格参数全面重校美术总监 + TA Lead
竞品市场变化评估是否需要风格进化制作人 + 美术总监
每季度定期Review清理过时规则 + 补充新场景规范主美

8.2 风格指南版本号规则

风格指南版本号：Major.Minor
  Major 升级 → 整体视觉方向调整（极少发生，需制作人审批）
  Minor 升级 → 新增子类规范/细化规则/补充案例

示例：
  StyleGuide v1.0 — 预研期初版发布
  StyleGuide v1.1 — 新增"冰系阵营"子类锚点
  StyleGuide v1.2 — 补充"坐骑类"造型比例规范
  StyleGuide v2.0 — 引擎升级后全面重修渲染风格参数
🚨 禁忌：量产中期（已有 50+ 角色上线后）严禁做 Major 版本风格变更！这会导致旧角色与新角色风格割裂，"翻新旧资产"的成本远超想象。如确需变更需制作人/总监双签 + 旧资产翻新排期。

🎯 9. 制作人关注：风格管控常见翻车与应对

9.1 Top 5 风格翻车场景

#翻车场景根因避坑策略
1"画风不统一"被玩家群嘲
不同角色明显像不同画师画的		预研期未建立风格锚点 / 锚点图质量不足 / 缺乏审核机制严格执行"六步法"+ 四级审核 + 风格偏差评分
2外包交付风格严重偏离
CP 画出来完全不是想要的风格		外包培训包不完善 / 风格指南描述过于抽象交付风格培训包（视频+锚点图+正负面案例）+ 首批测试稿先行评审
3"主美个人风格绑架项目"
主美离职后团队无法维持风格		风格标准停留在主美脑中，未文档化风格指南必须文档化到"换人也能执行"的程度
4新角色品质越来越高，老角色显得寒酸
"品质通胀"导致早期角色被嫌弃		风格指南只定了"下限"没定"上限"明确各品质（N/R/SR/SSR）的细节密度上限和工时预算
5IP 联动角色与自家角色不搭
联动角色插入后像"PS 上去的"		联动美术未经过本项目风格标准化处理所有联动角色必须经过"风格转译"流程，按本项目规则重绘而非直接套用
