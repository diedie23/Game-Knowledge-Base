2.1 光源分类与使用建议
光源类型UE5Unity适用性能开销投射阴影
定向光 (Directional)DirectionalLightDirectional Light太阳/月亮主光低（全局 1 盏）✅ CSM 级联阴影
点光源 (Point)PointLightPoint Light灯泡、火焰中（按距离衰减）⚠️ 按需开启

• 面光源 禁止 在移动端使用（性能不可接受）

4️⃣ LightMap UV 制作规范

⛔ 仅适用于烘焙/混合方案。使用全动态 Lumen 方案时，无需制作 LightMap UV（Lumen 使用 Mesh Card 代替）。
4.1 LightMap 分辨率标准
物体类型LightMap 分辨率参考 TD说明
地板 (4m²)64~16 px/m大面积平整表面
墙体 (标准)64~128~16~32 px/m视距离和重要度
柱体 / 楼梯32~64~8~16 px/m结构件
装饰 / 道具32~8 px/m小物件低分辨率即可
Hero Asset128~256~32~64 px/m视觉焦点物体
地形 (Landscape)由 Landscape 系统管理—不单独设置 LightMap
4.2 LightMap UV 制作规则
规则说明违规后果
UV 不可重叠LightMap UV（Channel 1）所有面片不可重叠光照漏光/黑斑
UV 不可翻转所有面片朝向一致（无 Mirror）阴影方向错误
边缘间距 (Padding)≥ 2 texels（推荐 4 texels）烘焙时光照渗色
UV 利用率≥ 50%（大型物件 ≥ 60%）浪费 LightMap 空间
UV 岛数量尽量少（Flatten → Pack）过多碎片影响烘焙质量
UV ChannelUE5: Channel 1（默认）；Unity: Channel 2引擎找不到 LightMap UV
💡 自动化建议：UE5 可使用 Generate Lightmap UVs 自动生成 Channel 1 UV。但复杂物体建议 DCC 手动制作以获得更高质量。

5️⃣ 光照探针与间接光规范

5.1 光照探针放置策略
场景区域探针密度UE5 方案Unity 方案
室外空旷每 4~8m 一组Volumetric Lightmap（自动）Light Probe Group（手动排布）
室内走廊每 2~4m 一组同上沿走廊方向密集排布
明暗交界处加密至每 1~2m同上过渡区域加密放置
垂直空间每层至少 1 组同上多层楼宇需逐层放置
⛔ 红线：

• 动态角色/物体 必须 通过探针获取间接光，不可使用平面一色的环境光

• Unity Light Probe 总数单场景 ≤ 5000 个（过多影响烘焙和内存）

• 探针组内探针间距 ≤ 角色身高的 2 倍（约 4m），避免光照跳变

5.2 反射探针 (Reflection Probe) 规范
场景反射探针数量分辨率更新模式
高反射区（水面/金属地板）按区域覆盖256~512Baked / Realtime (按需)
普通室内每个房间 1~2 个128~256Baked
室外关键区域 + 远景128Baked + 天空球 Fallback

注：UE5 Lumen 方案下，Screen Space Reflection + Lumen Reflection 可大幅减少对静态反射探针的依赖。

9️⃣ Do/Don't 红黑榜

✅ Do — 正确做法

• 预研期锁定光照方案，写入技术文档
• 使用项目统一的 Tone Mapper 和 LUT
• 动态角色通过探针/Volumetric Lightmap 获取间接光
• 在明暗交界处加密探针密度
• 后处理 Volume 限定有效区域
• LightMap UV 无重叠、无翻转、Padding ≥ 2 texels
• 每次光照调整后 Profile 验证预算

❌ Don't — 错误做法

• 量产中期切换光照方案（全量返工）
• 无限范围后处理 Volume（影响全局）
• 每个灯光师自定义 Tone Mapper（色调不一致）
• 动态角色无间接光（"黑人"现象）
• 移动端使用面光源 / Lumen（性能不可承受）
• LightMap UV 重叠（光照漏光/黑斑）
• 同屏 > 4 盏投射阴影的动态光源

🔟 常见问题 FAQ

❓ Lumen 在移动端能用吗？
截至 UE5.4，Lumen 仅支持 PC / 主机 / 高端安卓（部分 Vulkan 设备实验性支持）。移动端建议使用烘焙 + 动态主光的混合方案。
❓ 烘焙时间太长怎么优化？
① 使用 GPU Lightmass（UE5）替代 CPU Lightmass，速度提升 5~10 倍；② 降低非关键物体 LightMap 分辨率；③ 减少间接光弹射次数（3 次通常足够）；④ 使用 Swarm Agent 分布式烘焙。
❓ 角色在室内外过渡时"闪白/闪黑"怎么办？
这通常是 Auto Exposure 响应速度问题。解决方案：① 缩窄 Min/Max EV 范围；② 使用 Exposure Compensation Volume 在过渡区域预调曝光；③ 增加 Auto Exposure Speed Down 的过渡时间。
❓ 开放世界的昼夜循环用什么 GI 方案？
• 主机/PC → Lumen（原生支持太阳角度实时变化）
• 移动端 → 动态主光 + 预烘焙的多套探针/LUT 插值（按时段切换预制数据）
