## 🧮 算法解析

### 为什么旧版无法区分浅灰和深灰？

旧版使用 **HSV 色相距离** (`M.C.hD`) 作为颜色匹配的核心指标：

```
色相距离 = |H₁ - H₂| (角度差)
```

**问题**：灰色系颜色的**饱和度(S)极低**，此时**色相(H)值是不稳定的**。

| 颜色 | RGB | H | S | V |
|------|-----|---|---|---|
| 浅灰 | (200,200,200) | **0°** | 0% | 78% |
| 深灰 | (80,80,80) | **0°** | 0% | 31% |
| 色相距离 | | **0°** | — | — |

色相差 = 0°，算法认为它们是"同一种颜色"！

### CIELAB ΔE 如何解决？

**CIELAB** 是一个**感知均匀**的色彩空间，其中：
- **L\*** (Lightness)：明度，0（纯黑）→ 100（纯白）
- **a\*** ：绿-红轴
- **b\*** ：蓝-黄轴

```
ΔE(CIE76) = √( (L₁-L₂)² + (a₁-a₂)² + (b₁-b₂)² )
```

| 颜色 | RGB | L* | a* | b* |
|------|-----|----|----|-----|
| 浅灰 | (200,200,200) | **80.6** | 0 | 0 |
| 深灰 | (80,80,80) | **34.0** | 0 | 0 |
| **ΔE** | | **46.6** | — | — |

ΔE = 46.6 → 巨大差异！人眼在 ΔE > 5 时就能清晰分辨。

### v6.2 带明度权重的 ΔE

```
ΔEw = √( (Lw × ΔL)² + Δa² + Δb² )
```

- `Lw = 1.0`：标准 CIE76 公式
- `Lw = 1.5`（默认推荐）：强化明度区分 50%
- `Lw = 2.0-3.0`：极端明度敏感模式

### RGB → CIELAB 转换流程

```
RGB [0-255]
  ↓ 移除 sRGB gamma (逆伽马校正)
线性 RGB [0-1]
  ↓ sRGB → D65 矩阵变换
CIE XYZ
  ↓ 非线性压缩 f(t) = t^(1/3) or 线性段
CIELAB L*a*b*
```

## 📖 使用指南

### 场景一：区分相近灰色

1. 加载含有多种灰度的贴图
2. 确保容差面板选择 **CIELAB ΔE** 模式
3. 将 **全局容差** 调低至 ΔE 5-8（严格匹配）
4. 将 **明度权重** 调高至 2.0-3.0x
5. 点击浅灰区域吸色 → 分配到 R 通道
6. 点击深灰区域吸色 → 分配到 G 通道
7. 画布上会实时高亮当前容差下的匹配区域
8. 确认后点击「生成多通道 Mask」

### 场景二：智能分析含灰度的贴图

1. 开启 **「强制区分明度」** 开关
2. 将 **提取色数** 设为 8 色或更高
3. 点击 **「✨ 智能分析主色调」**
4. 算法会按色相 + 明度双轴聚类，自动将浅灰/深灰拆分为独立通道
5. 在结果面板中查看每个色域的 **L\* 值**
6. 点击 **「✅ 一键应用到基础通道」** 完成分配

### 场景三：兼容旧版 HSV 工作流

1. 在容差面板中切换为 **HSV 色相距离** 模式
2. 所有旧版参数（色相容差、饱和度下限）照常工作
3. 导入旧版 v5/v6.0/v6.1 工程文件自动兼容

## 🔧 技术细节

### 新增核心函数

| 函数 | 用途 |
|------|------|
| `M.C.r2lab(R,G,B)` | RGB → CIELAB L\*a\*b\* 转换 |
| `M.C.deltaE(lab1,lab2)` | 标准 ΔE(CIE76) 色差计算 |
| `M.C.deltaEw(lab1,lab2,Lw)` | 带明度权重的 ΔE |
| `M.C.batchLab(px,n)` | 批量预计算图片所有像素的 LAB 值 |
| `M.Tol.match(i,refRGB,refLab,hue)` | 统一匹配接口，返回匹配强度 [0-1] |
| `M.Tol.isMatch(i,refRGB,refLab,hue)` | 布尔匹配判断 |
| `M.Tol.setMode(mode,btn)` | 切换 HSV/LAB 算法模式 |
| `M.Tol.update()` | 防抖更新容差预览 |
| `M.AD.toggleStrictL()` | 切换强制明度区分 |

### 数据结构变更

```javascript
// M.S (全局状态) 新增字段
M.S.lab         // Float32Array[n*3], 预计算的 CIELAB 数据
M.S.pkLab       // [L*,a*,b*], 当前吸取颜色的 LAB 值
M.S.tolMode     // 'hsv' | 'lab', 容差算法模式
M.S.strictLightness  // boolean, 是否强制区分明度

// Region (颜色区域) 新增字段
region.lab      // [L*,a*,b*], 参考颜色的 LAB 值
region.deltaE   // number, 该区域使用的 ΔE 容差
region.lightW   // number, 该区域使用的明度权重
region.tolMode  // 'hsv' | 'lab', 该区域使用的匹配算法
```

### 性能说明

- **CIELAB 预计算**：在 `M._enterPick` 中一次性计算整张图片的 LAB 值（`batchLab`），存储为 `Float32Array`，后续匹配时直接查表，无需重复计算
- **防抖实时预览**：容差滑块的 `update()` 使用 80ms 防抖，避免高频重绘
- **向后兼容**：v5/v6.0/v6.1 工程文件导入时，旧 region 默认使用 HSV 模式匹配，不受 CIELAB 影响
