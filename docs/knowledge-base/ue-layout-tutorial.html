1.1 插槽 (Slot) 区域 —— 决定控件“怎么放”

注意：“插槽”面板的参数会根据父节点容器的不同而完全改变！

父节点类型面板显示内容参数代表什么意思？怎么调？
覆层
(Overlay)		 覆层插槽 (Overlay Slot) • 边距 (Padding): 展开后有左、上、右、下四个值。决定该控件距离父容器边缘多少像素。
• 水平对齐 (Horizontal Alignment): 四个小图标分别代表 居左、居中、居右、水平拉伸铺满。

(H/V Box)		 水平/垂直框插槽 • 大小 (Size): 可选 Auto (按自身内容自适应大小) 或 Fill (自动拉伸填满剩余空间)。
• 边距 (Padding): 控制兄弟节点排队时的相互间距。

1.4 行为 (Behavior) 与 渲染变换 (Render Transform)

  • 已启用 (Is Enabled)： 取消勾选后，按钮会变成灰色的“禁用状态”，无法点击。
  • 渲染不透明度 (Render Opacity)： 调节整个控件（及里面所有子节点）的透明度。1.0 为完全不透明，0.0 为全透明。
  • 渲染变换 (Render Transform)：
    • 平移 (Translation X/Y)：视觉上偏移，但不改变实际的排版位置。常用于做飘字动画、按钮按下时的缩放效果。
    • 缩放 (Scale) / 旋转 (Angle)：制作 UI 动效时调整。

📦 2. 布局面板（Panel）详解与对比

在 UMG 中，面板决定了子元素如何排列。选择正确的面板是 UI 拼接的第一步。

2.1 画布面板 (Canvas Panel) vs 覆层 (Overlay)

🖼️ 画布面板 (Canvas Panel)

作用： 允许你在其中任意拖拽摆放控件。子控件依靠 Anchor (锚点) 进行定位。

  • 定位方式： 通过 Position X/Y 和 Size X/Y 控制位置。
  • 适用场景： 界面最底层的根容器；需要固定在屏幕特定角落（如右下角技能）的零散元素。
  • 缺点： 性能开销略大。不要在复杂的 UI 内部频繁嵌套 Canvas。

🥞 覆层 (Overlay)

作用： 将多个控件像三明治一样“叠”在一起。最先放入的在底层，后放入的在上层。

  • 定位方式： 通过 Padding (边距) 和 Alignment (水平/垂直对齐) 来排布。
  • 适用场景： 按钮的内部（底图叠上文字）；装备格（底图叠上图标再叠上边框）。
  • 优点： 性能极好！官方强烈推荐：能用 Overlay 解决的层叠，绝对不要用 Canvas。

2.2 线性布局：Horizontal Box vs Vertical Box

这两种盒子用于让控件自动排队，无需手动计算坐标。只需把控件拖进去，调整插槽里的 Padding 即可控制间距。

🔤 3. 常用交互控件图鉴与核心参数

3.1 图片 (Image) —— 必须掌握的九宫格

🖼️ [配图说明：展示 Image 组件的 Details 面板，高亮 Brush 选项卡卡中的 Draw As 和 Margin 属性，并在旁边放一个九宫格切图拉伸对比图]
  • Brush -> Image (图片资产)： 选择你的切图。
  • Brush -> Draw As (绘制为)：
    • Image：默认模式，改变尺寸时图片会被强行拉伸变形。
    • Box：九宫格模式。用于各种圆角底板、按钮背景。选择 Box 后，必须在下方调整 Margin (边距)。Margin 通常是一个小数（如 0.25），代表保护四个圆角不被拉伸的比例。

3.2 玩家文本输入：Editable Text / Multi-Line

当需要玩家在界面上打字（如输入名字、聊天、密码）时，必须使用输入框组件。

⌨️ [配图说明：展示单行输入框 Editable Text 和多行输入框 Multi-Line 的属性面板对比截图]

🔡 可编辑文本 (Editable Text)

用于单行输入（如注册账号、兑换码）。

  • Hint Text： 提示文案（如“请输入玩家姓名…”）。输入后自动消失。
  • Is Password： 勾选后，输入的内容会变成星号 ***。
  • Is Read Only： 勾选后，框存在但禁止输入文字，可用于纯文本复制展示。

🔠 多行可编辑文本 (Multi-Line)

用于大段输入（如意见反馈、公会宣言）。

  • Auto Wrap Text： 强烈建议勾选。当玩家输入到边界时自动换行。
  • 注意： 这两个输入框默认背景都是透明的。你通常需要在它下面垫一张 Image 作为底图，再用 Overlay 把它们包起来。

3.3 按钮 (Button)

最基础的点击交互组件。Button 内部只能包含一个子节点（通常放一个 Text 或 Overlay）。

  • Style (样式)： 包含 Normal（正常）、Hovered（悬停）、Pressed（按下）、Disabled（禁用）四种状态。你需要分别为这四个状态配置对应的 Image 和颜色。
  • Click Method (点击方法)： 决定是鼠标按下时触发（Down and Up），还是鼠标抬起时才触发（Mouse Release）。

📜 4. 列表与滚动控件全解析

当内容超过屏幕时，我们需要滚动功能。UMG 提供了三种不同级别的控件。

4.1 Scroll Box（滚动框） vs Wrap Box（自动换行框）

↕️ Scroll Box (滚动框)

最基础的滚动视图。超出范围自动出现滚动条。

  • 适用场景： 内容固定、条目较少（如只有 10 条的设置面板）。
  • 致命缺点： 如果列表有 1000 条数据，会同时渲染 1000 个 UI 实例，瞬间卡死游戏。大型列表千万别用！

↩️ Wrap Box (自动换行框)

这不是滚动视图，只是排版容器。从左排到右，满行自动掉到下一行。

  • 适用场景： 技能标签词云、不规则排列的掉落展示。
  • 注意： 如果满屏后还需滚动，必须把 Wrap Box 整体丢进 Scroll Box 里。

4.2 性能救星：List View 与 Tile View（数据驱动列表）

针对拥有成百上千条数据的界面（如排行榜、大背包），必须使用这两种视图。

📜 [配图说明：左侧展示一个单列的 List View，右侧展示一个九宫格网格状的 Tile View。下方截图展示 Entry Widget Class 参数设置]
控件视觉表现核心原理与配置方法
List View
列表视图		 单列垂直往下排队滚动。 原理： 采用“对象池复用”。屏幕最多显示 10 个，引擎就只创建 12 个 UI，滑动时循环利用，性能极佳。

配置方法：
1. 必须单独创建一个继承了 User Object List Entry 接口的 UMG 蓝图作为“单条子项”。
2. 在右侧面板的 Entry Widget Class 参数中，选择刚才创建的子项蓝图。
3. 不能手动拖拖拽拽 UI 进去，必须通过蓝图代码动态传入数据对象。
Tile View
瓦片视图		 网格状排队（像九宫格），一行排满再排下一行。
💡 Tile View 专属重要参数：
• Entry Height / Entry Width：定义每个“格子”的固定尺寸。它决定了一行能排下多少个。
• Tile Alignment：当最后一行不满时，是居左、居中还是分散对齐。

🎯 5. 核心实操：锚点 (Anchor)、自适应与层级搭建

了解完节点后，如何组合成一个不随屏幕分辨率变形的界面？最核心的就是锚点 (Anchor)。

5.1 第一法则：打死定在原地的“锚点”对照表

在 Canvas Panel 下的子控件，它的位置是相对于锚点（那朵像小菊花一样的四叶草图标）来计算的。

🧲 [配图说明：极其重要的锚点九宫格图解。插入 UMG 锚点面板预设的 16 个小方块截图，并详细用箭头标出：左上吸附、居中吸附、横向拉伸、四边全屏拉伸 对应的方块]
UI 元素场景该选哪个锚点预设？面板对应的调整
界面左上角的返回按钮↖️ 左上角对齐Position X/Y 控制据左上角的距离
界面右下角的开战按钮↘️ 右下角对齐如果你选错了锚点，在带鱼屏手机上，按钮就会飘到屏幕中间去！
全屏纯色背景底图🔲 四边拉伸 (右下角的大方块)原有的 Size 面板会变成 Offsets (Left, Right, Top, Bottom)，把这四个值全部归 0，背景就会死死贴满屏幕。
顶部导航底栏 (只变宽不变高)↔️ 顶部横向拉伸 (中上方的长条)Offset Left/Right 设为 0，高度保持固定。
屏幕正中央的弹窗🎯 中心点设为中心点后，必须把 Alignment (对齐) 设为 X:0.5, Y:0.5，这样弹窗的真正物理中心才会和屏幕中心对齐。

5.2 标准面板搭建结构推荐

一个标准的、性能好的系统界面，它的层级树（Hierarchy）应该是这样的：

[根目录] WBP_MainUI
 └── Canvas Panel (界面最底层画板)
      ├── Image (全屏背景图，锚点: 四边拉伸, Offsets 全为 0, 可见性: Hit Test Invisible)
      └── Safe Zone (安全区，勾选Pad四边，完美适配所有刘海屏/打孔屏)
           └── Overlay (覆层，铺满整个安全区)
                ├── Image (通用弹窗大底板图，水平/垂直居中对齐)
                ├── Vertical Box (左中右布局中的某个列)
                │    ├── Text (主标题)
                │    └── List View (核心道具列表，Size 设为 Fill 填满剩余空间)
                └── Button (右上角关闭按钮，在Overlay里设为 右上角对齐)
