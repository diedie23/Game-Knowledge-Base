📦 5. 布局容器面板（ScrollBox、安全区、切换器等）

5.1 Scroll Box (滚动框) — 极深解析

💡 核心作用： 给那些内部内容远远超过屏幕物理尺寸的元素提供一个滚动视口。玩家可以用鼠标滚轮或手指上下划动。
🎯 适用场景： 长篇大论的用户协议、带大量文字的装备详细描述说明、不经常变化长短的设置项列表。
⚠️ 注意：如果列表条目超过 30 条且格式一致（如排行榜），绝对不能用 Scroll Box，必须用 List View 性能复用池！

➔

滚动条可视性 (Scrollbar Visibility) 秘籍：
很多设计图是不画丑陋滚动条的，但是内容又要能划动。千万别选 Hidden（隐藏），因为隐藏后其实它还在右侧暗中占据空间！
正确做法：选择「已折叠 (Collapsed)」，这样滚动条完全消失，内部空间自动向右填满，且用鼠标滚轮和手指依然能丝滑滚动！

面板分组具体参数名效果与如何调节即时预览

5.5 Horizontal/Vertical Box (排队框) 与 Overlay (覆层)

💡 Box 的核心： 子节点顺向排队，拒绝手工对坐标。
💡 Overlay 的核心： 子节点以 Z 轴重叠，专门用于汉堡包式叠加（底图层+图标层+数字层）。

被放进去后的子节点专属插槽(Slot)参数清单
插槽环境具体参数名效果与如何调节
身处 H/V Box 中 大小 (Size)
  • Auto (自动): 自己有多大就占多少排队位。
  • Fill (填充): 动态数值比例分赃！如果 A填1，B填2，那剩余空间按 1:2 分割强行拉伸组件填充框体。
Padding (边距) 控制这个排队的家伙与其他兄弟节点之间的间隔缝隙像素。
身处 Overlay 中 水平/垂直 对齐 四向对齐，或选择拉伸(Fill)。选满铺(Fill)会直接盖住整个底层。

♾️ 6. 高级列表视图（List View / Tile View）

⚠️ 核心理念：对于超长列表（排行榜、大背包），决不能用 Scroll Box 无脑塞子节点！必须使用对象池复用的 ListView (单列) 或 TileView (九宫格)，它通过复用划出屏幕外的那几个节点来实现极低的性能开销。

6.1 List/Tile View

面板分组具体参数名效果与如何调节即时预览
列表 (List) Entry Widget Class (条目类) 【唯一生死核心】！必须选定一个事先做好的、实现了 User Object List Entry 接口的蓝图类，作为生成的“一截小火车”。 ✅ 假数据刷新
Orientation (方向) 单行/单列滚动的横竖方向切换。 ✅ 立刻刷新
Tile View 专属 Entry Height / Width 由于要往一行塞多个格子，这里强行规定每个格子多大，系统才能算出一行能塞下几个。 ✅ 立刻刷新

8.2 对齐 (Alignment) —— 绝对居中的密码

当你要把弹窗绝对居中时，光把锚点设为“居中”是不够的！因为那只是以控件左上角对准屏幕中心。
【核心铁律】必须同时将 Alignment (对齐) 设为 X: 0.5, Y: 0.5，这代表用控件自己的正中心去贴合锚点，实现完美居中。
