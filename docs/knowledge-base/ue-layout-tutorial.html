1.1 插槽 (Slot) 区域 —— 决定控件“怎么放”

注意：“插槽”面板的参数会根据父节点容器的不同而完全改变！

父节点类型面板显示内容参数代表什么意思？怎么调？
覆层
(Overlay)		 覆层插槽 (Overlay Slot) • 边距 (Padding): 展开后有左、上、右、下四个值。决定该控件距离父容器边缘多少像素。
• 水平对齐 (Horizontal Alignment): 四个小图标分别代表 居左、居中、居右、水平拉伸铺满。
• 垂直对齐 (Vertical Alignment): 四个小图标分别代表 居顶、居中、居底、垂直拉伸铺满。

• 位置 (Position X/Y): 距离锚点的坐标偏移。
• 大小 (Size X/Y): 控件的绝对宽高。
• ZOrder (Z 排序): 渲染层级，数字越大的控件越显示在最上层，会盖住数字小的控件。
水平/垂直框
(H/V Box)		 水平/垂直框插槽 • 大小 (Size): 可选 Auto (按自身内容自适应大小) 或 Fill (自动拉伸填满剩余空间)。
• 边距 (Padding): 控制兄弟节点排队时的相互间距。

1.4 行为 (Behavior) 与 渲染变换 (Render Transform)

  • 已启用 (Is Enabled)： 取消勾选后，按钮会变成灰色的“禁用状态”，无法点击。
  • 渲染不透明度 (Render Opacity)： 调节整个控件（及里面所有子节点）的透明度。1.0 为完全不透明，0.0 为全透明。
  • 渲染变换 (Render Transform)：
    • 平移 (Translation X/Y)：视觉上偏移，但不改变实际的排版位置。常用于做飘字动画、按钮按下时的缩放效果。
    • 缩放 (Scale) / 旋转 (Angle)：制作 UI 动效时调整。

📦 2. 布局面板（Panel）详解与对比

在 UMG 中，面板决定了子元素如何排列。选择正确的面板是 UI 拼接的第一步。

2.1 画布面板 (Canvas Panel) vs 覆层 (Overlay)

🖼️ 画布面板 (Canvas Panel)

作用： 允许你在其中任意拖拽摆放控件。子控件依靠 Anchor (锚点) 进行定位。

  • 定位方式： 通过 Position X/Y 和 Size X/Y 控制位置。
  • 适用场景： 界面最底层的根容器；需要固定在屏幕特定角落（如右下角技能）的零散元素。
  • 缺点： 性能开销略大。不要在复杂的 UI 内部频繁嵌套 Canvas。

🥞 覆层 (Overlay)

作用： 将多个控件像三明治一样“叠”在一起。最先放入的在底层，后放入的在上层。

  • 定位方式： 通过 Padding (边距) 和 Alignment (水平/垂直对齐) 来排布。
  • 适用场景： 按钮的内部（底图叠上文字）；装备格（底图叠上图标再叠上边框）。
  • 优点： 性能极好！官方强烈推荐：能用 Overlay 解决的层叠，绝对不要用 Canvas。

2.2 线性布局：Horizontal Box vs Vertical Box

这两种盒子用于让控件自动排队，无需手动计算坐标。只需把控件拖进去，调整插槽里的 Padding 即可控制间距。

📜 4. 列表与滚动控件全解析

当内容超过屏幕时，我们需要滚动功能。UMG 提供了三种不同级别的控件。

4.1 Scroll Box（滚动框） vs Wrap Box（自动换行框）

↕️ Scroll Box (滚动框)

最基础的滚动视图。超出范围自动出现滚动条。

  • 适用场景： 内容固定、条目较少（如只有 10 条的设置面板）。
  • 致命缺点： 如果列表有 1000 条数据，会同时渲染 1000 个 UI 实例，瞬间卡死游戏。大型列表千万别用！

↩️ Wrap Box (自动换行框)

这不是滚动视图，只是排版容器。从左排到右，满行自动掉到下一行。

  • 适用场景： 技能标签词云、不规则排列的掉落展示。
  • 注意： 如果满屏后还需滚动，必须把 Wrap Box 整体丢进 Scroll Box 里。

🎯 5. 核心实操：锚点 (Anchor)、自适应与层级搭建

了解完节点后，如何组合成一个不随屏幕分辨率变形的界面？最核心的就是锚点 (Anchor)。

5.1 第一法则：打死定在原地的“锚点”对照表

在 Canvas Panel 下的子控件，它的位置是相对于锚点（那朵像小菊花一样的四叶草图标）来计算的。当遇到不同比例的屏幕（如带鱼屏、iPad 平板）时，界面就是依据锚点来进行拉伸和固定的。

5.2 标准面板搭建结构推荐

一个标准的、性能高的系统界面，它的蓝图层级树（Hierarchy）应该是这样的：

[根目录] WBP_MainUI
 └── Canvas Panel (界面最底层画板)
      ├── Image (全屏遮罩背景图，锚点:四边拉伸, Offsets:全0, 可见性:Hit Test Invisible)
      └── Safe Zone (安全区，勾选Pad四周，完美避开所有刘海屏/打孔屏)
           └── Overlay (覆层，铺满整个安全区)
                ├── Image (通用弹窗底板，水平/垂直居中对齐)
                ├── Vertical Box (左中右布局中的主列)
                │    ├── Text (主标题)
                │    └── List View (核心道具列表，Size设为Fill填满剩余空间)
                └── Button (右上角关闭按钮，在Overlay里直接设为 右上角对齐)
