📋 工具概览

资源分拣与增量同步工具是专为 UGC 美术资源入库流程 设计的桌面端自动化工具。它能从暂存的切图目录中，自动将图片按 图标 和 实际用图 两种类型精准分拣到对应的引擎/美术目录，同时实现增量同步、内容去重和自动命名规范化。

核心解决的问题：美术同学切完图后，不需要再手动区分图标/用图、手动检查重复、手动重命名后缀，一键分析 + 一键分发全搞定。

🚀 核心功能

🔍

智能尺寸分拣

根据可配置的图标尺寸阈值（默认 180×180），自动将图片分类为"图标"或"实际用图"，分发到不同目标目录

🔐

图像哈希去重

基于 SHA-256 像素级哈希计算（忽略元数据差异），精准识别内容完全相同的图片，自动跳过重复资源

📊

增量状态分析

三级状态判定：New（新增）、Modified（已修改）、Unchanged（重复），一目了然哪些资源需要入库

✏️

自动后缀管理

图标自动追加 _S 后缀，实际用图自动移除 _S 后缀，确保引擎目录命名规范

⚠️

尺寸合规检测

自动检测实际用图尺寸是否为 4 的倍数，不合规资源标记为 Error 状态并阻止入库

🖼️

图片预览对比

左右对比面板：左侧显示暂存区新图，右侧显示目标目录旧图/重复资源，一眼看清差异

🤖

智能配对重命名

多维度匹配算法（文件名相似度 50% + 修改时间 30% + 图像内容 20%），自动寻找图标与用图的最佳配对

🔄

批量字符替换

查找/替换暂存区文件名中的指定字符，快速清理前缀、后缀或特殊标记，替换后自动刷新分析

💾

哈希缓存加速

自动缓存已计算过的图像哈希（基于文件修改时间判断），后续分析速度显著提升

🏷️ 资源状态说明

New
目标目录中不存在同名文件，将作为新资源入库
Modified
目标目录中存在同名但内容不同的文件，将覆盖更新
Unchanged
内容哈希完全一致（或与其他资源重复），自动跳过
Error
存在合规问题（如尺寸非4倍数），阻止入库并保留在暂存区

🤖 智能配对重命名

当暂存区同时存在某个资源的图标版和实际用图版时，工具支持三种重命名方式：

操作方式触发方法说明
单文件重命名 右键单个文件 → "✏️ 重命名该文件" 手动输入新文件名（含扩展名）
自动配对重命名 右键单个文件 → "🤖 自动寻找配对并重命名" 基于多维度算法自动匹配最佳配对，弹出确认对话框
手动配对重命名 选中 1 个图标 + 1 个实际用图 → 右键 "🔗 智能配对重命名" 提取最长公共子串作为建议名称，图标自动加 _S

自动配对匹配算法权重

维度权重计算方式
文件名相似度50%SequenceMatcher 字符匹配率
修改时间接近度30%120 秒内线性衰减
图像内容相似度20%缩至 32×32 灰度图后逐像素对比

匹配阈值：综合得分 ≥ 0.35 才视为有效配对。

📤 分发入库规则

  • New 和 Modified 状态的资源将被分发到目标目录
  • Unchanged 状态的资源自动跳过（重复资源）
  • Error 状态的资源阻止入库，保留在暂存区待修复
  • 分发后默认删除暂存区原文件；勾选"分发后保留原文件"可保留副本
  • 分发完成后自动刷新分析结果
