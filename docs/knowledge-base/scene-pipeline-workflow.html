2️⃣ 白盒阶段 Blockout

2.1 白盒制作标准
维度标准说明
工具BSP (UE5) / ProBuilder (Unity) / 引擎原语不使用美术模型
尺寸精度对齐 100cm Grid与最终模块化 Kit 匹配
颜色编码地面=灰色；墙=白色；可交互=绿色；危险=红色便于策划/QA 识别
碰撞BSP 自带碰撞需可玩可测试
光照仅主光 + 天空光（不调细节）提供基本可见性
NavMesh需生成并验证 AI 寻路白盒阶段即验证动线
2.2 白盒评审清单
#检查项通过标准责任人
1空间尺寸关键区域尺寸与设计文档一致 (±10%)关卡策划
2玩法动线主动线畅通、无死角卡点关卡策划
3战斗空间战斗区域满足掩体/回合需求关卡策划 + 战斗策划
4AI 寻路NavMesh 覆盖所有可行走区域关卡策划 + 程序
5性能预估粗估同屏面数在预算 80% 以内TA

3️⃣ 灰盒阶段 Graybox

3.1 灰盒制作标准
维度标准说明
替换范围所有 BSP 替换为模块化 Kit Mesh遵循 模块化规范
Kit 覆盖率≥ 60%允许 40% 定制件（后续迭代提升）
材质使用 Kit 标准材质实例（非最终精度）确认 Texel Density 一致
地形高度图导入 + 基础地表 2~3 层参考 地形植被规范
光照主光方向确认 + 临时 GI参考 光照规范
碰撞更新Kit 自带碰撞替换 BSP 碰撞需重新验证 NavMesh
3.2 灰盒评审清单
#检查项通过标准责任人
1Kit 覆盖率≥ 60%场景美术 + TA
2Grid 对齐所有 Kit 对齐 100cm Grid场景美术
3视觉氛围整体氛围方向明确（色调/光照/材质倾向）美术总监
4模块命名符合命名规范场景美术
5碰撞/NavMesh玩法动线无回退关卡策划
6性能初测同屏面数 ≤ 预算 120%（留优化空间）TA

8.2 门禁不通过处置方案
情况处置PM 动作
白盒门禁不通过返回白盒，修改动线/尺寸后重审记录延期风险；协调策划/场景美术排期
灰盒门禁不通过补充 Kit / 修复对齐问题追溯是否 Kit 库不足导致覆盖率低
精制门禁不通过补充细节 / 调整光照 / 修复 TD 差异评估是否需增加人力
优化门禁不通过继续优化直到性能达标升级为风险项；每日跟踪 Profile 数据
交付验收不通过修复 Critical/High Bug 后重新提交协调 QA 优先级；确认修复不影响其他关卡

9️⃣ 风险识别与应对预案

风险项概率影响预警信号应对措施
Kit 库不足高高灰盒阶段 Kit 覆盖率 < 50%增加 Kit 制作人力；临时使用 Marketplace 资产占位
性能超预算中高灰盒阶段同屏面数 > 预算 150%提前引入 TA 做性能评估；缩减场景规模
光照方案变更低极高量产期提出切换光照方案绝对避免！在预研期锁定；若确需变更需制作人审批
美术外包质量中中外包 Kit 验收一次通过率 < 60%加强验收标准培训；增加中期检查频率
工期压缩高中实际进度落后计划 > 20%优先完成主动线；次要区域降为灰盒交付
跨部门协调中中白盒评审后策划频繁变更需求设定需求冻结期（灰盒后禁止大改）

10.3 数据驱动决策模型

✅ 健康状态

• 门禁通过率 ≥ 70%
• 各阶段耗时 ≤ 计划 110%
• 走查 Bug 密度 ≤ 5

动作：维持节奏；关注长期趋势优化

❌ 预警状态

• 门禁通过率 < 50%
• 某阶段耗时 > 计划 150%
• 走查 Critical Bug > 2/关卡

动作：启动专项治理；增加人力或缩减范围；每日站会跟踪

10.4 跨关卡资源调度模型
场景问题PM 调度策略
关卡 A 提前完成人力闲置将场景美术调配至落后关卡支援
关卡 B 精制阶段严重延期影响总进度① 评估是否降级为灰盒交付 ② 增调人力 ③ 缩减非核心区域
Kit 库缺口多个关卡等待新 KitKit 制作优先级提升；按关卡权重排序
TA 资源紧张优化阶段堆积提前规划 TA 排期；工具化自动优化流程
✅ 最佳实践：PM 每周三进行「场景管线周检」—— 逐关卡检查当前阶段、门禁预期日期、风险项状态。使用「交付漏斗图」展示各阶段堆积情况，提前发现瓶颈。
