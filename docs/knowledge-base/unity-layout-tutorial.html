📐 1. 核心基石：Canvas 与 Rect Transform

在 UGUI 中，所有 UI 元素都必须挂载 Rect Transform 组件，并且必须放在 Canvas（画布）的子节点下才能显示。

1.1 画布 (Canvas) 与 缩放器 (Canvas Scaler)

Canvas 就像一块画板，Canvas Scaler 决定了这块画板在不同手机屏幕上是放大还是缩小。

  • UI Scale Mode： 绝大多数游戏必须选择 Scale With Screen Size（跟随屏幕尺寸缩放）。
  • Reference Resolution： 基准分辨率（设计稿尺寸）。通常填 X: 1920, Y: 1080。
  • Screen Match Mode：
    - 横屏游戏推荐：Match Width Or Height，且 Match 值设为 1（以高度为基准缩放）。
    - 竖屏游戏推荐：Match 设为 0（以宽度为基准缩放）。
    - 通用折中方案：Match 设为 0.5（取宽高中间值）。

1.2 矩形变换 (Rect Transform)

它是 3D Transform 的 2D 升级版，多出了宽高、锚点和中心点控制。

  • Pos X/Y/Z： 控件相对于锚点的位置。注意：Unity 的 Y 轴是向上为正的！
  • Width / Height： 控件的具体宽高像素。当锚点设置为“拉伸 (Stretch)”时，这两个属性会变成 Left, Right, Top, Bottom 的边距 (Margin)。

📦 2. 布局组（Layout Group）详解与对比

UGUI 本身不像 UE 那样提供直接拖拽的面板控件（如 Horizontal Box）。你需要先创建一个空的 GameObject，再挂载各种 Layout Group 脚本组件。

2.1 水平与垂直布局组

➡️ 水平布局组 (Horizontal Layout Group)

子元素从左到右自动排队。适用于导航栏菜单、资源货币栏。

  • Padding（内边距）： 容器四周留出的空白空间。
  • Spacing（间距）： 两个子控件之间的间隔距离。
  • Child Alignment： 当子节点总宽度填不满容器时，它们是靠左对齐还是居中对齐？
  • Child Force Expand： 勾选后，所有子节点会被强制拉伸瓜分剩余空间。通常不建议勾选 Width 和 Height，而是用 Layout Element 单独控制。

⬇️ 垂直布局组 (Vertical Layout Group)

子元素从上到下自动排队。适用于纵向菜单、滚动列表内容。

  • 参数与水平布局组完全相同。
  • 注意控制权问题： 如果勾选了 Control Child Size 的 Width 或 Height，子节点的宽或高就会被布局组强行接管，你无法在子节点上修改宽度或高度了！

2.2 网格布局组 (Grid Layout Group)

用于排列像背包一样的九宫格/多宫格内容。它不是自动换行的流式布局，而是固定大小的格子。

  • Cell Size： 极其重要！强制规定每个格子（子节点）的固定宽高，无视子节点原本的尺寸。
  • Constraint (约束)：
    - Flexible：自动计算行列数（比如空间够放几个就是几个）。
    - Fixed Column Count：强制固定列数（比如每行只能放 5 个道具）。
💡 布局元素 (Layout Element) 组件：
如果一个排队容器里，你希望有的按钮大一点，有的按钮小一点，怎么办？
给那个特殊的子节点挂载一个 Layout Element 组件。开启 Min Width 或 Preferred Width，告诉父容器“我的最小宽度必须是这么多”。

📜 3. 滚动与列表容器全解析

3.1 Scroll Rect (滚动视图) 的三大核心

Unity 的滚动视图通常由三个层级组成。这是最容易拼错的地方：

ScrollView (挂载 Scroll Rect 组件)
 ├── Viewport (带 Mask 组件，作为遮罩遮住超出的内容)
 │    └── Content (所有条目的父节点，必须能自动伸缩！)
 └── Scrollbar (可选的滚动条)

3.2 让列表无限延伸：Content Size Fitter

Scroll Rect 的原理是：当 Content 节点比 Viewport 大时，就可以滑动。但你怎么保证 Content 的大小刚好包住里面所有的条目？

  • 必须给 Content 节点挂载 Content Size Fitter（内容大小自适应器）组件。
  • 如果是垂直滑动列表，将 Vertical Fit 设为 Preferred Size。
  • 同时给 Content 挂载一个 Vertical Layout Group 让子节点排队。
  • 这样，每当你塞进去一个新条目，Content 就会自动拉长高度，Scroll Rect 就会自动识别并允许滚动。
🚀 性能警告：无限循环列表 (Loop Scroll Rect)
Unity 自带的 Scroll Rect 也会把所有条目一次性实例化（比如 1000 个道具）。对于大型游戏，必须使用程序员提供的对象池复用列表组件（如常见的 LoopScrollRect、SuperScrollView 等第三方插件），其配置方式因项目而异。

🔤 4. 基础与交互组件（图片、文本、输入框）

4.1 图片 (Image) 与 九宫格 (Sliced)

确保图片在拉伸时不模糊、圆角不变形，必须使用九宫格切图：

  1. 在 Project 面板选中源图片，在 Inspector 找到 Sprite Editor（需安装 2D Sprite 包）。
  2. 拖拽图片四周的绿色边框（Border），设定不需要拉伸的四个角区域。点击 Apply 保存。
  3. 回到 UI，选中挂有 Image 的节点。将 Image Type 从 Simple 改为 Sliced (切片)。
  4. 若没有 Fill Center 需求可以取消勾选，能省一点性能。

4.2 高清文本 (TextMeshPro)

永远不要用 Unity 原生的 Legacy Text 组件（字非常模糊），必须使用 TextMeshPro - Text (UI)。

  • Font Asset： 选择生成好的 TMP 字体资产（SDF格式，无限放大不模糊）。
  • Vertex Color： 字体颜色。甚至能开渐变色 (Color Gradient)。
  • Alignment： 对齐方式（上中下左中右）。
  • Auto Size： 开启后，如果文本太长挤不下框框，字体会自动缩小。

4.3 玩家文本输入：TMP_InputField 组件

当需要玩家在界面上打字（聊天、改名、输密码）时，必须使用输入框组件。

🔡 TMP_InputField (可编辑输入框)

它是一个复合组件，一般包含一个输入框背景图，以及内部的 Text 和 Placeholder。

  • Text Component： 指向实际显示输入内容的子节点（TMP 文本）。
  • Placeholder： 指向显示提示文案的子节点（如“请输入兑换码”）。玩家一输入内容，这个节点就会自动隐藏。
  • Character Limit： 字符长度限制。填 0 表示不限制，填 10 表示最多输入 10 个字。
  • Content Type（内容类型）：
    - Standard：普通内容。
    - Password：密码模式，输入的内容变成星号（可配置星号字符）。
    - Integer Number：只能输入数字。
    - Email Address：唤起手机系统键盘的邮箱专用布局。
  • Line Type：
    - Single Line：单行输入。按回车键表示结束输入。
    - Multi Line Submit / Newline：多行输入。按回车键换行。通常聊天框或者留言板使用。
  • Read Only： 勾选后，框无法被激活输入文字，但里面的文本允许被鼠标/手指长按并复制（取决于脚本支持）。

🎯 5. 核心实操：锚点、中心点与层级搭建

5.1 锚点预设 (Anchor Presets) 与 轴心 (Pivot)

点开 Rect Transform 左上角的那个带方块的小图标，就是 Anchor Presets (锚点预设面板)。

  • 按住 Shift 点击： 同时改变 Pivot (中心点)。比如你想让弹窗不仅挂在中间，它的中心点也是正中间，按住 Shift 点正中间的方块。
  • 按住 Alt 点击： 同时改变 Position (位置)。按住 Alt 点左上角方块，UI 会瞬间飞到屏幕绝对的左上角贴边。
  • 右下角四个带箭头的方块： 它们是 Stretch (拉伸模式)。点击水平和垂直双向拉伸的那个（按住 Alt点击），这个 UI 就会瞬间铺满它的父节点。适合做全屏背景图。

5.2 异形屏安全区 (Safe Area) 的处理

与 UE 不同，Unity 没有自带的无脑拖拽组件，需要通过脚本处理：

在所有 UI 的最顶层（Canvas 之下第一层），建一个名叫 SafeAreaPanel 的空物体，锚点设为全屏拉伸，并给它挂载程序员提供的 SafeArea 脚本（核心原理是根据 Screen.safeArea 动态修改锚点）。

5.3 标准面板搭建结构推荐

一个标准的系统界面层级树应该是这样的：

Canvas (根画布)
 └── SafeAreaPanel (挂载脚本的安全区面板，全屏拉伸)
      ├── Background_Img (全屏底图，全屏拉伸拉满)
      ├── TopBar_Panel (顶部导航栏面板)
      │    ├── Anchor: 顶部横向拉伸 (Top-Stretch)
      │    ├── Title_Text (居中对齐文字)
      │    └── Back_Btn (左上角锚点，左上角贴边)
      └── MainContent_Panel (中间主内容区)
           ├── Anchor: 居中拉伸，留出顶部边距
           └── ScrollView (滚动的装备列表)
                └── Content (带 ContentSizeFitter 和 Grid Layout Group)
                     ├── Item_Prefab (预制体，各种格子)
                     ├── Item_Prefab
                     └── Item_Prefab
