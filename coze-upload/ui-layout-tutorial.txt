  📖 1. 前置知识：什么是 UI 拼接？

  1.1 概念解释
  UI 拼接（也叫"界面重构"/"UI 搭建"）是指把设计师在 Photoshop / Figma / Sketch 中做好的 UI 设计稿，在游戏引擎（UE5 / Unity）中用引擎原生 UI 系统搭建还原出来的过程。
  类比理解：设计稿 = 房屋设计图纸，UI 拼接 = 按图纸把房子砌起来。你需要把设计稿里的按钮、文字、图片、背景等元素在引擎里逐个"拼装"出来，并确保在不同屏幕尺寸上都正常显示。

  1.2 两大引擎 UI 系统对照
  
    概念UE5（UMG）Unity（UGUI）
    UI 编辑器UMG Designer（Widget Blueprint 编辑器）Canvas + Inspector 面板
    根容器Canvas Panel / OverlayCanvas
    定位系统Anchors + Alignment + OffsetsAnchors + Pivot + Position
    文件格式.uasset（Widget Blueprint）.prefab（UGUI Prefab）
    布局容器Horizontal Box / Vertical Box / Grid PanelHorizontal/Vertical/Grid Layout Group
    图片节点Image WidgetImage Component
    文字节点Text WidgetTextMeshPro (TMP)
    按钮节点Button WidgetButton Component
  

  1.3 你需要准备什么？
  
    1切图资源
    
      
        设计师提供的 PNG 切图（按钮图、背景图、图标等）
        文件命名遵循切图与命名规范
        确认分辨率基准：1920×1080
      
    
  
  
    2设计标注
    
      
        每个 UI 元素的位置坐标（距离边缘的像素值）
        每个元素的锚点模式（贴左上角？居中？全屏拉伸？）
        字号、颜色、间距标注
      
    
  
  
    3引擎环境
    
      
        UE5：项目已打开，编辑器可正常运行
        Unity：项目已打开，Canvas 已存在于场景中
      
    
  




  🔷 2. UE5 UMG 拼接实操全流程
  Unreal Engine 5 · UMG

  2.1 创建 Widget Blueprint
  
    1新建 Widget Blueprint
    
      
        在 Content Browser（内容浏览器）空白处右键
        选择 User Interface → Widget Blueprint
        命名规则：WBP_界面名称（如 WBP_MainMenu）
        双击打开，进入 UMG Designer 编辑界面
      
    
  

  
    2认识 UMG 编辑器界面
    
      
        左侧 Palette（控件面板）：所有可拖入的控件列表
        中央 Canvas（画布）：可视化拖拽编辑区域
        右侧 Details（细节面板）：选中控件的所有属性
        左下 Hierarchy（层级面板）：控件父子层级结构
        顶部工具栏：分辨率预览切换下拉
      
    
  

  2.2 DPI 缩放与根节点设置
  
    3配置 DPI 缩放规则
    
      
        菜单栏 → Edit → Project Settings
        左侧搜索 User Interface
        找到 DPI Scaling 区域
        将 DPI Scaling Rule 设为 Shortest Side
        设置 DPI Curve：确保 1080 对应值为 1.0
      
      Shortest Side 表示以屏幕短边作为缩放基准。横屏游戏推荐此设置，可确保 UI 在 16:9、21:9 等各种宽高比下不会被意外拉伸。
    
  

  
    4选择根容器节点
    
      Widget Blueprint 打开后，需要在 Palette 中拖入一个根容器：
      
        根容器功能说明适用场景
        Canvas Panel自由定位，子控件通过 Anchor + Offset 绝对定位✅ 主界面、HUD（最常用）
        Overlay所有子控件叠在同一区域，后面的遮住前面的弹窗叠加层
        Vertical Box子控件从上到下自动排列列表、设置面板
        Horizontal Box子控件从左到右自动排列底部导航栏
      
      大多数情况下选 Canvas Panel 作为根节点。只有当整个界面本身就是纯列表排列时才用 Box 作根。
    
  

  2.3 添加 UI 元素与设置锚点（⭐最关键）
  
    5拖入控件到画布
    
      
        在 Palette 搜索框输入控件名（如 Image、Text、Button）
        将控件直接拖入中央 Canvas 画布
        或者拖到 Hierarchy 面板中对应父节点下
        在 Hierarchy 中调整层级：拖动控件即可改变父子关系
      
    
  

  
    6⭐ 设置 Anchor 锚点
    
      
        在 Canvas 上选中一个控件
        右侧 Details 面板找到 Slot (Canvas Panel Slot) 区域
        点击 Anchors 旁的预设方块图标
        弹出 9 宫格 Anchor 预设面板，选择合适位置
      
      
        元素类型选哪个预设含义
        左上角返回按钮左上角方块始终贴左上角
        右上角设置按钮右上角方块始终贴右上角
        全屏背景四角全选（Stretch）铺满整个屏幕
        居中弹窗中心点方块始终在屏幕正中
        底部导航栏底部横向拉伸贴底且宽度随屏幕变化
      
      新手最常犯的错：不设 Anchor 或所有元素都用默认左上角。一旦切换分辨率，不在左上角的元素全部跑偏！每放一个元素，第一件事就是设锚点。
    
  

  
    7设置 Offset 偏移量
    
      Anchor 确定"相对于哪里定位"，Offset 确定"距离那里多远"：
      
        参数说明
        Position X水平偏移（正=向右，负=向左）
        Position Y垂直偏移（正=向下，负=向上）
        Size X / Size Y控件宽度/高度（像素）
        Offset Left/Right/Top/Bottom拉伸模式下四边距父容器距离
      
      直接对照设计稿标注填入。例如"返回按钮距左边 24px、距顶部 16px"→ Position X=24, Position Y=16。
    
  

  
    8设置 Alignment 对齐点
    
      Alignment 决定控件自身哪个点与 Anchor 位置对齐：
      
        (0, 0) → 控件左上角对齐到 Anchor（默认）
        (0.5, 0.5) → 控件中心对齐到 Anchor
        (1, 1) → 控件右下角对齐到 Anchor
      
      居中弹窗必须把 Alignment 设为 (0.5, 0.5)！否则弹窗左上角对准中心点，视觉上会偏右偏下。
    
  

  2.4 图片 / 文字 / 按钮 参数设置
  
    9Image 图片控件
    
      
        参数路径说明建议
        Appearance → Brush → Image选择贴图资源导入的 PNG 切图
        Brush → Draw As绘制模式Image=普通 / Box=九宫格
        Brush → Margin九宫格切割边距仅 Draw As=Box 时需设
        Color and Opacity叠加颜色+透明度白色=原色，可用于换色
        Slot → Size To Content ✅自动按图片原始尺寸无需手动填宽高
      
    
  

  
    10Text 文字控件
    
      
        参数说明
        Content → Text文字内容
        Appearance → Font Family字体（需先导入 .ttf/.otf）
        Appearance → Size字号（px）
        Appearance → Color文字颜色
        Justification对齐：Left / Center / Right
        Auto Wrap Text自动换行
      
    
  

  
    11Button 按钮控件
    
      
        参数说明
        Style → Normal默认态图片/颜色
        Style → Hovered悬停态图片/颜色
        Style → Pressed按下态图片/颜色
        Style → Disabled禁用态图片/颜色
      
      常见按钮层级：
      Button
  └── Overlay
        ├── Image（按钮背景·九宫格）
        └── Horizontal Box
              ├── Image（图标）
              └── Text（按钮文字）
    
  

  2.5 Safe Zone 安全区
  
    12添加 SafeZone 组件
    
      
        Palette 中搜索 Safe Zone
        将其作为最外层容器包裹 UI 内容
        Details 面板确认 Pad Left/Right/Top/Bottom 全部 ✅
      
      Canvas Panel（根）
  └── SafeZone（安全区）
        ├── 返回按钮（左上锚点）
        ├── 设置按钮（右上锚点）
        └── 主内容区域
      SafeZone 自动检测设备安全区域（刘海/药丸/圆角），确保子控件不被遮挡。
    
  

  2.6 多分辨率预览
  
    13切换分辨率测试
    
      在 UMG Designer 顶部工具栏的分辨率下拉菜单中切换：
      
        分辨率比例检查重点
        1920×108016:9基准，应与设计稿完全一致
        2560×108021:9角落元素不应飘到中间
        1024×7684:3内容不应上下裁切
        1334×75016:9 低分内容不应溢出屏幕
      
    
  




  🟢 3. Unity UGUI 拼接实操全流程
  Unity · UGUI

  3.1 创建 Canvas 与缩放设置
  
    1创建 Canvas
    
      
        Hierarchy 面板空白处右键 → UI → Canvas
        自动创建 Canvas + EventSystem
        选中 Canvas，在 Inspector 中设置：
      
      
        组件 / 参数值说明
        Canvas → Render ModeScreen Space - OverlayUI 覆盖在画面最上层
        Canvas Scaler → UI Scale ModeScale With Screen Size按参考分辨率等比缩放
        Canvas Scaler → Reference Resolution1920 × 1080设计稿基准分辨率
        Canvas Scaler → Screen Match ModeMatch Width Or Height混合匹配
        Canvas Scaler → Match0.50.5=兼顾宽高；1=优先高度
      
      设好 Reference Resolution = 1920×1080 后，设计稿上标注的像素值可以直接填入 Unity，无需换算。
    
  

  
    2Canvas Scaler Match 值怎么选
    
      
        Match 值行为适用
        0完全按宽度缩放竖屏游戏
        0.5取宽高中间值通用推荐
        1完全按高度缩放横屏游戏（保证纵向内容不裁切）
      
    
  

  3.2 添加 UI 元素
  
    3添加 UI 对象
    
      在 Canvas 下右键 → UI → 选择控件：
      
        控件菜单路径用途
        ImageUI → Image图片/背景/图标
        Text (TMP)UI → Text - TextMeshPro文字（推荐 TMP）
        Button (TMP)UI → Button - TextMeshPro可点击按钮
        PanelUI → Panel半透明背景面板
        Scroll ViewUI → Scroll View可滚动列表
        SliderUI → Slider滑动条（血条/进度条）
      
    
  

  3.3 设置 Anchor 锚点（⭐最关键）
  
    4⭐ 设置 Rect Transform Anchor
    
      
        选中 UI 对象，在 Inspector 找到 Rect Transform
        点击左上角的方形 Anchor 图标（带四条小线的方块）
        弹出 Anchor Presets 面板
        选择合适的预设位置
      
      快捷操作：按住 Alt 键再点击预设 = 同时设置 Anchor + Position；按住 Shift 键 = 同时设置 Anchor + Pivot。
      
        元素类型操作结果
        左上角返回按钮点左上角方块Anchor Min(0,1) Max(0,1)
        右上角设置按钮点右上角方块Anchor Min(1,1) Max(1,1)
        全屏背景点右下角的全拉伸图标Anchor Min(0,0) Max(1,1)
        居中弹窗点正中心方块Anchor Min(0.5,0.5) Max(0.5,0.5)
        底部导航栏点底行最右的横向拉伸Anchor Min(0,0) Max(1,0)
      
      每放一个 UI 元素，第一件事就是设 Anchor！否则切换分辨率全部错位。
    
  

  
    5设置 Pivot 中心点
    
      Pivot 是控件自身的旋转/缩放/定位参考点：
      
        (0.5, 0.5) → 中心（默认，常用）
        (0, 1) → 左上角
        (1, 0) → 右下角
      
      一般保持默认 (0.5,0.5) 即可。做展开动画时才需要改 Pivot。
    
  

  
    6设置 Position 和 Size
    
      点锚点模式（Anchor Min = Anchor Max）：
      
        Pos X / Pos YPivot 到 Anchor 点的偏移距离
        Width / Height控件宽高
      
      拉伸锚点模式（Anchor Min ≠ Anchor Max）：
      
        Left / Right边缘到 Anchor 边的距离
        Top / Bottom边缘到 Anchor 边的距离
      
      注意 Unity 的 Y 轴：向上为正。所以"距顶部 16px"在 top-anchor 时 Pos Y = -16。
    
  

  3.4 Image / Text / Button 参数
  
    7Image 组件
    
      
        参数说明建议
        Source ImageSprite 资源PNG 导入后在 Inspector 设 Texture Type = Sprite
        Image TypeSimple/Sliced/Tiled/Filled按钮背景用 Sliced（九宫格）
        Preserve Aspect保持宽高比图标/头像 ✅
        Raycast Target是否可点击纯装饰图片 ❌（提升性能）
      
      九宫格设置：选 Sprite → Inspector → Sprite Editor → 拖动绿色边框定义切割区域 → Apply → Image Type 设为 Sliced。
    
  

  
    8TextMeshPro 文字
    
      
        参数说明
        Text Input文字内容
        Font AssetTMP 字体（需通过 Font Asset Creator 生成）
        Font Size字号
        Vertex Color文字颜色
        Alignment水平+垂直对齐
        Wrapping → Enabled自动换行
        OverflowEllipsis=省略号 / Truncate=截断
      
    
  

  
    9Button 按钮
    
      
        参数说明
        TransitionColor Tint/Sprite Swap/Animation
        Normal/Highlighted/Pressed/Disabled Color四种状态颜色
        Navigation手柄/键盘导航（手游设 None）
      
      Button (Button + Image 背景)
  └── Text (TMP) 按钮文字
    
  

  3.5 Safe Area 安全区
  
    10Safe Area 适配脚本
    
      Unity 无内置 SafeZone 组件，通过脚本实现：
      
        在 Canvas 下创建空对象 SafeArea，Anchor 设为全屏拉伸
        挂载脚本 SafeAreaAdapter.cs：
      
      using UnityEngine;

public class SafeAreaAdapter : MonoBehaviour
{
    RectTransform panel;
    void Awake()
    {
        panel = GetComponent<RectTransform>();
        Rect safe = Screen.safeArea;
        Vector2 min = safe.position;
        Vector2 max = safe.position + safe.size;
        min.x /= Screen.width;  min.y /= Screen.height;
        max.x /= Screen.width;  max.y /= Screen.height;
        panel.anchorMin = min;
        panel.anchorMax = max;
    }
}
      然后所有 UI 内容放在 SafeArea 对象下。
    
  

  3.6 多分辨率预览
  
    11Game 视图切换分辨率
    
      
        Game 视图左上角点击分辨率下拉（默认 Free Aspect）
        点 + 添加自定义分辨率进行测试
      
      
        名称分辨率用途

        21:9 Ultra2560×1080超宽屏
        iPad 4:32048×1536方屏

  🧩 4. 核心节点与组件用法详解

  4.1 布局容器对比
  
    
      UE5 布局容器
      
        容器功能
        Canvas Panel自由定位（Anchor+Offset）
        Horizontal Box水平排列
        Vertical Box垂直排列
        Overlay叠加显示
        Grid Panel网格排列
        Size Box强制指定子控件尺寸
        Scale Box子控件等比缩放
        Scroll Box可滚动容器
        Widget Switcher页签切换（只显示一个子控件）
      
    
    
      Unity 布局组件
      
        组件功能
        Rect Transform自由定位（默认）
        Horizontal Layout Group水平排列
        Vertical Layout Group垂直排列
        Grid Layout Group网格排列
        Content Size Fitter内容自适应大小
        Layout Element覆盖单个子对象布局属性
        Aspect Ratio Fitter保持宽高比
        Scroll Rect可滚动容器
        Canvas Group统一控制透明度/交互
      
    
  

  4.2 UE5 Box Slot 子控件参数
  
    Slot 参数说明何时用
    Size → Auto按自身内容大小，不填充空间图标、固定大小按钮
    Size → Fill (权重)按权重比例填充剩余空间自适应宽度元素
    Padding子控件四周内边距控制间距
    H/V AlignSlot 内对齐方式Center/Fill 等
  
  示例：底部导航栏 5 个等分按钮

Horizontal Box
  ├── Btn_Home    (Fill=1.0)   ← 5个按钮权重相同
  ├── Btn_Shop    (Fill=1.0)      均匀占满整个宽度
  ├── Btn_Play    (Fill=1.0)
  ├── Btn_Social  (Fill=1.0)
  └── Btn_Profile (Fill=1.0)

  4.3 Unity Layout Group 参数
  
    参数说明典型值
    Padding容器内边距（上下左右）10~20
    Spacing子对象间距8~16
    Child Alignment子对象对齐方式（9种）Middle Center
    Control Child SizeLayout 是否控制子对象大小等分时 ✅ Width
    Child Force Expand子对象是否强制扩展等分时 ✅ Width
  
  示例：底部导航栏 5 个等分按钮

NavBar (Horizontal Layout Group)
  ├── Spacing: 8
  ├── Control Child Size: ✅ Width ✅ Height
  ├── Child Force Expand: ✅ Width ❌ Height
  │
  ├── Btn_Home      ← 自动等分，无需手动设宽度
  ├── Btn_Shop
  ├── Btn_Play
  ├── Btn_Social
  └── Btn_Profile

  4.4 滚动列表搭建
  
    
      UE5 Scroll Box
      Scroll Box
  ├── Orientation: Vertical
  ├── Scroll Bar Visibility: Auto
  └── [子控件直接放入即可]
      子控件自动从上到下排列，超出部分可滚动。
    
    
      Unity Scroll Rect
      Scroll View
  └── Viewport (Mask)
        └── Content
              ├── Vertical Layout Group
              ├── Content Size Fitter
              │   (Vertical=Preferred Size)
              ├── Item_1
              ├── Item_2
              └── ...
      Content 需挂 Layout Group + Content Size Fitter 自动撑高。

  🚫 7. 常见操作错误与排雷

  
    1所有元素用默认 Anchor
    
      现象：1080p 下正常，切分辨率后右边/底部元素全跑偏。
      原因：未设 Anchor，所有元素相对默认锚点绝对偏移。
      ✅ 修复：每放一个元素，第一件事设正确 Anchor。
    
  

  
    2居中弹窗的 Alignment/Pivot 没改
    
      现象：弹窗 Anchor 设了中心，但弹窗看起来偏右偏下。
      原因：UE5 Alignment 默认 (0,0)，Unity Pivot 默认 (0.5,0.5)。UE5 不改 Alignment 会导致控件左上角对齐到中心点。
      ✅ 修复：UE5 → Alignment=(0.5,0.5)；Unity 一般默认即正确。
    
  

  
    3导航栏用固定宽度而非拉伸
    
      现象：窄屏上导航栏超出屏幕，宽屏上两侧空白。
      原因：Anchor 是点锚点 + 固定 Width 值。
      ✅ 修复：Anchor 改为底部拉伸 (0,0)→(1,0)，用 Left/Right offset 而非固定宽度。
    
  

  
    4图片被拉伸变形
    
      现象：图标/头像被拉变形。
      原因：Image 拉伸模式但未锁定宽高比。
      ✅ 修复：UE5 → 勾选 Size To Content 或用 Size Box 控制；Unity → 勾选 Preserve Aspect Ratio。
    
  

  
    5刘海屏遮挡顶部按钮
    
      现象：iPhone 刘海/药丸区域遮住了返回按钮。
      原因：未使用 Safe Zone 处理。
      ✅ 修复：UE5 → 添加 SafeZone 组件包裹；Unity → 使用 SafeAreaAdapter 脚本。
    
  

  
    6Unity 中 Image 不响应点击
    
      现象：Button 的 OnClick 事件不触发。
      原因：Button 上方有其他 Image 的 Raycast Target = true，挡住了点击。
      ✅ 修复：所有纯装饰 Image（背景、分割线等）的 Raycast Target 设为 ❌。
    
  

  
    7UE5 中 Box 内元素大小异常
    
      现象：Horizontal Box 内的按钮大小与预期不一致。
      原因：子控件 Slot 的 Size 设为 Fill 但权重不对，或没有勾选 Size To Content。
      ✅ 修复：固定大小的元素用 Auto，需要自适应的用 Fill 并设置合理权重。
