  📱 1. 多分辨率适配原则

  1.1 设计基准分辨率
  项目统一基准：以 1920×1080（16:9）为设计稿基准分辨率，所有标注值基于此尺寸。

  
    目标设备分辨率范围宽高比适配策略
    iPhone SE / 小屏手机1334×75016:9等比缩放，无裁切
    iPhone 15 Pro / 主流手机2556×117919.5:9两侧留黑或扩展背景
    iPad / 平板2048×15364:3上下留黑或扩展背景
    1080P PC1920×108016:9基准，1:1 显示
    2K / 4K PC2560×1440 / 3840×216016:9DPI 缩放
  

  1.2 缩放策略
  
    🔴 铁律：禁止使用固定像素坐标定位 UI 元素。所有 UI 元素必须通过锚点（Anchor）+ 偏移量（Offset）定位。固定坐标在非基准分辨率下会导致 UI 偏移出屏幕。
  

  
    策略适用场景引擎实现
    等比缩放（Scale with Width）横屏游戏主流方案UE5: DPI Scaling Rule = Shortest SideUnity: Canvas Scaler → Scale With Screen Size
    安全区裁切（Safe Zone）异形屏（刘海/挖孔）UE5: SafeZone 组件Unity: Screen.safeArea
    固定 DPI 缩放PC 端 2K/4KUE5: DPI Scale 1.5/2.0
  




  ⚓ 2. 锚点 Anchor 打法铁律

  ⛔ 核心原则：UI 元素的 Anchor 决定了它相对于父容器哪个点定位。搞错 Anchor = 换分辨率就炸。

  2.1 三大 Anchor 模式
  
    模式Anchor 设置行为适用元素
    全屏拉伸(Stretch)Min(0,0) Max(1,1)四边始终贴合父容器边缘，大小随容器变化全屏背景、遮罩层、根 Canvas
    居中固定(Center)Min(0.5,0.5) Max(0.5,0.5)始终在父容器正中心，大小固定不变弹窗面板、Logo、主菜单
    贴边吸附(Edge Pin)锚定到对应边与指定边缘保持固定偏移距离返回按钮（左上）、设置按钮（右上）、底部导航栏
  

  2.2 贴边吸附详表
  
    UI 元素吸附位置Anchor MinAnchor Max偏移方向
    返回按钮左上角(0, 1)(0, 1)Right + Down
    设置/菜单按钮右上角(1, 1)(1, 1)Left + Down
    聊天输入框左下角(0, 0)(0, 0)Right + Up
    小地图右下角(1, 0)(1, 0)Left + Up
    血条/战斗 HUD顶部居中(0.5, 1)(0.5, 1)Down
    底部导航栏底部拉伸(0, 0)(1, 0)Up，宽度随屏幕
    侧边栏左侧拉伸(0, 0)(0, 1)Right，高度随屏幕
  

  
    🔴 铁律 #1：角落元素（返回键/设置键）必须锚定到对应角，禁止锚定到中心再偏移。否则宽屏设备上按钮会"飘"到屏幕中间区域。
    🔴 铁律 #2：底部导航栏必须使用水平拉伸锚点 Anchor(0,0)→(1,0)，禁止固定宽度。否则窄屏设备上导航栏不够宽，宽屏设备上两侧空白。
    🔴 铁律 #3：弹窗/对话框必须居中锚定 Anchor(0.5,0.5)，禁止贴边。弹窗在不同分辨率上应始终在视觉中心。
  




  ✅❌ 3. 图文红黑榜

  3.1 锚点选错 → 异形屏 UI 跑偏
  
    
      ❌ Don't — 返回按钮锚定到中心
      [此处插入图片：16:9 下返回按钮在左上角位置正确；切到 21:9 超宽屏后，返回按钮飘到了屏幕偏中间的位置，远离左上角]
      后果：Anchor 设为 Center(0.5,0.5)，偏移量 (-800,-450) 计算到左上角。在 21:9 屏幕上，中心点向右偏移，按钮也跟着偏移，不在左上角了。
    
    
      ✅ Do — 返回按钮锚定到左上角
      [此处插入图片：16:9 和 21:9 两种分辨率，返回按钮始终固定在左上角相同距离处]
      方法：Anchor = (0, 1)，Offset = (20, -20)。无论屏幕多宽多窄，按钮始终距左边 20px、距顶部 20px。
    
  

  3.2 导航栏固定宽度 → 窄屏溢出
  
    
      ❌ Don't — 底部导航栏固定 1920px 宽度
      [此处插入图片：720p 设备上导航栏超出屏幕右侧，右侧按钮不可见；4:3 iPad 上导航栏左右两侧有大块空白]
      后果：固定宽度在低分辨率上超出屏幕被裁切，在宽屏上两侧大量空白。
    
    
      ✅ Do — 导航栏水平拉伸 + 内容等分
      [此处插入图片：导航栏 Anchor(0,0)→(1,0) 水平拉伸，内部按钮均匀分布，各分辨率表现一致]
      方法：Anchor Min(0,0) Max(1,0)，固定高度，宽度自适应。内部按钮使用 HorizontalBox + 等分权重。

  ❓ 5. 常见问题排查 FAQ

  
    💬 Q1：iPad（4:3）上 UI 上下被裁切？
    
      原因：缩放策略设为 "Scale with Width"，4:3 竖向比例更大，高度不够导致上下裁切。
      修复：改用 Shortest Side 或 Expand 策略。关键信息元素不要放在顶部/底部 10% 区域内。
    
  

  
    💬 Q2：刘海屏上状态栏按钮被遮挡？
    
      原因：未考虑 Safe Zone Insets，UI 元素侵入了刘海/药丸区域。
      修复：UE5 使用 SafeZone 组件包裹顶部 UI；Unity 使用 Screen.safeArea 计算 padding。所有顶部按钮偏移量加上 Safe Zone Inset。
    
  

  
    💬 Q3：HorizontalBox 内按钮在宽屏上间距过大？
    
      原因：按钮设为等分权重（Fill），宽屏上每个按钮被拉宽，间距视觉上太大。
      修复：内容区域设最大宽度限制（Max Width = 1200px），超出部分用居中 + 两侧留白。或改用 "Auto Size" + 固定间距。
    
  




  🔄 6. 引擎差异对照表

  6.2 坐标系差异（⚠️ 必须注意）
  
    差异项UE5Unity
    Y 轴方向↓ 向下为正（与屏幕坐标一致）↑ 向上为正（数学坐标系）
    原点位置父容器左上角取决于 Anchor 位置
    "距顶部 20px"Position Y = 20（top-anchor时）Pos Y = -20
    "距底部 20px"（底部anchor）Position Y = -20（bottom-anchor时）Pos Y = 20
    对齐中心Alignment 属性 (0~1)Pivot 属性 (0~1)
  
  ⚠️ 设计稿标注转换提示：设计稿标注通常用"距顶部 Xpx"的形式。在 UE5 中直接填正值即可；在 Unity 中使用 top-anchor 时需要填负值。建议锚点标注文档统一标注"距 Anchor 方向的距离"并注明引擎填法。

  6.3 DPI 缩放策略对比
  
    
      UE5 DPI Scaling Rule 选项
      
        规则行为适用
        Shortest Side以短边为基准缩放✅ 横屏游戏首选
        Longest Side以长边为基准竖屏游戏
        Horizontal以水平方向为基准特殊需求
        Vertical以垂直方向为基准特殊需求
      
    
    
      Unity Canvas Scaler Match 值
      
        Match行为适用
        0完全匹配宽度竖屏游戏
        0.5取宽高中间值✅ 通用推荐
        1完全匹配高度✅ 横屏游戏推荐
      
    
  

  6.4 安全区实现差异
  
    
      UE5 SafeZone
      引擎内置 SafeZone 组件，直接从 Palette 拖入即可。
      Canvas Panel
  └── SafeZone (勾选 Pad 四边)
        └── 所有 UI 内容
      ✅ 开箱即用，无需写代码。
    
    
      Unity SafeArea
      需要自行挂载脚本实现：
      Canvas
  └── SafeArea (RectTransform + 脚本)
        └── 所有 UI 内容

// 脚本核心逻辑：
Rect safe = Screen.safeArea;
panel.anchorMin = safe.min / screenSize;
panel.anchorMax = safe.max / screenSize;
      ⚠️ 需要在运行时动态计算，Editor 中无法预览效果。
    
  




  📦 7. 布局容器使用规范

  7.1 容器选择决策树
  面对一个 UI 需求，按以下顺序判断该用哪种容器：
  
    问题答案 → 选择
    需要自由放置各元素到任意位置？是 → Canvas Panel / 自由定位
    元素从左到右排成一行？是 → Horizontal Box / Horizontal Layout Group
    元素从上到下排成一列？是 → Vertical Box / Vertical Layout Group
    多个元素排成网格（如道具背包）？是 → Grid Panel / Grid Layout Group
    内容超出一屏需要滚动？是 → Scroll Box / Scroll Rect
    多个子元素叠放在同一位置（如弹窗层级）？是 → Overlay / 自由定位叠放
    只显示多个子视图中的一个（页签切换）？是 → Widget Switcher / 脚本切换 Active
  

  7.2 嵌套层级规范
  
    🔴 规范：UI 控件嵌套层级不得超过 8 层。超过 8 层需重构，拆分为独立子 Widget/Prefab。
  
  
    层级深度状态处理
    1~5 层✅ 健康正常
    6~8 层⚠️ 关注评估是否可精简
    >8 层❌ 超标必须拆分子组件
  

  7.3 自适应 vs 固定尺寸决策
  
    场景策略实现方式
    导航栏、顶/底部条宽度自适应，高度固定Anchor 水平拉伸，固定 Height
    弹窗面板固定宽高，居中显示Anchor 居中，固定 Width/Height
    按钮文字文字自适应宽度UE5: Size To Content; Unity: Content Size Fitter
    背景图全屏拉伸Anchor 四边拉伸，Offset=0
    列表项宽度跟随容器，高度固定或自适应Layout Group + Layout Element
    等分按钮组均匀分配宽度UE5: Fill 权重相同; Unity: Force Expand Width
  

  7.4 间距与边距标准
  
    间距类型推荐值（基于 1080p）适用场景
    全局安全边距24~32 px内容距屏幕边缘最小距离
    组件间大间距24~32 px不同功能区块之间
    组件间中间距12~16 px同类元素之间（如按钮列表）
    组件间小间距6~8 px紧密相关元素（图标+文字）
    最小触控间距≥ 8 px两个可点击元素之间
    最小触控区域≥ 44×44 px任何可交互元素的点击区域
  




  ⚡ 8. 性能与优化准则

  8.1 UI 性能红线
  
    指标红线值说明
    单界面 Draw Call≤ 30超过需合批优化
    全屏 Overlay 数量≤ 3 层过多全屏透明层严重影响填充率
    单界面节点总数≤ 200超过需拆分子界面/虚拟列表
    嵌套层级深度≤ 8超过需重构
    动态文字节点尽量少于 20频繁变化的文字节点增加重绘开销
  

  8.2 引擎优化要点
  
    
      UE5 UMG 优化
      
        ✅ 静态元素使用 Is Volatile = false（默认）减少重绘
        ✅ 不需要 Hit Test 的元素设 Visibility = Hit Test Invisible
        ✅ 大量相同项使用 List View 替代 Scroll Box 手动堆叠
        ✅ 隐藏界面使用 Collapsed 而非 Hidden（Collapsed 不占布局也不计算）
        ❌ 避免每帧在 Tick 中更新 UI 属性，改用绑定或事件驱动
        ❌ 避免 Canvas Panel 中放大量子控件（超过 50 个考虑分层）
      
    
    
      Unity UGUI 优化
      
        ✅ 不可交互的 Image 关闭 Raycast Target
        ✅ 将频繁变动的 UI 和静态 UI 放在不同 Canvas 下（减少 Canvas Rebuild）
        ✅ 大列表使用对象池/虚拟化（如 LoopScrollRect）
        ✅ 图集打包（Sprite Atlas）减少 Draw Call
        ✅ 隐藏面板使用 CanvasGroup.alpha=0 + blocksRaycasts=false
        ❌ 避免使用 Layout Group 嵌套超过 3 层
        ❌ 避免大量 Outline/Shadow 组件（增加顶点数）
      
    
  

  8.3 图集与合批规范
  
    规范项要求原因
    同一界面图片归入同一图集✅ 必须同图集的 Image 可合并 Draw Call
    图集最大尺寸2048×2048兼容低端设备
    全屏大背景图不放入图集，独立加载避免图集过大
    通用元素（关闭按钮等）放入公共图集跨界面复用
    九宫格图片越小越好（建议 ≤ 64×64）节省内存
  

  8.4 多语言适配注意事项
  
    问题规范要求
    文字容器大小预留 30%~50% 的空间余量（德/法/俄语普遍比中/英长）
    按钮宽度建议使用自适应宽度而非固定宽度
    文字截断必须设置 Overflow 处理（Ellipsis 省略号）
    从右到左（RTL）如需支持阿拉伯语，布局容器需支持镜像翻转
    竖排文字日/韩特殊场景，需专门处理
