  🗂️ 1. UMG 常用控件分类总览
  在开始拼接界面前，我们先梳理 UE5 中最常用的控件类型，它们是你搭建 UI 的积木：
  
  🔹 基础与交互控件 (Common)
  
    
      🖼️
      Image (图像)显示背景、图标及九宫格
    
    
      👆
      Button (按钮)提供点击与悬停状态切换
    
    
      🔤
      Text (文本)显示文字、标题或动态数值
    
    
      ⌨️
      Editable Text (输入框)允许玩家输入文字或密码
    
  

  🔹 布局与容器面板 (Panels)
  
    
      🎨
      Canvas Panel (画布)依靠锚点自由拖拽摆放控件
    
    
      🥞
      Overlay (覆层)将子节点层叠摆放，性能极佳
    
    
      ⏸️
      Horizontal Box (水平框)子节点自动横向从左排到右
    
    
      ⏬
      Vertical Box (垂直框)子节点自动纵向从上排到下
    
  

  🔹 高级列表与滚动 (Lists)
  
    
      ↕️
      Scroll Box (滚动框)基础容器，用于少量静态内容超出滚动
    
    
      ♾️
      List View (列表视图)高性能动态复用列表（处理成千上百条）
    
    
      🔲
      Tile View (瓦片视图)高性能网格复用列表（如背包格子）
    
  




  🛠️ 2. 基础交互控件详解与参数对照
  选中任何图层后，参数面板会出现该图层的各项参数，这里为你详解配置它们的最标准方式。

  2.1 图像控件 (Image) 参数详解
  
  
    
    
      
        
          
            ▼ 插槽 (画布插槽)
            
              锚点 (Anchors)锚点 ▼
              位置X / Y0.0
              尺寸X / Y100.0
              大小到内容☑
              ZOrder1
            
          
        
      
      ➔
      
        改变控件的位置和尺寸：通过修改“位置 X/Y”和“尺寸 X/Y”直接控制它在界面中的绝对坐标和长宽。
        固定尺寸至理想大小 (大小到内容)：非常重要。如果是图片，点选这个勾选框，你的美术切图是多大分辨率，界面里直接呈现同样的一比一大小，无需手动量。
        ZOrder (层级)：控制该图层的视觉渲染顺序。数值越大，越在顶层显示（避免后面的层挡住前面的）。
      
    

    
    
      
        
          
            ▼ 外观
            Brush -> 图像T_Bg_01 (资产)
            
              绘制为 (Draw As)盒体 ▼
              Margin (边界)0.25
            
          
        
      
      ➔
      
        怎么替换图片？
        方法一：选中外部内容浏览器的图片，点击这里的向左小箭头即可赋值。
        方法二：拖拽外部文件，悬停在 UI 选项卡等界面出现后扔进去。
        绘制为 (Draw As) —— 九宫格核心！
        • 无：就是空集。
        • 盒体 (Box)：做“九宫格”必须选它！配合下方的 Margin（例如设为 0.25）以保护边角在拉伸时不变形。
        • 图像 (Image)：默认拉伸选项。如果不做九宫格处理，默认选这个即可。

            R / G / B1.0
            
              A (Alpha)0.5
            
          
          
            ▼ 渲染变换
            倾斜角度X: 0.0    Y: 0.0

      ➔
      
        调整颜色与不透明度：单击色块进入调色板，可随意调节 RGBA。色号规则跟 Photoshop 是互通的。
        其中 A 为不透明度：0 表示 0%（全透明），1 表示 100%（全实色）。
        渲染变换：可以通过“倾斜角度”和“旋转角度”做一些基于原点的几何变换（常用于动效或斜体标签）。
      
    
  

  2.2 按钮控件 (Button) 状态详析
  按钮自带交互响应，我们需要为其不同的交互状态填充图片，以实现按压反馈。

  
    
      
        
          
            ▼ 样式 (Style)
            ▶ 普通 (Normal)Btn_Base
            ▶ 已悬停 (Hovered)Btn_Hover
            ▶ 已按压 (Pressed)Btn_Press
            ▶ 已禁用 (Disabled)Btn_Base
          
        
      
      ➔
      
        ▶ 普通：按钮正常的静止状态。
        ▶ 已悬停：鼠标指针放在按钮上方时的状态（移动端不常用，PC端必备）。
        ▶ 已按压：鼠标点击/手指触摸按下去瞬间的状态。
        ▶ 已禁用：当你在逻辑里把它设为“不可用”时显示的灰化状态。
        注：如果按钮没有提供多张图片作为点击反馈，务必这几个状态全用同一张普通图拖入覆盖！不可以留空！否则点击时会出现默认的亮瞎眼白块！
      
    
  

  2.3 文本与输入控件
  
    
      🔡 可编辑文本 (Editable Text)
      用于单行输入（如注册账号、兑换码）。支持设置 Is Password（星号遮罩）。
    
    
      🔠 多行可编辑文本 (Multi-Line)
      用于大段输入（如意见反馈、公会宣言）。不支持密码，但支持内容自动到边缘换行 (Auto Wrap Text)。
    
  




  📦 3. 布局与容器面板详解
  
    
      🖼️ 画布面板 (Canvas Panel)
      作用： 允许自由拖拽摆放。依靠 Anchor (锚点) 定位。
      
        适用场景： 界面最底层的根容器；右下角自由摆放的技能球。
        缺点： 计算复杂，绝对不要在列表内部套娃使用 Canvas。
      
    
    
      🥞 覆层 (Overlay)
      作用： 控件像汉堡一样叠放，依靠水平/垂直对齐(Alignment)排版。
      
        适用场景： 头像底图叠上头像再叠上边框。
        优点： 性能极高！能用 Overlay 解决的层叠，绝对不用 Canvas。
      
    
  




  📜 4. 高级列表与滚动控件
  
    
      ↕️ Scroll Box (基础滚动框)
      超出范围自动出现滚动条。由于会把所有子节点同时渲染，所以千万不要用来装几百条数据，会严重掉帧。适合做简单的设置界面。
    
    
      ♾️ List View & Tile View (高性能)
      采用对象池动态复用原理。只渲染屏幕看得见的那几个格子，滑动时复用外面的。适合处理背包、邮件等几百上千条的数据。必须通过指定 Entry Widget Class 蓝图来渲染子条目。
    
  




  👁️ 5. 通用核心属性：可见性与变换
  
  可见性 (Visibility) —— 极度重要！
  
    可见性选项视觉状态是否挡击穿透？性能消耗最适用场景
    可见 (Visible)👀 看得见👆 挡击/可点高普通按钮、面板
    折叠 (Collapsed)❌ 看不见❌ 彻底消失极低隐藏UI首选！完全不计算也不占空间。
    隐藏 (Hidden)❌ 看不见❌ 无法点击中看不见但仍然占据排队占位空间。
    不可见命中测试(Hit Test Invisible)👀 看得见👻 穿透(不可点)低纯装饰的背景底图！不挡住下方的按钮。
    自身不可见命中测试(Self Hit Test Invisible)👀 看得见👻 自身穿透👆 子可点低作为父容器（Canvas/Overlay）的最佳选择！
