  🛠️ 2. 基础图形与交互 (Image & Button)

  2.1 Button (按钮) — 极深解析
  💡 核心作用 (Core Function)： 提供给玩家点击交互的载体。具备普通 (Normal)、悬停 (Hovered)、按压 (Pressed)、禁用 (Disabled) 四种视觉状态反馈。
  🎯 适用场景 (Use Cases)： 确认/取消按键、关闭叉叉、底部的页签切换按钮、技能释放图标。

  右侧面板详细参数清单 (Details Panel Parameters)
  
    
      面板分组 (Group)层级与具体参数名 (Parameter)效果与如何调节 (Description)即时预览 (Preview)
    
    
      
        Slot (插槽)以 Horizontal Box Slot 为例
        Padding (填充)▶ Left, Top, Right, Bottom
        控制该按钮距离父容器边缘或其他兄弟节点的四周空隙像素值。比如 Left 填 10，它就会和左边的元素拉开 10 像素距离。
        ✅ 立刻刷新
      
      
        Size (尺寸)
        
          
            Auto (自动): 按钮多大，它就占多大物理空间。
            Fill (填充): 按钮强制放大，吃掉父容器所有剩余空间。如果有多个填充，按比例平分。
          
        
        ✅ 立刻刷新
      
      
        Horizontal Alignment (水平对齐)
        当分配的空间比按钮大时，控制按钮是靠左、居中、靠右，还是拉伸填满 (Fill) 整个宽度。
        ✅ 立刻刷新
      
      
        Vertical Alignment (垂直对齐)
        控制按钮靠上、居中、靠下，或是拉伸填满 (Fill)。
        ✅ 立刻刷新
      
      
        
          ⬇ 以下为 Appearance (外观) -> Style (样式) 层层展开解析 ⬇
        
      
      
        Style (样式)▶ Normal (普通)其余状态参数一致
        Image (图像)
        按键静止状态下的图片切图贴图。
        ✅ 立刻刷新
      
      
        Image Size (图像大小)X / Y
        强行定义这张图片的逻辑基准渲染宽高（不是父框体的大小，而是图片本身的缩放基准）。
        ✅ 立刻刷新
      
      
        Style (样式)▶ Normal (普通)▶ (继续展开)
        Image Design / Actual Size (设计尺寸/实际大小)
        只读属性，展示美术最初画原图时候的像素分辨率，供你参考用。
        -
      
      
        Tint (着色)
        给按钮图叠一层有色滤镜。默认白色=原图。改成半透明黑色，图片就会变暗。
        ✅ 立刻刷新
      
      
        Draw As (绘制为)
        
          
            Box (盒体): 最核心！做九宫格弹窗或圆角按钮不拉伸变形的必选！
            Image (图像): 默认拉伸变形。
          
        
        ✅ 立刻刷新
      
      
        Margin (边缘)
        仅在 Draw As = Box 时生效。填入 0.1~0.4，保护图片四角的圆角不被拉伸破坏。
        ✅ 立刻刷新
      
      
        Style (样式)
        Hovered / Pressed / Disabled (悬停/已按压/禁用)
        
          【致命避坑】：如果 Pressed (已按压) 里面不填图片，当玩家点击按钮时，它会瞬间变成一个全白的正方形方块！务必把 Normal 的图拷过来！
        
        ❌ 播放运行
      
      
        Interaction (交互)
        Is Focusable (可聚焦)
        【避坑】PC纯鼠标或手机游戏必须取消勾选，否则点击后周围会有一圈诡异的虚线！主机手柄才需勾选。
        ❌ 播放运行
      
      
        Click Method (点击方法)
        通常设为 Down And Up（按下并抬起后才算触发），防止玩家反悔滑动取消。
        ❌ 播放运行

  🎯 适用场景 (Use Cases)： 纯色背景底板、技能图标、带圆角的九宫格弹窗框、装饰性花纹。

        ✅ 立刻刷新

  📦 5. 布局容器面板 (Panels & Containers)

  5.1 Scroll Box (滚动框) — 极深解析
  💡 核心作用 (Core Function)： 给那些内部内容远远超过屏幕物理尺寸的元素提供一个滚动视口。玩家可以用鼠标滚轮或手指上下划动。
  🎯 适用场景 (Use Cases)： 长篇大论的用户协议、带大量文字的装备详细描述说明、不经常变化长短的设置项列表。
  ⚠️ 注意：如果列表条目超过 30 条且格式一致（如排行榜），绝对不能用 Scroll Box，必须用 List View 性能复用池！

  
    
      
        
          
            ▼ Scroll (滚动)
            朝向 (Orientation)垂直 (Vertical) ▼
            滚动条可视性 (Scrollbar Visibility)已折叠 (Collapsed) ▼
            消耗鼠标滚轮 (Consume Mouse Wheel)总是 (Always) ▼
          
        
      
      ➔
      
        Scrollbar Visibility (滚动条可视性) 秘籍：
        很多设计图是不画丑陋滚动条的，但是内容又要能划动。千万别选 Hidden（隐藏），因为隐藏后其实它还在右侧暗中占据空间！
        正确做法：选择「已折叠 (Collapsed)」，这样滚动条完全消失，内部空间自动向右填满，且用鼠标滚轮和手指依然能丝滑滚动！
      
    
  

  右侧面板详细参数清单 (Details Panel Parameters)
  
    
      面板分组 (Group)具体参数名 (Parameter)效果与如何调节 (Description)即时预览 (Preview)
    
    
      
        Slot (插槽)以 Canvas Panel Slot 为例
        Anchors (锚点)Min / Max

        Draw As (绘制为)
        Image(图像)：拉伸渲染。Box(盒体)：九宫格保护圆角。None(无)：不绘制。阴影笔刷一般用 Image 即可。

        垂直顶部插槽图像 (Vertical Top Slot Image)
        滚动条轨道最顶部的装饰图片（如箭头或圆角帽）。大多数游戏留空即可。
        ✅ 立刻刷新
      
      
        水平顶部插槽图像 (Horizontal Top Slot Image)

        ✅ 立刻刷新
      
      
        垂直底部插槽图像 (Vertical Bottom Slot Image)

        ✅ 立刻刷新

        ✅ 立刻刷新
      
      
        法线缩略图 / 悬浮缩略图 / 拖动缩略图(Normal/Hovered/Dragged Thumb Image)
        中间那个可拖动的小滑块的三态图片。Normal=日常状态；Hovered=鼠标悬停变亮；Dragged=正在拖动时。替换这三张图即可自定义滑块外观。
        ✅ 立刻刷新
      
      
        Style (样式)▶ 条样式▶ 每个图像的子属性
        Image (图像)
        每个笔刷/背景/缩略图的贴图资源引用。点击右侧放大镜搜索项目素材。

        Image Size (图像大小)X / Y
        该图片的逻辑像素渲染宽高。注意这不是父容器的尺寸，而是图片自身的缩放基准。
        ✅ 立刻刷新
      
      
        Tint (着色)
        叠色滤镜。白色=原图不变。可以用来做同一张素材出不同配色变体（如灰色滑块→高亮白色滑块）。
        ✅ 立刻刷新
      
      
        Draw As (绘制为) + Margin (边距)

        ✅ 立刻刷新
      
      
        Scroll (滚动)
        Orientation (朝向)
        横向滑动 (Horizontal) 还是竖向滑动 (Vertical)。
        ✅ 立刻刷新

            Collapsed (折叠): 彻底隐藏滑块并把右侧空间让出来，但依然可以通过手指滑动。
          
        
        ✅ 立刻刷新
      
      
        Consume Mouse Wheel (消耗鼠标滚轮)
        当鼠标在这个框里滚轮时，是否拦截事件。如果你外层还有一个大滚动条，设为"从不"可以让内层不吃掉滚轮指令。
        ❌ 播放运行
      
      
        Scrollbar Thickness (滚动条厚度)
        如果需要极其纤细的滑动条，调小这个数值 (X和Y)。

  5.4 Widget Switcher (控件切换器)
  💡 核心作用 (Core Function)： 同一个位置放多个子控件，但同时只显示其中一个。通过切换"活跃索引"来实现页签/抽屉式切换，无需反复创建删除控件。
  🎯 适用场景 (Use Cases)： 底部页签栏（商城/背包/社交切页）、状态面板切换（详情/属性/技能子页面）、加载中↔内容区域切换。

  右侧面板详细参数清单 (Details Panel Parameters)
  
    
      面板分组 (Group)具体参数名 (Parameter)效果与如何调节 (Description)即时预览 (Preview)
    
    
      
        Active Widget (活跃控件)
        Active Widget Index (活跃索引)
        核心！决定当前显示第几个子控件（从0开始计数）。通过蓝图 SetActiveWidgetIndex 动态切换页签。
        ✅ 立刻刷新
      
      
        Active Widget (活跃控件引用)
        也可以通过传入具体的子控件引用来切换，而不是用索引号。蓝图节点 SetActiveWidget。
        ✅ 立刻刷新
      
      
        Transition (过渡)
        Transition Type (过渡类型)
        切换时的动画效果：None(无) 瞬切、Fade(淡入淡出)、Horizontal/Vertical(水平/垂直滑动)。一般游戏选 None 或 Fade 即可。
        ✅ 立刻刷新
      
      
        Behavior (行为)
        Is Widget Switcher Variable (公开为变量)
        勾选"Is Variable"后才能在蓝图中拖出来调用 SetActiveWidgetIndex。必须勾选！
        ❌ 播放运行
      
    
  
  💡 避坑：Widget Switcher 的所有子控件在创建时就已经全部实例化了（即使没显示的也在内存里）。如果你有 20 个重量级页面都塞进同一个 Switcher，可能吃爆内存。超过5个子页面时建议结合代码动态加载。

  5.5 Horizontal Box / Vertical Box (水平框/垂直框)
  💡 核心作用 (Core Function)： UMG 中最常用的线性排布容器。Horizontal Box 让子节点从左到右自动排队，Vertical Box 让子节点从上到下自动排队。
  🎯 适用场景 (Use Cases)： 按钮行排列、表单标签+输入框横排、设置项纵向列表、底部导航栏图标横排。

  右侧面板详细参数清单 (Details Panel Parameters)
  
    
      面板分组 (Group)层级与具体参数名 (Parameter)效果与如何调节 (Description)即时预览 (Preview)
    
    
      
        Slot (子节点插槽)每个子节点都有
        Padding (填充)Left / Top / Right / Bottom
        子节点与相邻兄弟/容器边缘的间距像素值。等效于 CSS margin。
        ✅ 立刻刷新
      
      
        Size (尺寸) → Auto / Fill
        Auto：子节点按自身内容大小占位。Fill：子节点吃掉父容器的所有剩余空间（多个 Fill 按比例瓜分 Fill Ratio）。
        ✅ 立刻刷新
      
      
        Horizontal / Vertical Alignment (对齐)
        当子节点比分配空间小时，控制它在格子内靠左/居中/靠右/拉伸填满（Horizontal Box 对应垂直方向对齐，反之亦然）。
        ✅ 立刻刷新
      
      
        Fill Ratio (填充比例)
        仅 Size=Fill 时生效。多个 Fill 子节点之间的空间瓜分比例。A 设 1、B 设 2，则 B 占 2/3 空间。
        ✅ 立刻刷新
      
    
  
  🔑 核心理解：Horizontal/Vertical Box 本身没有什么参数，它的核心全在每个子节点的 Slot 属性上。拖入子控件后，选中子控件才能看到 Slot 相关设置。

  5.6 Overlay (覆层)
  💡 核心作用 (Core Function)： 将所有子控件层叠在同一个区域（像 PS 图层那样堆叠）。列表越靠后的子控件越在上面。
  🎯 适用场景 (Use Cases)： 头像+等级徽章叠放、背景图+前景内容的分层、角色模型上叠加血条和名字。
  ⚡ 性能优势：Overlay 不做任何排列计算，只是简单堆叠，性能开销比 Canvas Panel 低得多，能用 Overlay 就不用 Canvas Panel。

  右侧面板详细参数清单 (Details Panel Parameters)
  
    
      面板分组 (Group)具体参数名 (Parameter)效果与如何调节 (Description)即时预览 (Preview)
    
    
      
        Slot (子节点插槽)每个子节点都有
        Padding (填充)
        子控件相对于 Overlay 容器四边的内距。可以用来微调某个图层的位置偏移。
        ✅ 立刻刷新
      
      
        Horizontal / Vertical Alignment (对齐)
        每个子控件在 Overlay 中的水平/垂直对齐：Left/Center/Right/Fill × Top/Center/Bottom/Fill。Fill = 拉伸铺满整个 Overlay。
        ✅ 立刻刷新
      
    
  
  💡 技巧：Overlay 中第一个子节点（最底层）一般是铺满的背景 Image，设 Alignment 为 Fill×Fill。后续的子节点选择 Center×Center 居中摆放。这就是最经典的"背景+前景"两层结构。

  👁️ 7. 通用核心属性：可见性与变换 (Visibility & Transform)
  所有控件都有可见性和渲染变换属性，这是 UI 交互的基础：
  
    
      可见性选项 (Visibility Options)视觉状态 (Visual State)点击状态 (Hit State)性能消耗 (Cost)最适用场景 (Best Use Case)
    
    
      Visible (可见)👀 看得见👆 挡击/可点高 (High)普通可交互按钮、弹窗面板。
      Collapsed (折叠)❌ 看不见❌ 完全消失极低 (Very Low)隐藏 UI 首选！完全不计算也不占据排队槽位空间。
      Hidden (隐藏)❌ 看不见❌ 无法点击中 (Medium)看不见，但仍然会占据排队槽位空间（占坑）。
      Hit Test Invisible(不可见命中测试)👀 看得见👻 鼠标穿透低 (Low)纯装饰用的背景图或文字！不抢焦点，不挡底下的按钮。

  8.2 对齐 (Alignment) —— 绝对居中的密码
  当你要把弹窗绝对居中时，光把锚点设为"居中"是不够的！因为那只是以控件左上角对准屏幕中心。
  【核心铁律】必须同时将 Alignment (对齐) 设为 X: 0.5, Y: 0.5，这代表用控件自己的正中心去贴合锚点，实现完美居中。
