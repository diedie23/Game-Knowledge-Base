  🛠️ 2. 基础图形与交互（Image & Button）

  2.1 Button (按钮控件) — 极深解析
  💡 核心作用： 提供给玩家点击交互的载体。具备正常、悬停、按压、禁用四种视觉状态反馈。
  🎯 适用场景： 确认/取消按键、关闭叉叉、底部的页签切换按钮、技能释放图标。

  右侧面板详细参数清单 (完全拆解)
  
    
      面板分组层级与具体参数名效果与如何调节即时预览
    
    
      
        插槽 (Slot)(以水平框插槽为例)
        填充 (Padding)▶ 左边, 顶部, 右边, 底部
        控制该按钮距离父容器边缘或其他兄弟节点的四周空隙像素值。比如左边填 10，它就会和左边的元素拉开 10 像素距离。
        ✅ 立刻刷新
      
      
        尺寸 (Size)
        
          
            自动 (Auto): 按钮多大，它就占多大物理空间。
            填充 (Fill): 按钮强制放大，吃掉父容器所有剩余空间。如果有多个填充，按比例平分。
          
        
        ✅ 立刻刷新
      
      
        水平对齐 (Horizontal)
        当分配的空间比按钮大时，控制按钮是靠左、居中、靠右，还是拉伸填满整个宽度。
        ✅ 立刻刷新
      
      
        垂直对齐 (Vertical)
        控制按钮靠上、居中、靠下，或是垂直拉伸填满。
        ✅ 立刻刷新
      
      
        
          ⬇ 以下为 外观 (Appearance) -> 样式 (Style) 层层展开解析 ⬇
        
      
      
        样式 (Style)▶ 普通 (Normal)(其余状态Hover/Pressed/Disabled参数完全一致)
        图像 (Image)
        按键静止状态下的图片切图贴图。
        ✅ 立刻刷新
      
      
        图像大小 (Image Size)X / Y
        强行定义这张图片的逻辑基准渲染宽高（不是父框体的大小，而是图片本身的缩放基准）。
        ✅ 立刻刷新
      
      
        样式 (Style)▶ 普通 (Normal)▶ (继续展开)
        Image Design/Actual Size
        只读属性，展示美术最初画原图时候的像素分辨率，供你参考用。
        -
      
      
        着色 (Tint)
        给按钮图叠一层有色滤镜。默认白色=原图。改成半透明黑色，图片就会变暗。
        ✅ 立刻刷新
      
      
        绘制为 (Draw As)
        
          
            盒体 (Box): 最核心！做九宫格弹窗或圆角按钮不拉伸变形的必选！
            图像 (Image): 默认拉伸变形。
          
        
        ✅ 立刻刷新
      
      
        边缘 (Margin)
        仅在 Draw As=Box 时生效。填入 0.1~0.4，保护图片四角的圆角不被拉伸破坏。
        ✅ 立刻刷新
      
      
        样式 (Style)
        已悬停 / 已按压 / 已禁用
        
          【致命避坑】：如果已按压 (Pressed) 里面不填图片，当玩家点击按钮时，它会瞬间变成一个全白的正方形方块，极其难看！务必把 Normal 的图拷过来！
        
        ❌ 播放运行
      
      
        交互 (Interaction)
        可聚焦 (Is Focusable)
        【避坑】PC纯鼠标或手机游戏必须取消勾选，否则点击后周围会有一圈诡异的虚线！主机手柄才需勾选。
        ❌ 播放运行
      
      
        点击方法 (Click Method)
        通常设为 Down And Up（按下并抬起后才算触发），防止玩家反悔滑动取消。
        ❌ 播放运行
      
    
  

  2.2 Image (图像控件)
  💡 核心作用： 界面中最基础的视觉元素，用于渲染一切位图贴图。
  🎯 适用场景： 纯色背景底板、技能图标、带圆角的九宫格弹窗框、装饰性花纹。

  右侧面板详细参数清单
  
    
      面板分组具体参数名效果与如何调节即时预览
    
    
      
        外观 (Appearance)
        Brush -> 图像 (Image)
        选择美术导入的纹理切图资源。
        ✅ 立刻刷新
      
      
        绘制为 (Draw As)
        盒体 (Box): 必须配合 Margin 使用（做九宫格不拉伸四角）。图像 (Image): 默认平铺满容器。无 (None): 空白不渲染。
        ✅ 立刻刷新
      
      
        边界 (Margin)
        只有在 Draw As = Box 时有效。通常取值 0.1~0.4，代表保护图片四边多大比例不被拉伸。
        ✅ 立刻刷新
      
      
        图像大小 (Image Size)
        原图的物理宽高参考值，通常靠外层插槽改尺寸，这里不动。
        ✅ 立刻刷新
      
      
        颜色和不透明度
        Tint (色相)
        纯白(1.0, 1.0, 1.0)代表原图颜色。通过修改 Tint 可以给贴图染色。
        ✅ 立刻刷新
      
      
        A (Alpha) 不透明度
        0 表示全透明看不见，1 表示全实色。
        ✅ 立刻刷新

        行为 (Behavior)
        Auto Wrap Text

  📦 5. 布局容器面板（ScrollBox、安全区、切换器等）

  5.1 Scroll Box (滚动框) — 极深解析
  💡 核心作用： 给那些内部内容远远超过屏幕物理尺寸的元素提供一个滚动视口。玩家可以用鼠标滚轮或手指上下划动。
  🎯 适用场景： 长篇大论的用户协议、带大量文字的装备详细描述说明、不经常变化长短的设置项列表。
  ⚠️ 注意：如果列表条目超过 30 条且格式一致（如排行榜），绝对不能用 Scroll Box，必须用 List View 性能复用池！

  
    
      
        
          
            ▼ 滚动 (Scroll)
            朝向垂直 ▼
            滚动条可视性已折叠 (Collapsed) ▼
            消耗鼠标滚轮总是 (Always) ▼
          
        
      
      ➔
      
        滚动条可视性 (Scrollbar Visibility) 秘籍：
        很多设计图是不画丑陋滚动条的，但是内容又要能划动。千万别选 Hidden（隐藏），因为隐藏后其实它还在右侧暗中占据空间！
        正确做法：选择「已折叠 (Collapsed)」，这样滚动条完全消失，内部空间自动向右填满，且用鼠标滚轮和手指依然能丝滑滚动！
      
    
  

  右侧面板详细参数清单
  
    
      面板分组具体参数名效果与如何调节即时预览
    
    
      
        插槽 (Slot)(以画布面板插槽为例)
        锚点 (Anchors)最小/最大
        决定这个滚动框四条边的受力点。全屏滚动面板通常设为四角拉伸。
        ✅ 立刻刷新
      
      
        偏移 (Offsets)左侧, 位置Y, 尺寸等
        决定滚动框的绝对坐标和宽高。注意：开启“大小到内容(Size to Content)”后，宽高设置无效，会被内部撑开。
        ✅ 立刻刷新
      
      
        样式 (Style)
        顶部/底/左/右阴影笔刷 (Shadow Brushes)
        高级打磨必备：列表没滚动到底时，边缘显示的黑灰色渐变遮罩。这里可以替换遮罩渐变图或调整颜色，让文字有一种“被切断进入暗区”的质感。
        ✅ 立刻刷新
      
      
        条样式 (Scrollbar Style)
        展开后非常长。里面可以依次替换 垂直/水平背景图 (滚动槽的底色) 以及各种 缩略图 (Thumb Image) (就是中间那个拖动条)。
        ✅ 立刻刷新
      
      
        滚动 (Scroll)
        朝向 (Orientation)
        横向滑动 (Horizontal) 还是 竖向滑动 (Vertical)。
        ✅ 立刻刷新
      
      
        滚动条可视性 (Visibility)
        
          
            可见 (Visible): 长期显示滑块。
            折叠 (Collapsed): 彻底隐藏滑块并把右侧空间让出来，但依然可以通过手指滑动。
          
        
        ✅ 立刻刷新
      
      
        消耗鼠标滚轮
        当鼠标在这个框里滚轮时，是否拦截事件。如果你外层还有一个大滚动条，设为“从不”可以让内层不吃掉滚轮指令。
        ❌ 播放运行
      
      
        滚动条厚度 (Thickness)
        如果需要极其纤细的滑动条，调小这个数值 (X和Y)。
        ✅ 立刻刷新
      
    
  

  5.2 Safe Zone (安全区)
  右侧面板详细参数清单
  
    
      面板分组具体参数名效果与如何调节
    
    
      
        安全区 (Safe Zone)
        Pad Left / Right (左右保护)
        勾选后系统识别底层OS，自动挤开手机左侧刘海和右侧微信小游戏胶囊按键。强烈建议全打勾。

  5.3 Widget Switcher (控件切换器)
  💡 核心作用： 抽屉式翻页。内部有 N 个面板但同时只渲染 1 个，通常和顶部的页签组合使用。
  右侧面板详细参数清单
  
    
      面板分组具体参数名效果与如何调节即时预览
    
    
      
        切换器 (Switcher)
        Active Widget Index
        激活的子面板索引号（从 0 开始算第一页）。修改此值在编辑器中会立刻翻页预览效果！
        ✅ 立刻刷新
      
    
  

  5.4 Size Box (尺寸框)
  💡 核心作用： 在无法自由控制宽高的自动排队容器里，给子节点套上一件强制锁死物理大小的紧身衣。
  右侧面板详细参数清单
  
    
      面板分组具体参数名效果与如何调节即时预览
    
    
      
        子布局 (Child Layout)
        Width/Height Override
        无视一切环境，强行把里面的元素撑到绝对的数值大小（如 150x50）。
        ✅ 立刻刷新
      
      
        Min Desired Width/Height
        保底尺寸。比如设置 100宽，即使里面文字只有一个字，框的宽度也绝不会小于 100。
        ✅ 立刻刷新
      
      
        Max Desired Width/Height
        做自适应聊天气泡的核心参数。设为 400 宽，当文字继续变多时，不会无限变宽，而是在 400 处强制往下换行。
        ✅ 立刻刷新
      
    
  

  5.5 Horizontal/Vertical Box (排队框) 与 Overlay (覆层)
  💡 Box 的核心： 子节点顺向排队，拒绝手工对坐标。
  💡 Overlay 的核心： 子节点以 Z 轴重叠，专门用于汉堡包式叠加（底图层+图标层+数字层）。
  被放进去后的子节点专属插槽(Slot)参数清单
  
    
      插槽环境具体参数名效果与如何调节
    
    
      
        身处 H/V Box 中
        大小 (Size)
        
          
            Auto (自动): 自己有多大就占多少排队位。
            Fill (填充): 动态数值比例分赃！如果 A填1，B填2，那剩余空间按 1:2 分割强行拉伸组件填充框体。
          
        
      
      
        Padding (边距)
        控制这个排队的家伙与其他兄弟节点之间的间隔缝隙像素。
      
      
        身处 Overlay 中
        水平/垂直 对齐
        四向对齐，或选择拉伸(Fill)。选满铺(Fill)会直接盖住整个底层。
      
    
  




  ♾️ 6. 高级列表视图（List View / Tile View）

  👁️ 7. 通用核心属性：可见性 (Visibility) 与 变换
  所有控件都有可见性和渲染变换属性，这是 UI 交互的基础：
  
    
      可见性选项视觉状态点击状态性能消耗最适用场景
    
    
      可见 (Visible)👀 看得见👆 挡击/可点高普通可交互按钮、弹窗面板。
      折叠 (Collapsed)❌ 看不见❌ 完全消失极低隐藏UI首选！完全不计算也不占排队空间。
      隐藏 (Hidden)❌ 看不见❌ 无法点击中看不见，但仍然会占据排队槽位空间（占坑）。
      不可见命中测试(Hit Test Invisible)👀 看得见👻 鼠标穿透低纯装饰用的背景图或文字！不抢焦点，不挡底下的按钮。
      自身不可见命中测试(Self Hit Test Invisible)👀 看得见👻 自身穿透👆 子可点低作为父容器（Canvas/Overlay/SizeBox等）的最佳选择！
    
  
  避坑指南：点不动按钮？99%是因为上面盖了张 Visible 的全屏透明图，把它改成 Hit Test Invisible！

  8.2 对齐 (Alignment) —— 绝对居中的密码
  当你要把弹窗绝对居中时，光把锚点设为“居中”是不够的！因为那只是以控件左上角对准屏幕中心。
  【核心铁律】必须同时将 Alignment (对齐) 设为 X: 0.5, Y: 0.5，这代表用控件自己的正中心去贴合锚点，实现完美居中。
