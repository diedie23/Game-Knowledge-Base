  🛠️ 2. 基础图形与交互（Image & Button）

  2.1 图像控件 (Image)
  💡 核心作用： 界面中最基础的视觉元素，用于渲染一切位图贴图。
  🎯 适用场景： 纯色背景底板、技能图标、带圆角的九宫格弹窗框、装饰性花纹。

  
    
      
        
          
            ▼ 外观 (Appearance)
            Brush -> 图像T_Bg_01
            
              绘制为 (Draw As)盒体 ▼
              Margin (边界)0.25
            
          
        
      
      ➔
      
        绘制为 (Draw As) —— 九宫格核心！
        做带圆角的弹窗底板时，拉伸会变形。必须将 Draw As 设为 盒体 (Box)，并将 Margin 设为图片圆角的安全百分比（如 0.25，代表保护四周 25% 不被拉伸）。
      
    
  

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    外观 (Appearance)Brush -> 图像 (Image)选择美术导入的纹理切图资源。✅ 立刻刷新
    绘制为 (Draw As)盒体 (Box): 必须配合 Margin 使用（做九宫格不拉伸四角）。图像 (Image): 默认拉伸。无 (None): 空白不渲染。✅ 立刻刷新
    边界 (Margin)只有在 Draw As = Box 时有效。通常取值 0.1~0.4，代表保护图片四边多大比例不被拉伸。✅ 立刻刷新
    图像大小 (Image Size)原图的物理宽高参考值，通常不需要在这里修改，而是靠布局插槽改尺寸。✅ 立刻刷新
    颜色和不透明度Tint (色相)纯白(1.0, 1.0, 1.0)代表原图颜色。通过修改 Tint 可以给白色贴图染色。✅ 立刻刷新
    A (Alpha) 不透明度0 表示全透明看不见，1 表示全实色。✅ 立刻刷新
  


  2.2 按钮控件 (Button)
  💡 核心作用： 提供给玩家点击交互的载体。
  🎯 适用场景： 确认/取消按键、关闭叉叉、技能释放图标。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    样式 (Style)普通 (Normal)按钮静止状态下的图片或颜色。✅ 立刻刷新
    已悬停 (Hovered)鼠标移动到上方时的图片（常用于PC端）。移动端可不管。❌ 需播放运行
    已按压 (Pressed)点击瞬间的图片。【必填项】若无点击图，务必将 Normal 图贴进来，绝不可留空，否则点击时闪瞎眼白块！❌ 需播放运行
    已禁用 (Disabled)按钮设为不可用时的灰化图片。✅ 立刻刷新
    交互 (Interaction)可聚焦 (Is Focusable)【避坑】PC/主机游戏用手柄才需勾选。纯鼠标或手机游戏强烈建议取消勾选，否则点击后按钮周围会有一圈虚线框！❌ 需播放运行
    点击方法 (Click Method)默认是 Down And Up（按下并抬起才算点击成功），符合大部分人类直觉。❌ 需播放运行
  




  🔤 3. 文本双雄（LGText 与 Rich Text）

  3.1 LGText (项目定制文本)
  💡 核心作用： 替代 UE 默认 Text 的首选，底层已集成我们项目的多语言(Localization)规范。
  🎯 适用场景： 各种界面标题、按钮上的文案、数值、玩家名字、单色提示语。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    内容 (Content)Text (文本)直接填入你要显示的默认字。多语言版本可能需要填 ID 键值。✅ 立刻刷新
    Text ID（如有）项目组专属配置，用于调用配表里的多语言化文本。✅ 立刻刷新
    外观 (Appearance)-> FontFont Family (字库)选择游戏规定的字体资产。✅ 立刻刷新
    Size (字号大小)文字的高低像素。✅ 立刻刷新
    Letter Spacing (字间距)拉宽或缩小字符间的距离。✅ 立刻刷新
    Outline Settings (描边)给文字加一圈外轮廓。可以设置描边的粗细(Size)和颜色(Color)。✅ 立刻刷新
    Color (颜色)修改文字的填充颜色。✅ 立刻刷新
    外观 (Appearance)-> ShadowShadow Offset (阴影偏移)字体的阴影位置。比如 X:1 Y:1 会在文字右下方垫一层阴影，提升层次感。✅ 立刻刷新
    Shadow Color (阴影色)通常用纯黑，透明度(A)开到 0.8 左右。✅ 立刻刷新
    对齐 (Justification)Justification控制文字在当前包围盒内是 居左、居中、居右 还是 两端对齐。✅ 立刻刷新
  

  3.2 Rich Text Block (富文本)
  💡 核心作用： 通过 HTML 样式的标签，让一句话里包含多种颜色、加粗，甚至夹杂图片表情。
  🎯 适用场景： 聊天频道文字（带表情包）、掉落提示（获得 倚天剑 x 1）、包含特殊词眼高亮的技能描述。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    内容 (Content)Text (文本)支持写标签，例如：受到 <Red>100</> 点伤害！，它会自动解析对应的标签。✅ 立刻刷新
    外观 (Appearance)Text Style Set (文本样式集)【必须填项】！要在这里挂载美术/策划配好的数据表(Data Table)，里面定义了 <Red> 对应红色，<Gold> 对应金色，否则标签会直接显示为纯文本源码。✅ 立刻刷新
    Decorator Classes (装饰器类)如果你要在文字里插入表情包图标（如 <img id="coin"/>），必须在这里添加项目里做好的 Image Decorator 蓝图类。✅ 立刻刷新
    行为 (Behavior)Auto Wrap Text (自动换行)强烈建议勾选！富文本通常很长，勾选后遇到边界会自动换行往下排。✅ 立刻刷新
  




  🎛️ 4. 数据展示控件（滑条、进度条、组合框、输入框）

  4.1 Slider (滑条)
  💡 核心作用： 提供一段有明确最小、最大值的轨道，让玩家用手指/鼠标拖拽定值。
  🎯 适用场景： 系统设置里的音量调节、亮度调节，或者商城里的购买数量拖动。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    外观 (Appearance)Value (当前值)默认滑块停在什么位置。✅ 立刻刷新
    Min Value / Max Value控制滑轨的最左侧代表什么数值（比如 0），最右侧代表什么数值（比如 100）。❌ 仅逻辑有效
    Widget Style (样式)核心换皮区。在这里替换 Normal Bar Image (滑轨底图) 和 Normal Thumb Image (滑块抓手图)。✅ 立刻刷新
    Orientation (方向)可设为 Horizontal(水平拖动) 或 Vertical(上下垂直拖动)。✅ 立刻刷新
    行为 (Behavior)Step Size (步长)拖动时每次跳跃多少值（比如设为 1，那就只能拖出整数；设为 0.1 则是小数点顺滑）。❌ 需播放运行
  

  4.2 Progress Bar (进度条)
  💡 核心作用： 将数值转化为图形化长度展示给玩家看。
  🎯 适用场景： 角色血条、经验槽、下载加载进度、体力值。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    进度 (Progress)Percent (百分比)核心参数！取值范围永远被锁死在 0.0 ~ 1.0 之间。如果血量是 500/1000，必须在代码里除一下变成 0.5 传给 UI。✅ 立刻刷新
    外观 (Appearance)Fill Color and Opacity进度条“涨起来”的那部分的颜色（比如红色代表血，蓝色代表蓝量）。✅ 立刻刷新
    Style (样式)在这里替换 Background Image (空掉时候的底槽图) 和 Fill Image (实心血条图)。✅ 立刻刷新
    Bar Fill Type (填充方式)可以选择 Left to Right (从左涨到右)、Right to Left (右到左，常用于右侧敌人的血条)、Fill from Center (从中间向两边张开)。✅ 立刻刷新
  

  4.3 ComboBox (组合框)
  💡 核心作用： 提供一个折叠的选项菜单，点击后弹出列表供玩家点选。
  🎯 适用场景： 设置界面的画质选项（低、中、高）、分辨率选择。
  【避坑注意】：ComboBox 自带的弹出列表样式修改极其困难（甚至需要改引擎C++层），因此对 UI 还原度要求高的高级游戏通常不用它，而是自己用按钮做假的弹出层！

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    内容 (Content)Default Options点击加号，往里面填入基础数据（如 "低画质", "中画质", "高画质"）。✅ 立刻刷新
    Selected Option输入其中一个选项名字，代表默认显示选中的是哪一项。✅ 立刻刷新
    外观 (Appearance)Max List Height点开下拉列表后，最大允许向下弹多高（超过就出滚动条）。❌ 需播放运行
    Has Down Arrow是否显示那个难看的倒三角箭头，可以去掉。✅ 立刻刷新
  

  4.4 Editable Text (可编辑文本 / 输入框)
  💡 核心作用： 呼出虚拟键盘/捕获物理键盘，让玩家在界面上打字。
  🎯 适用场景： 创建角色取名字、聊天框、兑换码输入栏、输入密码。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    内容 (Content)Text (文本)输入框里已经存在的实体字。✅ 立刻刷新
    Hint Text (提示字)输入框为空时，显示的灰色指引字（如“请输入兑换码...”），一旦玩家打字就会消失。✅ 立刻刷新
    Is Password (是密码)勾选后，玩家打进去的任何字都会被变成星号 ***。✅ 立刻刷新
    行为 (Behavior)Is Read Only (只读)勾选后，长得像输入框，但点击无法呼出键盘，只能看不能改。❌ 需播放运行
  




  📦 5. 布局容器面板（安全区/尺寸框/切换器/排队框）

  5.1 Overlay (覆层)
  💡 核心作用： 将多个节点像汉堡包一样重叠压在一起，性能极高。
  🎯 适用场景： 所有“底图+内容”的组合（例如：头像底板+英雄头像图+发光边框+右下角等级数字）。
  
  子节点放入 Overlay 后的专属插槽(Slot)参数清单
  
    插槽参数作用与调整方式
    Padding (边界)展开有左右上下。控制该节点距离父容器边缘的缝隙。
    水平对齐 (H-Align)选居左、居中、居右，或最右边的“拉伸 (Fill)”。选拉伸会让控件填满整个宽。
    垂直对齐 (V-Align)选居上、居中、居下，或最右边的“拉伸 (Fill)”。
  

  5.2 Safe Zone (安全区)
  💡 核心作用： 获取设备底层 OS 的硬件信息，自动挤压内部控件，防止 UI 被物理屏幕遮挡。
  🎯 适用场景： 作为整个 UMG 面板“背景层之上，内容层之下”的总包裹容器。
  
  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节
    安全区 (Safe Zone)Pad Left (保护左边)勾选后，如果手机左侧有刘海，左边会自动留出缝隙。
    Pad Right (保护右边)【微信小游戏极度重要】右侧必定有微信系统的胶囊按键，必须勾选此项，以防右上角的关闭按钮被胶囊按键盖住点不到！
    Pad Top / Bottom保护顶部状态栏 / 底部圆角手势区域。
  

  5.3 Widget Switcher (控件切换器)
  💡 核心作用： 抽屉式翻页。可以往里面塞 N 个子面板，但同一时间永远只渲染并显示 1 个。
  🎯 适用场景： 游戏主界面底部导航（英雄面板 / 抽卡面板 / 商城面板 的无缝切换）。
  
  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    切换器 (Switcher)Active Widget Index (激活索引)填入数字。0 代表显示放在它下面的第1个面板，1 代表显示第2个。在编辑器里更改此数字可立即看切页效果！✅ 立刻刷新
  

  5.4 Size Box (尺寸框)
  💡 核心作用： 当控件身处“强制拉伸”或“自动压缩”的父容器中时，给它套上一层无形的钢铁外壳，强制锁死物理大小。
  🎯 适用场景： 放在 Horizontal Box 里自动排队的按钮，想强制规定每个按钮尺寸为 120x50 像素。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    子布局 (Child Layout)Width / Height Override勾选并填入数值，强制把里面的子节点框死在这个长宽之下（比如 120和50）。✅ 立刻刷新
    Min Desired Width / Height设定最小值（比如最小必须保证 50 宽，如果文字很少也不允许压得更小）。✅ 立刻刷新
    Max Desired Width / Height设定最大值（用于做聊天气泡：文字一直变长气泡跟着拉宽，但最宽不超过 400，超过就换行）。✅ 立刻刷新
  

  5.5 Horizontal / Vertical Box (水平/垂直排队框)
  💡 核心作用： 扔进里面的所有子节点，自动排队（HBox向右排，VBox向下排），完全不用手动算坐标。
  🎯 适用场景： 左上角从左到右的头像+金币图标+数值列；从上到下罗列的任务目标条目。
  
  子节点放入 H/V Box 后的专属插槽(Slot)参数清单
  
    插槽参数作用与调整方式
    大小 (Size)Auto (自动)：它有多大就占多少排队空间。Fill (填充)：霸道占座！自动吃掉框里所有剩下的多余空间。
    Padding (边界)用来控制排队的元素之间的空隙。比如在第二个元素设置 Left Padding 10，就能和前一个隔开 10 像素。
  




  ♾️ 6. 高级列表视图（List View / Tile View）
  
    ⚠️ 核心理念：对于超长列表（如排行榜、背包），决不能用 Scroll Box 无脑塞，会卡死引擎。必须使用对象池复用的 ListView (单列) 或 TileView (九宫格瓦片)。
  

  6.1 List View (列表视图) & Tile View (瓦片视图)
  💡 核心作用： 数据驱动的高性能滚动展示容器。即使有 10000 条数据，引擎也只会渲染屏幕可见的那十几个格子，滑动时不断复用更新。
  🎯 适用场景： 邮件列表、公会成员名单（ListView）；大型物品背包、英雄图鉴九宫格（TileView）。

  右侧面板详细参数清单
  
    面板分组具体参数名效果与如何调节即时预览
    列表 (List)Entry Widget Class (条目类)【唯一生死核心】！绝不允许手动在列表下挂子节点！必须在这里选定一个事先做好的、实现了 User Object List Entry 接口的蓝图类（比如 WBP_BagItem）。✅ 假数据刷新
    Orientation (滚动方向)控制列表是上下滑动(Vertical)还是左右滑动(Horizontal)。✅ 立刻刷新
    Tile View 专属Entry Height / Width由于 TileView 是一行排几个，必须在这里锁死每一个“格子”的绝对物理尺寸（例如 100x100），引擎才能知道一行能塞下几个。✅ 立刻刷新
    Tile Alignment如果最后一行只有 2 个格子不满，控制它们是挤在左边(Left)，居中(Center) 还是分散对齐(Fill)。✅ 立刻刷新
    Tile Wrap / Scroll Dir通常瓦片视图都是上下滚动，遇到右边缘换行。✅ 立刻刷新
  




  👁️ 7. 通用核心属性：可见性 (Visibility) 与 变换
  所有控件都有可见性和渲染变换属性，这是 UI 交互的基础：
  
    可见性选项视觉状态点击状态性能消耗最适用场景
    可见 (Visible)👀 看得见👆 挡击/可点高普通可交互按钮、弹窗面板。
    折叠 (Collapsed)❌ 看不见❌ 彻底消失极低隐藏UI首选！完全不计算也不占排队空间。
    隐藏 (Hidden)❌ 看不见❌ 无法点击中看不见但仍然占据排队占位空间（占坑）。
    不可见命中测试(Hit Test Invisible)👀 看得见👻 鼠标穿透低纯装饰用的背景图！不挡住下方的按钮。
    自身不可见命中测试(Self Hit Test Invisible)👀 看得见👻 自身穿透👆 子可点低作为父容器（Canvas/Overlay/SizeBox等）的最佳选择！
  
  避坑指南：点不动按钮？99%是因为上面盖了张 Visible 的全屏透明图，把它改成 Hit Test Invisible！

  渲染变换 (Render Transform) 的妙用
  当你想让一个按钮往右偏移 10 个像素，是改插槽位置，还是改渲染变换(Translation X)？
  规范： 排版布局请永远用插槽和 Padding。渲染变换只留给“播放动效（UI Animation）”时使用！ 因为渲染变换不影响底层排版的占位，如果用来拼界面，一旦屏幕拉伸，控件就会出现意想不到的脱节偏移。




  🎯 8. 核心实操：锚点、对齐与多端/小游戏适配规范
  在不同分辨率屏幕下保持界面不拉伸，核心就是锚点对齐。尤其是多端游戏和微信小游戏（手机屏幕存在大量异形屏），必须遵循严格的拼图规范。
  
  8.1 锚点 (Anchor) 九宫格对照表
  锚点决定了控件的“根”扎在屏幕的哪个部位。屏幕拉宽/变高时，控件会死死保持与锚点的距离。

  8.2 对齐 (Alignment) —— 绝对居中的密码
  当你要把弹窗绝对居中时，光把锚点设为“居中”（九宫格最中间那个）是不够的！因为那只是以控件的左上角点对齐屏幕中心，弹窗还是偏右下。
  【核心铁律】必须同时将 Alignment (对齐) 设为 X: 0.5, Y: 0.5。这代表用控件自己的正中心去贴合锚点。这是所有居中弹窗 UI 必须遵守的铁律！

  8.3 多端与微信小游戏适配专属规范
  
    📱 微信小游戏/手游端必须遵循的层级树：
    由于微信小游戏在带鱼屏手机上有严重差异，且右上角必定有一个不可隐去的“胶囊按钮（更多/关闭）”，如果乱摆放会导致界面右上角按键完全被微信系统挡死。
  
  
  标准根节点搭建规范范例：
  
[根容器] Canvas Panel (画板)
 ├── 🖼️ Image [背景大底图] 
 │    └─ 规范：锚点选「四边全屏拉伸(右下大方块)」，Offsets全部归0。它无视刘海，直接铺满全屏幕作为视觉底盘。
 ├── 🛡️ Safe Zone [安全区] 
 │    ├─ 规范：紧接背景之上，必须勾选四边Pad，系统会自动挤开微信胶囊和刘海屏幕的黑边。
 │    └─ 🥞 Overlay [全屏覆层] (放在SafeZone里面，对齐设为四边拉伸满)
 │         ├── 📦 VBox/HBox (主界面的核心战斗面板、技能，在此内部绝对安全)
 │         └── 👆 Button [退出/设置/返回按钮] 
 │              └─ 规范：即使锚点设为右上角，因为有SafeZone保护，绝对会和微信的胶囊按钮错开！
